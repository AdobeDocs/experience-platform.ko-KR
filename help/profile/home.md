# 실시간 고객 프로필 개요

Adobe Experience Platform을 사용하면 고객과 브랜드 간의 상호 작용 시기와 상관없이 고객의 기대에 부응하는 경험을 일관되게 제공할 수 있습니다. 실시간 고객 프로파일을 이용하면 온라인, 오프라인, CRM, 서드파티 데이터 등 다양한 채널의 데이터를 결합하는 개별 고객의 전체 상황을 파악할 수 있습니다. 프로필을 사용하면 서로 다른 고객 데이터를 하나의 통합 뷰로 통합하여 고객 상호 작용에 대한 실행 가능하고 타임스탬프가 지정된 계정을 제공할 수 있습니다. 이 개요는 Adobe Experience Platform에서 실시간 고객 프로필의 역할과 사용을 이해하는 데 도움이 됩니다.

## 실시간 고객 프로파일 이해

실시간 고객 프로필은 다양한 엔터프라이즈 데이터 자산의 데이터를 수집한 다음 개별 고객 프로파일 및 관련 시계열 이벤트의 형태로 해당 데이터에 대한 액세스를 제공하는 범용 조회 엔티티 스토어입니다. 이 기능을 통해 마케터는 아래 비디오에 요약된 바와 같이 다양한 채널에서 고객과 연관성 있고 조율된 경험을 제공할 수 있습니다.

>[!VIDEO](https://video.tv.adobe.com/v/27251?quality=12&enable10seconds=on&speedcontrol=on)

### 프로필 데이터 저장소

실시간 고객 프로필은 인제스트된 데이터를 처리하고 Adobe Experience Platform Identity Service를 사용하여 ID 매핑을 통해 관련 데이터를 병합하지만 프로필 스토어에 자체 데이터를 유지합니다. 즉, 프로필 저장소는 카탈로그 데이터(Data Lake) 및 ID 서비스 데이터(ID 그래프)와 별개입니다.

### 프로필 및 플랫폼 서비스

실시간 고객 프로필과 Experience Platform 내의 다른 서비스 간의 관계는 다음 다이어그램에서 강조 표시되어 있습니다.

![프로필과 기타 Experience Platform 서비스 간의 관계.](images/profile-overview/profile-in-platform.png)

### 프로필 및 데이터 기록

프로파일은 제목, 조직 또는 개인을 레코드 데이터라고도 합니다. 예를 들어 제품 프로필에는 SKU와 설명이 포함될 수 있지만 사람의 프로필에는 이름, 성 및 이메일 주소와 같은 정보가 포함되어 있습니다. Adobe Experience Platform을 사용하면 프로파일을 사용자 정의하여 비즈니스와 관련된 데이터 유형을 사용할 수 있습니다. 표준 경험 데이터 모델(XDM) 개별 프로필 클래스는 고객 레코드 데이터를 설명할 때 스키마를 작성할 수 있는 기본 클래스로, 플랫폼 서비스 간의 여러 상호 작용에 데이터 통합 요소를 제공합니다. 경험 플랫폼에서 스키마 작업에 대한 자세한 내용은 XDM 시스템 개요를 [읽어 보시기](../xdm/home.md)바랍니다.

### 시계열 이벤트

시간 시리즈 데이터는 작업을 직접 또는 간접적으로 수행한 당시의 시스템 스냅샷뿐만 아니라 이벤트 자체에 대한 자세한 데이터를 제공합니다. 표준 스키마 클래스 XDM ExperienceEvent로 표현되는 시간 시리즈 데이터는 장바구니에 추가되는 항목, 클릭되는 링크 및 본 비디오와 같은 이벤트를 설명할 수 있습니다. 시간 시리즈 데이터를 사용하여 세분화 규칙을 기반으로 할 수 있으며, 이벤트는 프로파일 컨텍스트에서 개별적으로 액세스할 수 있습니다.

### ID

모든 기업은 개인화된 방식으로 고객과 커뮤니케이션하고 싶어합니다. 그러나 고객에게 연관성 있는 디지털 경험을 전달하는 데 있어 당면 과제 중 하나는 단절된 데이터를 서로 연결하는 방법을 이해하는 것입니다. 이러한 데이터는 태블릿과 휴대폰, 노트북과 같은 다양한 디지털 채널에 걸쳐 널리 퍼집니다. Identity Service를 사용하면 여러 채널의 ID를 연결하여 각 고객에 대한 ID 그래프를 만들어 고객의 전체 상황을 파악할 수 있습니다. 자세한 내용은 [ID 서비스 개요를](../identity-service/home.md) 참조하십시오.

### 세그먼테이션

Adobe Experience Platform Segmentation Service는 개별 고객을 위한 경험을 제공하는 데 필요한 고객을 제공합니다. 대상 세그먼트가 만들어지면 해당 세그먼트의 ID가 모든 자격 프로필에 대한 세그먼트 구성원 목록에 추가됩니다. 세그먼트 규칙은 RESTful API 및 세그먼트 빌더 사용자 인터페이스를 사용하여 실시간 고객 프로필 데이터에 작성 및 적용됩니다. 세그멘테이션에 대한 자세한 내용은 세그멘테이션 서비스 [개요를](../segmentation/home.md)읽으십시오.

### 프로필 조각 및 결합 보기 {#profile-fragments-and-union-schemas}

실시간 고객 프로필의 주요 기능 중 하나는 멀티채널 데이터를 통합하는 기능입니다. 실시간 고객 프로필을 사용하여 엔티티에 액세스하면 통합 보기라고 하는 데이터 세트에서 해당 엔티티에 대한 모든 프로필 조각에 대한 통합 보기를 제공할 수 있습니다. 실시간 고객 프로필 데이터는 엔티티 또는 프로파일이 ID로 액세스되거나 세그먼트로 내보내지면 소스 간에 병합됩니다. 프로필 및 조합 보기에 액세스하는 방법에 대한 자세한 내용은 &quot;프로필 액세스&quot;라고도 하는 [엔티티의 실시간 고객 프로필 API 개발자 하위 안내서를 참조하십시오](api/entities.md).

### 정책 병합

여러 소스에서 데이터를 취합하여 각 개별 고객의 전체 상황을 파악할 수 있도록 결합하는 경우 병합 정책은 Platform에서 데이터의 우선 순위가 어떻게 지정되고 통합 뷰를 구축하기 위해 어떤 데이터가 결합될 것인지를 결정하는 규칙입니다. RESTful API 또는 사용자 인터페이스를 사용하여 새 병합 정책을 만들고 기존 정책을 관리하고 조직에 대한 기본 병합 정책을 설정할 수 있습니다. API를 사용한 병합 정책 작업에 대한 자세한 내용은 실시간 고객 프로필 API [병합 정책 하위 가이드](api/merge-policies.md) 또는 플랫폼 UI를 사용하여 병합 정책을 사용하는 방법에 대한 [병합 정책 사용 안내서를](ui/merge-policies.md) 참조하십시오.

## 실시간 구성 요소

이 섹션에서는 실시간 고객 프로파일을 통해 기록 및 시간 시리즈 데이터를 실시간으로 업데이트하고 모니터링할 수 있는 구성 요소를 소개합니다.

### 스트리밍 통합 및 스트리밍 세분화

실시간 입력은 스트리밍 통합 프로세스를 통해 가능합니다. 프로필 및 시간 시리즈 데이터가 수집되면 실시간 고객 프로필은 기존 데이터와 병합하고 통합 뷰를 업데이트하기 전에 스트리밍 세그먼테이션이라는 진행 중인 프로세스를 통해 해당 데이터를 세그먼트에서 포함하거나 제외하기로 자동으로 결정합니다. 따라서 계산을 즉시 수행하고 브랜드와 상호 작용할 때 개인화된 향상된 경험을 고객에게 제공하는 결정을 내릴 수 있습니다. 인제스트하는 동안 데이터도 제대로 인제스트되고 데이터 세트가 기반이 되는 스키마를 따르는지 확인하기 위해 유효성 검사를 받습니다. 통합 중에 수행되는 유효성 검사에 대한 자세한 내용은 [데이터 통합 품질 개요를](../ingestion/quality/overview.md)읽으십시오.

### 가장자리 예상

여러 채널에서 실시간으로 고객의 기대에 부응하는 개인화된 경험을 제공하기 위해서는 변경 사항이 발생하면 즉시 정확한 데이터를 제공하고 지속적으로 업데이트해야 합니다. Adobe Experience Platform을 사용하면 데이터를 실시간으로 액세스할 수 있습니다. Edge는 지리적으로 배치된 서버로서 데이터를 저장하고 애플리케이션에서 쉽게 액세스할 수 있습니다. 예를 들어 Adobe Target 및 Adobe Campaign과 같은 Adobe 애플리케이션은 실시간으로 개인화된 고객 경험을 제공하기 위해 가장자리를 사용합니다. 데이터는 투영에 의해 가장자리로 라우팅되고, 투영 대상은 데이터를 전송할 가장자리를 정의하고, 모서리에서 사용할 수 있는 특정 정보를 정의하는 투영 구성을 사용합니다.

자세한 내용을 살펴보고 가장자리 및 예상 작업을 시작하려면 실시간 고객 프로필 API 에지 [예측 하위 안내서를](api/edge-projections.md)참조하십시오.

## 데이터 거버넌스 및 개인 정보

데이터 거버넌스는 고객 데이터를 관리하고 데이터 사용에 적용되는 규정, 제한 및 정책을 준수하는 데 사용되는 일련의 전략과 기술입니다.

데이터 액세스와 관련된 데이터 거버넌스는 다양한 수준에서 경험 플랫폼 내에서 중요한 역할을 합니다.
* 데이터 사용 레이블 지정
* 데이터 액세스 정책
* 마케팅 작업을 위한 데이터 액세스 제어

데이터 거버넌스는 여러 지점에서 관리됩니다. 이러한 데이터에는 인제스트된 데이터와 해당 마케팅 활동을 위해 인제스트 후 액세스할 수 있는 데이터를 결정하는 것이 포함됩니다. 자세한 내용은 [데이터 거버넌스 개요를](../data-governance/home.md)읽으십시오.

### 옵트아웃 및 데이터 개인 정보 보호 요청 처리

Adobe Experience Platform을 통해 고객은 실시간 고객 프로필 내에서 데이터의 사용 및 저장과 관련된 옵트아웃 요청을 보낼 수 있습니다. 옵트아웃 요청을 처리하는 방법에 대한 자세한 내용은 옵트아웃 요청 [수집에](../segmentation/honoring-opt-outs.md)대한 설명서를 참조하십시오.

## 실시간 고객 프로필에 데이터 추가

플랫폼에서는 실시간 스트리밍 통합 및 일괄 처리를 지원하여 기록 및 시간 시리즈 데이터를 프로필로 보내도록 구성할 수 있습니다. 자세한 내용은 실시간 고객 프로필에 데이터를 [추가하는 방법을 요약하는 자습서를 참조하십시오](tutorials/add-profile-data.md).

>[!Note]
>Analytics Cloud, Marketing Cloud 및 Advertising Cloud를 비롯한 Adobe 솔루션을 통해 수집된 데이터는 Experience Platform으로 전송되어 프로필에 수집됩니다.

## 고객 세그먼트 만들기

마케팅 캠페인의 핵심은 고객입니다. 실시간 고객 프로필은 고객 기반을 사용자가 필요로 하는 정확한 기준을 충족하는 구성원으로 구성된 대상으로 세그먼트화하는 도구를 제공합니다. 세그멘테이션을 사용하면 다음과 같은 조건을 사용하여 대상자 구성원을 분리할 수 있습니다.

* 마지막으로 구매한 후 1주가 지난 고객
* 구매 금액이 $10,000보다 큰 고객
* 캠페인 ID로 지정된 사전 정의된 목록에서 설정된 수의 고유 마케팅 캠페인을 보고 30분 이내에 탐색한 고객

세그멘테이션을 시작하려면 [세그멘테이션 개요를](../segmentation/home.md)참조하십시오.

## (알파) 계산된 속성 구성

>[!IMPORTANT]
>이 문서에 설명된 계산된 속성 기능은 alpha입니다. 설명서 및 기능은 변경될 수 있습니다.

계산된 속성을 사용하면 다른 값, 계산 및 표현식을 기반으로 필드의 값을 자동으로 계산할 수 있습니다. 계산된 속성은 프로필 수준에서 작동하며, 이는 모든 레코드와 이벤트에 대해 값을 집계할 수 있음을 의미합니다. 계산된 각 속성에는 들어오는 데이터를 평가하고 결과 값을 프로필 속성 또는 이벤트에 저장하는 표현식 또는 &quot;규칙&quot;이 포함됩니다. 이러한 계산을 사용하면 정보가 필요할 때마다 복잡한 계산을 수동으로 수행하지 않고도 라이프타임 구매 값, 구매 간 시간, 애플리케이션 열기 횟수 등과 같은 문제와 관련된 질문에 쉽게 응답할 수 있습니다. 계산된 속성에 대한 자세한 내용 및 이를 사용한 단계별 지침은 계산된 속성에 대한 실시간 고객 프로필 API [하위 안내서를 참조하십시오](api/computed-attributes.md). 이 안내서는 Adobe Experience Platform에서 작동하는 역할 계산 속성을 보다 잘 이해하는 데 도움이 되며 실시간 고객 프로필 API를 사용하여 기본 CRUD 작업을 수행하기 위한 샘플 API 호출을 포함합니다.
