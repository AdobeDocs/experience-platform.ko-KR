# [!DNL Platform] 가설 [!DNL Real-time Customer Profile]

[!DNL Real-time Customer Profile] 행동 인사이트와 고객 속성을 기반으로 개인화된 크로스채널 경험을 제공할 수 있는 개별 프로파일을 제공합니다. 이러한 타깃팅을 수행하기 위해, Adobe Experience Platform [!DNL Profile] 의 세그멘테이션 엔진은 고도로 표준화된 하이브리드 데이터 모델을 사용하므로 고객 프로파일을 개발하는 데 새로운 방법을 제공합니다. 이 하이브리드 데이터 모델을 사용하면 수집 중인 데이터가 올바르게 모델링되는 것이 매우 중요합니다. 프로필 데이터를 유지 관리하는 [!DNL Profile] 데이터 저장소는 관계형 저장소가 아니지만, 세그먼트를 단순화하고 직관적으로 만들기 위해 작은 차원 엔티티와의 통합을 [!DNL Profile] 허용합니다. 이 통합을 다중 엔티티 세그먼테이션이라고 합니다.

Adobe Experience Platform은 지원할 수 없는 데이터 모델을 만들지 않도록 하는 데 도움이 되는 일련의 [!DNL Real-time Customer Profile] 지침을 제공합니다. 이 문서에서는 차원 엔티티, 특히 일괄 세그먼테이션에서 사용할 때의 우수 사례 및 제한 사항에 대해 간략히 설명합니다.

>[!NOTE]
>
>이 문서에 나와 있는 보증과 제한 사항은 지속적으로 개선되고 있습니다. 업데이트를 정기적으로 확인하시기 바랍니다.

## 시작하기

에서 사용할 데이터 모델을 만들기 전에 다음 Experience Platform 서비스 설명서를 읽는 것이 좋습니다 [!DNL Real-time Customer Profile]. 데이터 모델 및 이 문서에 설명된 지침을 사용하려면 개체 관리와 관련된 다양한 Experience Platform 서비스에 대한 이해가 [!DNL Real-time Customer Profile] 필요합니다.

* [[!DNL 실시간 고객 프로필]](home.md):여러 소스에서 집계된 데이터를 기반으로 통합된 실시간 소비자 프로필을 제공합니다.
* [Adobe Experience Platform ID 서비스](../identity-service/home.md):인제스트할 때 서로 다른 데이터 소스에서 ID를 연결하여 &quot;고객에 대한 단일 뷰&quot;를 만들 수 있습니다 [!DNL Platform].
* [[!DNL 경험 데이터 모델(XDM)]](../xdm/home.md):Platform(플랫폼)이 고객 경험 데이터를 구성하는 표준화된 프레임워크입니다.
   * [스키마 컴포지션의 기본 사항](../xdm/schema/composition.md):Experience Platform 내의 스키마 및 데이터 모델링에 대한 소개
* [Adobe Experience Platform 세그멘테이션 서비스](../segmentation/home.md):고객 행동 및 특성을 기반으로 고객 프로파일에서 고객 세그먼트를 만드는 데 [!DNL Platform] 사용되는 세그먼트 엔진입니다.
   * [다중 엔티티 세그먼테이션](../segmentation/multi-entity-segmentation.md):차원 엔티티를 프로필 데이터와 통합하는 세그먼트를 만드는 지침입니다.

## 엔티티 유형

스토어 데이터 모델은 두 가지 핵심 엔티티 유형으로 구성됩니다. [!DNL Profile]

* **기본 엔티티:** 기본 엔티티 또는 프로필 엔티티는 데이터를 결합하여 개인에 대한 &quot;하나의 진실&quot;을 형성합니다. 이 통합 데이터는 &quot;조합 뷰&quot;라고 하는 것을 사용하여 표현됩니다. 조합 뷰는 동일한 클래스를 구현하는 모든 스키마의 필드를 단일 조합 스키마로 집계합니다. 조합 스키마 [!DNL Real-time Customer Profile] 는 모든 프로필 특성 및 행동 이벤트에 대한 컨테이너로 작동하는 비정규화된 하이브리드 데이터 모델입니다.

   &quot;레코드 데이터&quot;라고도 하는 시간 독립적인 속성은 모델을 사용하여 모델링되고 시간 시리즈 데이터 [!DNL XDM Individual Profile]는 &quot;이벤트 데이터&quot;라고도 합니다 [!DNL XDM ExperienceEvent]. Adobe Experience Platform에서 기록 및 시간 시리즈 데이터가 수집되면 데이터 [!DNL Real-time Customer Profile] 가 사용하도록 설정된 데이터 인제스트를 시작합니다. 수집되는 인터랙션과 세부 사항이 많을수록 더욱 강력한 개별 프로필이 만들어집니다.

   ![](images/guardrails/profile-entity.png)

* **Dimension 엔티티:** 또한 조직은 XDM 클래스를 정의하여 매장, 제품 또는 속성과 같이 개인 이외의 사항을 설명할 수 있습니다. 이러한 비[!DNL XDM Individual Profile] 스키마는 &quot;차원 엔티티&quot;라고 하며 시간 시리즈 데이터를 포함하지 않습니다. Dimension 엔터티는 다중 엔티티 세그먼트 정의를 보조하고 단순화하는 조회 데이터를 제공하며, 세그멘테이션 엔진이 최적 처리를 위해 전체 데이터 세트를 메모리에 로드할 수 있을 만큼 작아야 합니다(빠른 지점 조회).

   ![](images/guardrails/profile-and-dimension-entities.png)

## 제한 유형

데이터 모델을 정의할 때는 제공된 지침에 따라 적절한 성능을 유지하고 시스템 오류를 방지하는 것이 좋습니다. 이 문서에서 제공하는 보증에는 두 가지 제한 유형이 있습니다.

* **소프트 한도:** 시스템 성능을 최적화하기 위해 권장 최대값을 제공하는 소프트 한도 시스템을 중단하거나 오류 메시지를 받지 않고 소프트 한도를 넘을 수도 있지만, 소프트 한도를 넘어가면 성능 저하가 발생합니다. 전반적인 성능 저하를 방지하려면 소프트 한도 내에 있는 것이 좋습니다.

* **한도:** 한도 제한은 시스템의 절대 최대값을 제공합니다. 하드 한도를 넘어서면 고장 및 오류가 발생하여 시스템이 예상대로 작동하지 않습니다.

## 데이터 모델 보장

에서 사용할 데이터 모델을 만들 때는 다음 지침을 준수하는 것이 좋습니다 [!DNL Real-time Customer Profile].

### 기본 엔티티 보증도

| 가드레일 | 제한 | 제한 유형 | 설명 |
| --- | --- | --- | --- |
| 조합 스키마에 기여하는 데 권장되는 데이터 [!DNL Profile] 수 | 20 | 부드러운 | **데이터 세트를 최대 20[!DNL Profile]개까지 사용하는 것이 좋습니다.** 다른 데이터 세트를 사용하려면 먼저 기존 데이터 세트 [!DNL Profile]를 제거하거나 비활성화해야 합니다. |
| 권장되는 다중 엔티티 관계 수 | 5 | 부드러운 | **기본 엔티티와 차원 엔티티 사이에 정의된 최대 5개의 다중 엔티티 관계가 권장됩니다.** 기존 관계가 제거되거나 비활성화될 때까지 추가적인 관계 매핑을 해서는 안 됩니다. |
| 다중 엔티티 관계에 사용된 ID 필드의 최대 JSON 깊이 | 4 | 부드러운 | **다중 엔티티 관계에 사용된 ID 필드에 대한 권장 최대 JSON 깊이는 4입니다.** 즉, 중첩된 스키마에서는 수준 깊이가 4개 이상인 중첩된 필드를 관계에 있는 ID 필드로 사용할 수 없습니다. |
| 프로필 조각의 배열 카디널리티 | &lt;=500 | 부드러운 | **프로필 조각에서 최적의 배열 카디널리티(시간 독립적인 데이터)는 &lt;=500입니다.** |
| ExperienceEvent의 배열 카디널리티 | &lt;=10 | 부드러운 | **ExperienceEvent(시간 시리즈 데이터)의 최적 배열 카디널리티는 &lt;=10입니다.** |

### Dimension 실체

| 가드레일 | 제한 | 제한 유형 | 설명 |
| --- | --- | --- | --- |
| 비엔티티에 대해 시간 시리즈 데이터가 허용되지[!DNL XDM Individual Profile] 않음 | 0 | 하드 | **프로필 서비스의 비엔터티에는 시간[!DNL XDM Individual Profile]계열 데이터가 허용되지 않습니다.** 시간 시리즈 데이터 집합이 ID가 아닌 ID와[!DNL XDM Individual Profile] 연결된 경우 데이터 집합에 대해 활성화되지 않아야 합니다 [!DNL Profile]. |
| 중첩 관계 없음 | 0 | 부드러운 | **두 가지 비-스키마 간에 관계를[!DNL XDM Individual Profile]만들면 안 됩니다.** 관계 생성 기능은 조합 스키마에 속하지 않는 모든 스키마에는 권장되지 [!DNL Profile] 않습니다. |
| 기본 ID 필드의 최대 JSON 깊이 | 4 | 부드러운 | **기본 ID 필드에 대한 권장 최대 JSON 깊이는 4입니다.** 즉, 중첩된 스키마에서는 필드가 4개 수준 보다 깊이 중첩된 경우 기본 ID로 선택하지 않아야 합니다. 중첩 수준이 4번째인 필드는 기본 ID로 사용할 수 있습니다. |

## 데이터 크기 보장

다음 보증서는 데이터 크기를 참조하며 의도한 대로 데이터를 수집, 저장 및 쿼리할 수 있도록 권장합니다.

>[!NOTE]
>
>데이터 크기는 수집 시 JSON에서 압축되지 않은 데이터로 측정됩니다.

### 기본 엔티티 보증도

| 가드레일 | 제한 | 제한 유형 | 설명 |
| --- | --- | --- | --- |
| 프로필 조각당 최대 크기 | 10KB | 부드러운 | **프로필 조각의 권장 최대 크기는 10KB입니다.** 큰 프로필 조각을 인제스트하면 시스템 성능에 영향을 줍니다. 예를 들어 일부 프로필 조각의 크기가 50kB인 무거운 CRM 데이터 세트를 로드하면 시스템 성능이 저하됩니다. |
| 프로필 조각당 절대 최대 크기 | 1MB | 하드 | **프로필 조각의 절대 최대 크기는 1MB입니다.** 1MB보다 큰 프로필 조각을 업로드하려고 하면 인제스트가 실패합니다. |

### Dimension 실체

| 가드레일 | 제한 | 제한 유형 | 설명 |
| --- | --- | --- | --- |
| 모든 차원 엔터티의 최대 총 크기 | 5GB | 부드러운 | **모든 차원 엔터티의 최대 권장 총 크기는 5GB입니다.** 큰 차원 엔티티를 인제스트하면 시스템 성능이 저하됩니다. 예를 들어 10GB 제품 카탈로그를 차원 엔티티로 로드하려는 것은 권장되지 않습니다. |
| 차원 엔티티 스키마당 데이터 집합 | 5 | 부드러운 | **각 차원 엔티티 스키마와 연결된 최대 5개의 데이터 세트가 권장됩니다.** 예를 들어 &quot;제품&quot;에 대한 스키마를 만들고 5개의 기여 데이터 집합을 추가하는 경우 제품 스키마에 연결된 6번째 데이터 집합을 만들 수 없습니다. |
