# 작업 일정 검사

>[!AVAILABILITY]
>
>[!UICONTROL Job schedules]은(는) 현재 제한된 릴리스로 사용할 수 있으며 다음 Real-Time CDP 작업에만 사용할 수 있습니다.
>
> * 일괄 데이터 레이크 수집
> * 일괄 프로필 수집
> * 일괄 처리 세분화
> * 일괄 처리 대상 활성화.

[!UICONTROL Job Schedules]은(는) 수집에서 대상 활성화에 이르기까지 데이터 파이프라인의 모든 예약된 일괄 처리 작업을 통합적으로 볼 수 있습니다. 비즈니스 운영에 영향을 미치기 전에 실행 상태를 검사하고, 예약 충돌을 식별하고, 구성 문제를 진단합니다.

작업 일정을 사용하여 실패를 조사하고, 작업 타이밍을 최적화하고, 데이터 레이크 수집, 프로필 처리, 세분화 및 대상 활성화 간의 종속성을 이해합니다. 일반적인 구성 문제를 해결하는 방법에 대한 지침은 [작업 일정 방지 패턴 식별](job-schedules-anti-patterns.md)에 대한 설명서를 참조하십시오.

## 전제 조건 {#prerequisites}

[!UICONTROL Job Schedules]에 액세스하려면 **[!UICONTROL View Job Schedules]** 및 **[!UICONTROL View Profile Management]** [액세스 제어 권한](/help/access-control/home.md#permissions)이 필요합니다.

적절한 권한이 있는지 확인하려면 시스템 관리자에게 문의하십시오.

## 시작 {#getting-started}

[!UICONTROL Job Schedules]을(를) 사용하기 전에 다음 Experience Platform 개념을 숙지해야 합니다.

* **[일괄 처리 수집](../ingestion/batch-ingestion/overview.md)**: 예약된 간격으로 데이터가 데이터 레이크 및 프로필 저장소에 로드되는 방식.
* **[세그먼테이션](../segmentation/home.md)**: 프로필 데이터 및 세그먼트 정의를 기반으로 대상을 평가하고 업데이트하는 방법입니다.
* **[실시간 고객 프로필](../profile/home.md)**: 프로필 데이터를 통합하고 세분화 및 활성화에 사용하는 방법입니다.
* **[대상](../destinations/home.md)**: 데이터를 다운스트림 시스템 및 마케팅 플랫폼으로 활성화하는 위치 및 방법.

이러한 구성 요소를 이해하면 작업 실행 패턴을 해석하고 문제가 발생할 때 진단할 수 있습니다.

## 작업 스케줄 인터페이스 이해 {#understanding-interface}

[!UICONTROL Job Schedules]에 액세스하려면:

1. Experience Platform UI의 왼쪽 탐색에서 **[!UICONTROL Run and Operate]**&#x200B;을(를) 선택합니다.
2. **[!UICONTROL Job Schedules]**&#x200B;를 선택합니다.

[!UICONTROL Job Schedules] 페이지에서는 모든 예약된 일괄 처리 작업에 대한 개요를 제공합니다.

![왼쪽 탐색 실행](assets/job-schedules/run-and-operate-left-nav.png)

### 요약 카드 {#summary-cards}

페이지 맨 위에는 일괄 처리 작업에 대한 빠른 인사이트를 제공하는 요약 카드가 표시됩니다.

![일괄 처리 작업에 대한 통찰력을 보여주는 작업 일정 요약 카드](assets/job-schedules/job-schedules-cards.png)

* **레이크 수집 실행**: 실행된 데이터 레이크 수집 작업 수입니다.
* **프로필 수집 실행**: 실행된 프로필 수집 작업 수입니다.
* **다음 세분화**: 예약된 다음 세분화 작업이 실행될 때.
* **다음 대상 활성화**: 예약된 다음 대상 활성화 작업이 실행될 때입니다.

이러한 카드를 통해 데이터 파이프라인의 활동 및 예정된 일정을 이해할 수 있습니다. **레이크 수집 실행** 및 **프로필 수집 실행**&#x200B;의 값이 선택한 시간 간격(오늘, 어제 또는 최근 7일)에 따라 변경됩니다. 다음에 실행되는 카드(**다음 세분화** 및 **다음 대상 활성화**)는 시간 선택기의 영향을 받지 않습니다.

### 기간 선택기 {#time-period}

기간 선택기를 사용하여 예약된 작업을 살펴볼 이전 시간을 선택합니다.

![작업 일정에서 기간 선택기 UI의 애니메이션 예](assets/job-schedules/time-selector.gif)

* **오늘**: 오늘 예약된 작업을 봅니다(기본 보기).
* **어제**: 어제 실행된 작업을 봅니다.
* **최근 7일**: 지난 주의 작업을 봅니다.

### 배치 작업 일정 세부 정보 {#job-schedules-details}

기본 보기는 배치 작업이 하루 종일 실행되도록 예약된 시간을 보여 줍니다. 다음과 같은 작업을 수행할 수 있습니다.

* **데이터 세트 또는 엔터티별로 작업 보기**: 왼쪽 열에는 데이터 세트 또는 처리 작업(예: 수집 데이터 세트 또는 세그먼테이션 작업)의 이름이 표시됩니다.
* **작업 시간 표시**: 타임라인은 각 작업이 실행하도록 예약된 시간을 표시하며 시각적 표시기가 예약된 시간을 표시합니다.
* **작업 필터링**: 필터 아이콘을 사용하여 보고서에 포함할 데이터 세트의 범위를 좁힙니다.
* **작업 유형 이해**: 맨 아래에 있는 색상으로 구분된 범례는 다른 작업 유형을 식별하는 데 도움이 됩니다.
   * **레이크 수집**(녹색): 데이터 레이크로 데이터 수집
   * **프로필 수집**(분홍색): 프로필 스토어로 데이터 수집
   * **세그먼테이션**(연한 파란색): 대상 평가 작업
   * **프로필 내보내기**(파란색): 프로필 데이터 내보내기
   * **활성화**(진한 회색): 대상 활성화 작업
   * **진행 중**(스트라이핑): 현재 실행 중이거나 대기 중인 작업

이 타임라인 보기를 통해 일정 충돌을 식별하고 작업 간의 종속성을 이해하며 일괄 처리 일정을 최적화할 수 있습니다.

## 구성 문제 식별 {#identifying-issues}

작업 일정을 검토할 때 구성 문제를 나타내는 패턴을 확인할 수 있습니다. 일반적인 문제는 다음과 같습니다.

* 작업이 너무 가깝게 예약되어 리소스 경합 발생
* 동일한 시간 창 내에서 너무 많은 배치가 실행됨
* 과도한 일별 일괄 처리 작업이 포함된 개별 데이터 세트
* 세그먼테이션이 실행되기 바로 전에 예약된 수집 작업

이러한 패턴은 작업 실패, 불완전한 데이터 처리 및 시스템 성능 저하로 이어질 수 있습니다. 이러한 문제를 식별하고 해결하는 방법은 [작업 일정 방지 패턴 식별](job-schedules-anti-patterns.md)에 대한 설명서를 참조하세요.

특정 데이터 세트 또는 작업 실행을 조사해야 하는 경우 세부 보기로 드릴다운하여 실행 내역, 오류 메시지, 성능 지표 및 종속성을 볼 수 있습니다. 이 자세한 데이터를 보는 방법에 대한 자세한 내용은 [작업 세부 정보 보기](job-schedules-details.md)에 대한 설명서를 참조하십시오.
