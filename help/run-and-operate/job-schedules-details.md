# 작업 일정 세부 정보 보기

>[!AVAILABILITY]
>
>[!UICONTROL Job schedules]은(는) 현재 제한된 릴리스로 사용할 수 있으며 다음 Real-Time CDP 작업에만 사용할 수 있습니다.
>
> * 일괄 데이터 레이크 수집
> * 일괄 프로필 수집
> * 일괄 처리 세분화
> * 일괄 처리 대상 활성화.

작업 실패 문제를 해결하거나 성능 문제를 조사할 때 특정 데이터 세트 및 해당 작업 실행에 대한 자세한 정보가 필요합니다. [작업 일정](job-schedules.md) 인터페이스를 사용하면 타임라인 보기에서 개별 데이터 세트 및 작업으로 드릴다운하여 실행 기록, 시간 및 상태를 이해할 수 있습니다.

이 상세 보기를 사용하여 다음을 수행할 수 있습니다.

* 특정 작업이 실패했거나 예상보다 오래 걸린 이유 알아보기
* 시간 경과에 따른 데이터 세트 실행 내역 검토
* 배치 작업의 시간 및 기간 패턴 이해
* 파이프라인 문제를 일으키는 특정 일괄 처리 식별
* Adobe 지원 관련 문제 해결에 필요한 정보 수집

## 전제 조건 {#prerequisites}

작업 세부 사항을 보기 전에 다음을 수행해야 합니다.

* [!UICONTROL Job Schedules] 및 **[!UICONTROL View Job Schedules]** **[!UICONTROL View Profile Management]**&#x200B;액세스 제어 권한[을 가진 ](/help/access-control/home.md#permissions)에 액세스할 수 있습니다.
* [작업 일정 인터페이스](job-schedules.md#understanding-interface) 및 타임라인 보기에 익숙해지도록 합니다.
* 다양한 [작업 유형](job-schedules.md#job-schedules-details)(레이크 수집, 프로필 수집, 세분화, 활성화)을 이해합니다.

## 세부 정보 계층 이해 {#details-hierarchy}

작업 일정은 세 가지 수준의 세부 정보를 제공하므로 광범위한 패턴에서 특정 문제로 이동할 수 있습니다.

| 보기 수준 | 표시 내용 | 사용하는 경우 |
|------------|---------------|----------------|
| **타임라인 보기** | 선택한 기간 동안의 모든 데이터 세트 및 예약된 작업 | 패턴 식별, [반대 패턴](job-schedules-anti-patterns.md) 표시 및 전체 파이프라인 개요 가져오기 |
| **데이터 집합 세부 정보** | 단일 데이터 세트에 대해 집계된 지표 및 실행 내역 | 한 데이터 세트의 전체 성능 추적, 데이터 볼륨 이해 및 작업 빈도 검토 |
| **작업 실행 세부 정보** | 개별 작업 실행에 대한 특정 실행 정보 | 특정 작업이 실패한 이유 조사, 정확한 타이밍 확인 및 처리된 레코드 확인 |

**탐색 흐름**: 타임라인 보기로 시작하여 문제를 식별하고 → 데이터 집합을 선택하여 세부 정보를 확인합니다 → 특정 작업 실행을 선택하여 세부 정보를 조사합니다.

### 타임라인 보기 이해 {#timeline-visualization}

타임라인 보기는 가로 및 세로 레이아웃을 사용하여 작업 일정과 중요한 처리 시간을 이해하는 데 도움이 됩니다.

* **가로축(시간 진행률)**: 데이터 세트 및 해당 작업 실행이 왼쪽에서 오른쪽으로 타임라인에 표시되며 선택한 기간(오늘, 어제 또는 최근 7일) 동안 작업이 실행되는 시간을 표시합니다. 각 색상 막대는 작업 실행을 나타내며 시작 및 종료 시간에 따라 가로로 배치됩니다.

* **세로축(예약된 시작 시간)**: 중요한 예약된 시작 시간이 모든 데이터 세트에 걸쳐 세로선으로 표시되므로 업스트림 작업과 다운스트림 처리 사이의 시간 관계를 쉽게 확인할 수 있습니다.
   * **파란색 세로선**: 세분화를 시작하도록 예약된 시기를 나타냅니다.
   * **검은색 세로줄**: 대상 활성화를 시작하도록 예약한 시기를 나타냅니다.

이 레이아웃을 사용하면 데이터 파이프라인 작업과 다운스트림 처리 간의 타이밍 관계를 빠르게 식별할 수 있습니다. 이상적으로 업스트림 작업(예: 데이터 레이크 및 프로필 수집)은 이러한 수직 마커의 왼쪽에 완료되어야 세그먼테이션 및 활성화가 시작되기 전에 데이터가 준비됩니다. 이러한 마커를 지나서 확장된 작업은 데이터가 완전히 준비되기 전에 다운스트림 프로세스가 시작될 수 있는 잠재적 타이밍 문제를 나타냅니다.

## 데이터 세트 세부 정보 보기 {#view-dataset-details}

특정 데이터 세트에 대한 세부 사항을 보려면 다음을 수행하십시오.

1. **[!UICONTROL Job Schedules]** 타임라인 보기에서 조사할 데이터 세트를 찾습니다.
2. 왼쪽 열에서 데이터 세트 이름을 선택합니다.

데이터 세트 세부 사항 보기가 오른쪽 패널에 열리고 이 데이터 세트와 연결된 모든 작업에 대한 정보가 표시됩니다.

![선택한 데이터 세트에 대한 집계된 레이크 및 프로필 수집 지표를 표시하는 데이터 세트 세부 정보 패널입니다.](assets/job-schedules/view-dataset-details.png)

데이터 세트 세부 정보 패널에는 작업 유형별로 구성된 데이터 세트 이름, ID 및 작업별 지표가 표시됩니다. 패널 맨 위에는 데이터 세트 ID가 클릭 가능한 링크로 표시됩니다. 이 ID를 선택하여 전체 데이터 세트 세부 정보 페이지로 이동합니다.

각 데이터 세트 세부 정보 패널에는 다음 지표가 포함됩니다.

### 레이크 수집 지표 {#lake-ingestion-metrics}

데이터 레이크 수집 작업이 포함된 데이터 세트의 경우 패널에 다음 지표가 표시됩니다.

| 지표 | 설명 | 다음에 사용 |
|--------|-------------|---------|
| **[!UICONTROL Total runs]** | 이 데이터 세트에 대해 완료된 총 데이터 레이크 수집 작업 수 | 활동 추적 |
| **[!UICONTROL Runs in progress]** | 현재 실행 중인 레이크 수집 작업 수 | 병목 현상 감지 |
| **[!UICONTROL Total records added]** | 모든 작업 실행에서 데이터 레이크에 추가된 누적 새 레코드 수 | 볼륨 모니터링 |
| **[!UICONTROL Total ingestion time]** | 모든 데이터 레이크 수집 작업의 결합된 기간 | 처리 시간 평가 |
| **[!UICONTROL Total records updated]** | 수집 중 업데이트된 기존 레코드의 누적 수 | 새로 고침 패턴 분석 |
| **[!UICONTROL Avg. ingestion speed (records/second)]** | 데이터 레이크 수집 작업의 평균 처리율 | 성능 비교 |

### 프로필 수집 지표 {#profile-ingestion-metrics}

프로필 수집 작업이 있는 데이터 세트의 경우 패널에 다음 지표가 표시됩니다.

| 지표 | 설명 | 다음에 사용 |
|--------|-------------|---------|
| **[!UICONTROL Total runs]** | 이 데이터 세트에 대해 완료된 총 프로필 수집 작업 수 | 활동 추적 |
| **[!UICONTROL Runs in progress]** | 현재 실행 중인 프로필 수집 작업 수 | 지연 감지 |
| **[!UICONTROL Total profiles created]** | 모든 작업 실행에 대해 이 데이터 세트에서 생성된 누적 새 프로필 수 | 프로필 증가 모니터링 |
| **[!UICONTROL Total profile ingestion time]** | 모든 프로필 수집 작업의 결합된 기간 | 시간 문제 식별 |
| **[!UICONTROL Total profiles updated]** | 이 데이터 세트의 데이터로 업데이트된 기존 프로필의 누적 수 | 빈도 추적 업데이트 |
| **[!UICONTROL Avg. profile ingestion speed (profiles/second)]** | 프로필 수집 작업의 평균 처리량 | 성능 모니터링 |

>[!NOTE]
>
> 이 지표는 이 데이터 세트에 대한 모든 작업 실행의 누적 합계를 보여 줍니다. 특정 실행에 대한 세부 정보를 보려면 타임라인에서 직접 작업을 선택합니다.

## 타임라인에서 데이터 세트 필터링 {#filter-datasets}

예약된 작업이 있는 데이터 세트가 많을 경우 한 번에 모두 보는 대신 특정 데이터 세트에 중점을 둘 수 있습니다. 데이터 세트 필터를 사용하면 타임라인 보기에 표시할 데이터 세트를 선택할 수 있습니다.

![타임라인 보기에 표시할 데이터 집합을 선택할 수 있는 데이터 집합 필터 패널입니다.](assets/job-schedules/view-datasets.gif)

타임라인에 표시된 데이터 세트를 필터링하려면 다음을 수행하십시오.

1. 타임라인 보기의 왼쪽 상단에서 데이터 세트 카운터를 찾습니다(예: &quot;2 데이터 세트&quot;).
2. 데이터 세트 카운터 옆에 있는 필터 아이콘을 선택합니다.
3. 예약된 작업과 함께 사용 가능한 모든 프로필 사용 데이터 세트를 표시하는 데이터 세트 선택 패널이 열립니다.
4. 데이터 세트를 선택 또는 선택 해제하여 타임라인 보기에서 표시하거나 숨깁니다.
5. 타임라인이 즉시 업데이트되어 선택한 데이터 세트만 표시됩니다.

필터링을 사용하여 다음을 수행합니다.

* **특정 데이터 원본에 집중**: 특정 데이터 파이프라인의 문제를 해결할 때 관련 데이터 집합만 표시하도록 필터링하십시오.
* **시각적 복잡성 감소**: 데이터 세트가 많은 경우 필터링하면 데이터 하위 집합에 대한 패턴을 보다 명확하게 볼 수 있습니다.
* **관련 데이터 세트 비교**: 일정 관계를 이해하는 데 관련된 데이터 세트만 선택하십시오.
* **패턴 방지 조사**: 잠재적 [구성 문제](job-schedules-anti-patterns.md)를 식별하면 영향을 받는 데이터 세트로 필터링하여 더 자세히 검사하십시오.

필터는 세션 중에 지속되므로 데이터 세트 선택을 유지하면서 기간(오늘, 어제, 지난 7일) 간을 이동할 수 있습니다.

## 개별 작업 실행 세부 정보 보기 {#view-job-details}

특정 작업 실행을 조사해야 하는 경우 타임라인에서 해당 특정 실행에 대한 자세한 실행 정보를 보려면 타임라인에서 선택합니다.

### 작업 실행 세부 정보 액세스 {#access-job-details}

특정 작업 실행에 대한 세부 정보를 보려면 다음과 같이 하십시오.

1. [!UICONTROL Job Schedules] 타임라인 보기에서 조사할 특정 작업 실행을 찾습니다.
2. 타임라인에서 작업 표시기를 선택합니다(작업을 나타내는 색상 막대).

특정 작업 실행에 대한 정보를 보여주는 **[!UICONTROL Dataflow run details]** 패널이 열립니다.

![특정 작업 실행에 대한 실행 정보를 표시하는 데이터 흐름 실행 세부 정보 패널입니다.](assets/job-schedules/job-details.png)

### 데이터 흐름 실행 세부 정보 {#dataflow-run-details}

데이터 흐름 실행 세부 정보 패널에는 작업 유형별로 구성된 특정 작업 실행에 대한 정보가 표시됩니다. 수집 작업의 경우 레이크 수집 및 프로필 수집 단계에 대한 세부 정보가 표시됩니다.

#### 레이크 수집 작업 세부 정보 {#lake-ingestion-job-details}

| 필드 | 설명 |
|-------|-------------|
| **[!UICONTROL Dataflow run ID]** | 해당 레이크 수집 작업 실행에 대한 고유 식별자. 전체 데이터 흐름 모니터링 세부 정보를 보려면 ID를 선택하십시오. |
| **[!UICONTROL Run status]** | 작업 결과(성공, 실패, 진행 중, 큐에 있음). 녹색 표시기는 성공적으로 완료된 것을 보여 줍니다. |
| **[!UICONTROL Started at]** | 레이크 수집 작업 실행이 시작된 날짜 및 시간입니다. |
| **[!UICONTROL Completed at]** | 레이크 수집 작업 실행이 완료된 날짜 및 시간입니다. |

### 작업 실행 플로우 이해 {#job-execution-flow}

특정 작업 실행을 보는 경우 레이크 수집과 프로필 수집 간의 관계를 확인할 수 있습니다.

* **레이크 수집이 먼저 실행됩니다**: 데이터가 데이터 레이크에 로드되고 유효성이 검사됩니다.
* **프로필 수집 팔로우**: 레이크 수집이 완료되면 적격 레코드가 프로필 스토어로 처리됩니다.
* **타이밍 문제**: 레이크 수집이 완료되는 시점과 프로필 수집이 시작되는 시점 사이의 시간 차이를 확인합니다. 이러한 차이는 세그먼테이션과 같은 다운스트림 프로세스에 영향을 줄 수 있습니다.

**작업 실행 세부 정보를 사용하여**:

* 특정 작업이 성공적으로 완료되었는지 확인
* 작업 실행의 실제 기간 계산(완료 시간에서 시작 시간을 뺀 시간)
* 특정 실행에서 처리된 레코드 수 이해
* 다양한 작업 실행 간 성능 비교
* 문제 해결을 위한 상세 데이터 흐름 모니터링에 액세스
* 레이크 및 프로필 수집 단계 간 시간 문제 식별

## 작업 세부 정보를 사용한 문제 해결 {#troubleshooting}

작업 세부 정보를 사용하여 문제를 조사하고 다음 단계를 결정합니다.

**실패한 작업**: 모니터링 대시보드에서 오류 세부 정보를 보려면 데이터 흐름 실행 ID를 선택하십시오. [데이터 세트 세부 정보](#view-dataset-details)에서 반복 패턴을 확인하고, [타임라인](job-schedules.md)에서 리소스 경합을 검토하고, 구성에서 [앤티 패턴](job-schedules-anti-patterns.md)을(를) 식별하십시오.

**느린 작업**: [데이터 세트 지표](#view-dataset-details)의 기록 평균과 기간을 비교합니다. 일반적인 원인은 [일정 겹침](job-schedules-anti-patterns.md#schedule-overlap-pattern), [고밀도 배치 스택](job-schedules-anti-patterns.md#scheduled-density) 또는 데이터 볼륨 증가입니다.

**일치하지 않는 레코드**: 레이크 수집 레코드를 작업 실행 세부 정보의 프로필 수집 레코드와 비교합니다. 프로필 수집은 일반적으로 ID 요구 사항 및 데이터 품질 규칙으로 인해 더 적은 레코드를 표시합니다.

자세한 데이터 흐름 상태 정보는 [데이터 레이크 수집 모니터링](../dataflows/ui/monitor-sources.md), [프로필에 대한 데이터 흐름 모니터링](../dataflows/ui/monitor-profiles.md), [대상에 대한 데이터 흐름 모니터링](../dataflows/ui/monitor-audiences.md) 및 [대상에 대한 데이터 흐름 모니터링](../dataflows/ui/monitor-destinations.md)을 참조하십시오.

## 다음 단계 {#next-steps}

작업 세부 사항을 보는 방법을 학습한 후:

* 타임라인 보기와 인터페이스를 이해하려면 [작업 일정 개요](job-schedules.md)를 검토하십시오.
* 일반적인 구성 문제를 방지하기 위해 [패턴 방지](job-schedules-anti-patterns.md)에 대해 알아봅니다.
* 데이터 로드 일정을 최적화하려면 [일괄 처리 수집](../ingestion/batch-ingestion/overview.md)을(를) 이해하십시오.
* 전체 파이프라인 가시성을 위해 [대상 데이터 흐름 모니터링](../dataflows/ui/monitor-destinations.md)을 살펴보십시오.
