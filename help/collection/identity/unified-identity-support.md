# 데이터 수집에서 통합 ID 지원

>[!AVAILABILITY]
>
>이 기능은 현재 베타 버전입니다. 가용성, 비헤이비어 및 설명서는 변경될 수 있습니다.

통합 ID 지원을 통해 Edge Network은 자사 및 타사 ID 컨텍스트 모두에서 작동할 수 있습니다. 소유한 속성에 대한 지속적인 자사 식별과 타사 쿠키를 지원하는 브라우저의 타사 활성화 워크플로를 통합합니다. 웹 SDK에서 ECID, FPID 및 기타 ID 신호를 처리하는 방법에 대한 배경은 [데이터 수집의 ID](./overview.md)를 참조하십시오.

통합 ID 지원을 통해 다음과 같은 작업을 수행할 수 있습니다.

* **대상자 도달 범위 최대화**: 더 많은 트래픽 공유를 위해 서드파티 대상(DSP, SSP, 광고 네트워크)에서 Experience Platform 대상자를 활성화합니다.
* **측정 정확도 유지**: 소유한 속성과 광고 플랫폼에서 일관된 방문자 식별을 유지합니다.
* **향후 구현 증명**: 타사 활성화 워크플로와의 호환성을 유지하면서 자사 장치 ID를 기반으로 사용하십시오.

방문자가 사이트에 도달하면 Edge Network에서는 사용 가능한 ID 신호를 평가하여 조건이 허용될 때 자동으로 자사 컨텍스트와 타사 컨텍스트를 연결합니다. 서드파티 쿠키를 차단하는 브라우저는 구현에 영향을 주지 않고 자사 모드에서 계속 작동합니다.

## 작동 방식

Edge Network은 사용 가능한 ID 신호를 다음 우선 순위 순서로 평가하여 ECID를 생성합니다.

| 우선 순위 | 소스 | 컨텍스트 | 비헤이비어 |
| --- | --- | --- | --- |
| 1 | **Demdex ID** | 서드파티 | Demdex ID가 있으면 ECID가 시드됩니다. 이 시드는 동일한 타사 쿠키를 공유하는 도메인 간에 일관된 ECID를 생성합니다. |
| 2 | **FPID** | 자사 | Demdex ID가 없지만 FPID가 있는 경우 ECID는 FPID에서 시드되고 Demdex ID는 FPID에서 파생됩니다. |
| 3 | **Random** | 자사 | Demdex ID와 FPID를 모두 사용할 수 없는 경우 새로운 무작위 ECID가 생성되고 Demdex ID가 이 ID에서 파생됩니다. |

ECID 및 Demdex ID는 결정론적 알고리즘을 통해 암호학적으로 연결되며, 이는 하나가 다른 것으로부터 파생될 수 있음을 의미한다. 이 관계를 통해 Edge Network은 구현에 별도의 방문자 처리 논리 없이 자사 ID 컨텍스트와 타사 ID 컨텍스트 간을 번역할 수 있습니다.

관계는 결정적이므로 해당 Demdex ID를 사용할 수 있을 때 자사 ECID에 구축된 대상은 서드파티 인프라를 통해 활성화할 수 있습니다.

이미 FPID에서 파생된 ECID가 있는 방문자의 경우 Edge Network에서 자사 ID를 서드파티 ID 컨텍스트에 자동으로 연결할 수 있습니다. 이 문제는 브라우저가 서드파티 쿠키를 지원하고 구현을 변경할 필요가 없는 경우에 투명하게 발생합니다. 자동 연결이 발생할 때:

1. Edge Network은 방문자의 ECID가 Demdex ID에서 파생되지 않았음을 감지합니다.
1. 방문자의 브라우저가 타사 쿠키를 지원하는 경우 경량 ID 동기화가 트리거됩니다.
1. 시스템에서는 방문자의 자사 ECID와 타사 ID 간에 링크를 만듭니다.
1. 링크는 ID 저장소에 저장되므로 서드파티 대상에서 대상을 활성화할 수 있습니다.

자동 연결은 기존 ECID를 보존하고 방문자의 클리핑을 방지합니다. 시간이 지남에 따라 방문자가 돌아오고 연결이 발생함에 따라 점점 더 많은 대상자가 서드파티 활성화를 사용할 수 있게 됩니다.

서드파티 대상 활성화는 ID 동기화(ID 동기화)에 의존합니다. Edge Network이 타사 ID를 설정하거나 새로 고치면 응답에서 ID 동기화 지침을 반환합니다. 이러한 지침은 Experience Platform 대상을 해당 플랫폼에서 일치시키고 전달할 수 있도록 브라우저가 방문자의 ID를 파트너 도메인(DSP, 광고 네트워크 및 기타 활성화 플랫폼)과 동기화하도록 합니다.

## 전제 조건

통합 ID를 지원하려면 다음 조건을 모두 충족해야 합니다.

* 사이트는 사용자가 제어하는 도메인에서 자사 데이터 수집을 사용합니다.
* 구현에서는 FPID 또는 다른 자사 지속성 전략을 기반으로 합니다.
* 타사 쿠키는 웹 SDK 구성에서 활성화됩니다.
* 데이터 스트림에 대해 서드파티 ID 동기화가 활성화되었습니다.
* 방문자가 타사 쿠키를 허용하는 브라우저를 사용합니다(아래 [브라우저 호환성](#browser-compatibility) 참조).

## 구성

1. **웹 SDK에서 타사 쿠키 사용**: 웹 SDK 구현에서 **타사 쿠키 사용** 설정을 사용합니다. 태그 확장을 사용하는 경우 **[!UICONTROL Use third-party cookies]** ID 구성 설정[에서 &#x200B;](/help/tags/extensions/client/web-sdk/configure/identity.md#use-third-party-cookies)을(를) 사용하도록 설정하십시오. JavaScript 라이브러리를 사용하는 경우 [`thirdPartyCookiesEnabled`](/help/collection/js/commands/configure/thirdpartycookiesenabled.md)을(를) `true`(으)로 설정합니다.

1. **데이터 스트림에서 타사 ID 동기화 사용**: 데이터 스트림의 고급 설정에서 **[!UICONTROL Third-Party ID Sync]** 옵션을 사용합니다. [데이터스트림 만들기 및 구성](/help/datastreams/configure.md#advanced-options)을 참조하세요.

1. **퍼스트 파티 지속성 설정이 적용되었는지 확인**: 퍼스트 파티 지속성 전략(예: FPID)이 이미 소유한 도메인에 배포되어 있는지 확인합니다. [데이터 수집의 자사 장치 ID](fpid.md)를 참조하십시오.

## 유효성 검사

통합 ID 지원이 작동하는지 확인하려면 다음을 수행하십시오.

1. 브라우저의 개발자 도구를 열고 **네트워크** 탭으로 이동합니다.
1. 기존 요청을 지우고 새 세션이나 시크릿 세션에서 웹 SDK 이벤트(페이지 로드 또는 사용자 지정 이벤트)를 트리거합니다.
1. Edge Network 응답을 찾습니다(`adobedc.demdex.net` 및 자사 컬렉션 끝점에 대한 호출 검색).
1. 응답 페이로드에서 ID 동기화 지침을 검사합니다.

ID 동기화 명령이 있으면 다음과 유사한 `identity:exchange` 핸들이 응답에 포함됩니다.

```json
{
  "handle": [
    {
      "type": "identity:exchange",
      "payload": [
        {
          "type": "url",
          "id": 411,
          "spec": {
            "url": "https://example.com/...",
            "hideReferrer": false,
            "ttlMinutes": 10080
          }
        },
        {
          "type": "url",
          "id": 89,
          "spec": {
            "url": "https://example.org/...",
            "hideReferrer": true,
            "ttlMinutes": 10080
          }
        }
      ]
    }
  ]
}
```

| 요소 | 설명 |
| --- | --- |
| `type: "identity:exchange"` | ID 동기화 명령이 있음을 나타냅니다. |
| `payload` 배열 | 파트너 ID 동기화 URL 목록. |
| `url`개 값 | ID 동기화를 위해 URL을 파트너 도메인으로 리디렉션합니다. |
| `id`개 값 | 파트너 식별자. |

>[!TIP]
>
>응답에 `identity:exchange` 핸들이 표시되지 않는 경우:
>
>* 새 브라우저 세션 또는 시크릿 브라우저 세션을 사용하여 테스트하고 있는지 확인합니다. 기존 ID가 새 동기화를 트리거하지 않습니다.
>* 데이터 스트림과 웹 SDK 설정이 모두 올바르게 구성되어 있는지 확인합니다.
>* 서드파티 쿠키를 지원하는 브라우저를 사용하고 있는지 확인합니다(아래 표 참조).

ID 동기화 활동을 확인한 후 다음을 확인합니다.

* 자사 ID는 소유 도메인의 페이지 로드 시에도 예상대로 유지됩니다.
* 활성화 및 보고 흐름은 지원하는 환경에서 예상대로 동작합니다.

## 브라우저 호환성 {#browser-compatibility}

타사 ID 기능은 타사 쿠키에 대한 브라우저 지원에 따라 다릅니다. 다음 표에는 예상되는 동작이 요약되어 있습니다.

| 브라우저 | 서드파티 쿠키 지원 | Demdex 사용 가능 | ID 동작 |
| --- | --- | --- | --- |
| Google Chrome | 지원됨 | 예 | Demdex → ECID(도메인 간에 일관됨) |
| Microsoft Edge | 기본적으로 지원됨 | 예 | Demdex → ECID(도메인 간에 일관됨) |
| Mozilla Firefox | 기본적으로 차단됨(ETP) | 아니요(기본값) | FPID → ECID(도메인당) |
| Apple Safari | 차단됨(ITP) | 아니요 | FPID → ECID(도메인당) |

서드파티 쿠키를 차단하는 브라우저의 경우 자사 식별이 계속 정상적으로 작동합니다. 타사 활성화 기능은 브라우저에서 타사 쿠키를 허용하는 경우에만 사용할 수 있습니다.

## 제한 사항

* 서드파티 ID 동작은 전적으로 서드파티 쿠키를 허용하는 방문자의 브라우저에 따라 다릅니다. 이를 차단하는 브라우저에는 타사 활성화에 대한 대체 항목이 없습니다.
* 자동 연결을 사용하려면 방문자가 사이트로 돌아가야 합니다. 서드파티 활성화에 적합한 대상자의 비율은 시간이 지남에 따라 점차 증가합니다.
