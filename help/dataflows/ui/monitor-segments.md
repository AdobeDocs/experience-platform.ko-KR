# UI에서 세그먼트에 대한 데이터 흐름 모니터링

세그먼테이션 서비스를 사용하면 Adobe Experience Platform의 실시간 고객 프로필 데이터에서 세그먼트와 대상을 만들 수 있습니다. Platform은 데이터 흐름을 제공하여 소스에서 대상으로 이러한 데이터 흐름을 투명하게 추적합니다.

모니터링 대시보드는 데이터 세그먼테이션의 상태를 포함하여 세그먼트 내의 데이터 활동을 시각적으로 보여줍니다. 이 자습서에서는 Experience Platform 사용자 인터페이스를 사용하여 모니터링 대시보드를 사용하여 데이터의 세그멘테이션을 모니터링하고 세그먼트 활성화, 평가 및 내보내기 작업의 상태를 추적할 수 있는 방법에 대한 지침을 제공합니다.

## 시작하기 {#getting-started}

이 안내서에서는 Adobe Experience Platform의 다음 구성 요소를 이해하고 있어야 합니다.

- [데이터 흐름](../home.md): 데이터 흐름은 플랫폼 간에 데이터를 이동하는 데이터 작업을 나타냅니다. 데이터 흐름은 서로 다른 서비스 간에 구성되므로 소스 커넥터에서 대상 데이터 세트로 데이터를 [!DNL Identity] 및 [!DNL Profile], 및에 [!DNL Destinations].
   - [데이터 흐름 실행](../../sources/notifications.md): 데이터 흐름 실행은 선택한 데이터 흐름의 빈도 구성에 따라 반복되는 예약된 작업입니다.
- [세그먼테이션](../../segmentation/home.md): 세그먼테이션을 사용하면 실시간 고객 프로필 데이터에서 세그먼트와 대상을 만들 수 있습니다.
   - [활성화 작업](../../destinations/ui/activation-overview.md): 활성화 작업은 세그먼트를 지정된 대상에 활성화하는 데 사용됩니다.
   - [평가 작업](../../segmentation/tutorials/evaluate-a-segment.md#evaluate-a-segment): 평가 작업은 지정된 세그먼트를 기반으로 대상 세그먼트를 만드는 비동기 프로세스입니다.
   - [작업 내보내기](../../segmentation/api/export-jobs.md): 내보내기 작업은 대상 세그먼트 구성원을 데이터 세트에 유지하는 데 사용되는 비동기 프로세스입니다.
- [샌드박스](../../sandboxes/home.md): [!DNL Experience Platform] 단일 파티션을 생성하는 가상 샌드박스 제공 [!DNL Platform] 디지털 경험 애플리케이션을 개발하고 발전시키는 데 도움이 되는 별도의 가상 환경으로 인스턴스를 구축할 수 있습니다.

| **[!UICONTROL 마지막 활성화 ID]** | 세그먼트의 마지막 활성화 작업에서 활성화된 ID 수입니다. |

### 활성화 작업 세부 사항 {#activation-job-details}

활성화 작업 데이터 흐름 실행 세부 정보 페이지에는 실행 지표, 데이터 흐름 실행 오류 및 세그먼트 작업과 관련된 세그먼트에 대한 정보가 표시됩니다. 활성화 작업은 지정된 대상에 대해 세그먼트를 활성화하는 데 사용됩니다. 기본적으로 세부 정보 페이지에는 데이터 흐름 실행 오류가 표시됩니다.

![필터링된 세그먼트 대시보드. 이 세그먼트에 대해 실행된 다양한 세그먼트 작업에 대한 정보가 표시됩니다.](../assets/ui/monitor-segments/activation-job-details.png)

이 대시보드 보기에 다음 지표를 사용할 수 있습니다.

| 지표 | 설명 |
| ------ | ----------- |
| **[!UICONTROL 받은 프로필]** | 활성화 플로우에서 받은 총 프로필 수입니다. |
| **[!UICONTROL ID가 활성화됨]** | 수신한 프로필을 기반으로 대상에 성공적으로 활성화된 총 ID 수입니다. |
| **[!UICONTROL 제외된 ID]** | 수신한 프로필을 기반으로 하여 대상에서 제외된 총 ID 수입니다. 특성 또는 동의 위반이 누락되어 이러한 ID를 제외할 수 있습니다. |
| **[!UICONTROL 데이터 크기]** | 활성화 중인 데이터 흐름의 크기입니다. |
| **[!UICONTROL 총 파일 수]** | 데이터 흐름에서 활성화된 총 파일 수입니다. |
| **[!UICONTROL 상태]** | 활성화 작업의 현재 상태입니다. |
| **[!UICONTROL 데이터 흐름 실행 시작]** | 활성화 작업이 시작된 날짜 및 시간입니다. |
| **[!UICONTROL 데이터 흐름 실행 종료]** | 활성화 작업이 종료된 날짜 및 시간입니다. |
| **[!UICONTROL 데이터 흐름 실행 ID]** | 현재 활성화 작업의 ID입니다. |
| **[!UICONTROL IMS 조직 ID]** | 활성화 작업이 속한 IMS 조직의 ID입니다. |
| **[!UICONTROL 대상 이름]** | 데이터가 활성화되는 대상의 이름입니다. |

지표 아래에 데이터 흐름 실행 오류 간에 선택하는 토글이 표시되고 세그먼트가 표시됩니다.

![필터링된 세그먼트 대시보드. 데이터 흐름 실행 오류 간을 전환하는 데 사용되는 토글이 강조 표시됩니다.](../assets/ui/monitor-segments/activation-job-details-toggle.png)

데이터 흐름 실행 오류 섹션에서 전환을 선택하여 ID 실패 또는 ID 제외 필드를 확인합니다. 오류 섹션에는 오류 코드 및 실패 또는 제외된 ID 수에 대한 세부 정보가 포함되어 있습니다.

![필터링된 세그먼트 대시보드. 실패하거나 제외된 ID에 대한 정보가 강조 표시됩니다.](../assets/ui/monitor-segments/activation-job-details.png)

![필터링된 세그먼트 대시보드. 실패하거나 제외된 ID에 대한 정보가 강조 표시됩니다.](../assets/ui/monitor-segments/activation-job-details-segments.png)

세그먼트 섹션에서 다음 지표를 사용할 수 있습니다.

| 지표 | 설명 |
| ------ | ----------- |
| **[!UICONTROL 이름]** | 활성화한 세그먼트의 이름입니다. |
| **[!UICONTROL ID가 활성화됨]** | 수신한 프로필을 기반으로 대상에 성공적으로 활성화된 총 ID 수입니다. |
| **[!UICONTROL 제외된 ID]** | 수신한 프로필을 기반으로 하여 대상에서 제외된 총 ID 수입니다. 특성 누락 또는 동의 위반으로 인해 이러한 ID를 제외할 수 있습니다. |
| **[!UICONTROL 마지막 데이터 흐름 실행 상태]** | 해당 세그먼트에 대해 실행된 마지막 활성화 작업의 상태입니다. |
| **[!UICONTROL 마지막 데이터 흐름 실행 날짜]** | 해당 세그먼트에 대해 실행된 마지막 활성화 작업의 날짜 및 시간입니다. |

### 평가 작업 세부 사항 {#evaluation-job-details}

평가 작업 데이터 흐름 실행 세부 정보 페이지에는 세그먼트 작업과 관련된 실행 지표 및 세그먼트에 대한 정보가 표시됩니다. 평가 작업은 지정된 세그먼트를 기반으로 대상 세그먼트를 만드는 비동기 프로세스입니다. 평가 작업에 대한 자세한 내용은 다음 문서를 참조하십시오 [세그먼트 평가](../../segmentation/tutorials/evaluate-a-segment.md#evaluate-a-segment).

![평가 작업 대시보드입니다. 세그먼트 평가 작업에 대한 정보가 표시됩니다.](../assets/ui/monitor-segments/evaluation-job-details.png)

이 대시보드 보기에 다음 지표를 사용할 수 있습니다.

| 지표 | 설명 |
| ------ | ----------- |
| **[!UICONTROL 총 프로필]** | 평가되는 총 프로필 수입니다. |
| **[!UICONTROL 상태]** | 평가 작업의 상태입니다. 평가 작업의 가능한 상태는 다음과 같습니다 **[!UICONTROL 성공]** 및 **[!UICONTROL 실패]**. |
| **[!UICONTROL 작업 시작]** | 평가 작업이 시작된 날짜 및 시간입니다. |
| **[!UICONTROL 작업 종료]** | 평가 작업이 종료된 날짜 및 시간입니다. |
| **[!UICONTROL 작업 유형]** | 세그먼트 작업의 유형입니다. 이 경우 항상 세그먼트 평가 작업입니다. |
| **[!UICONTROL 평가 유형]** | 수행 중인 평가의 유형입니다. 다음 중 하나일 수 있습니다 **[!UICONTROL 일괄 처리]** 또는 **[!UICONTROL 스트리밍]**. |
| **[!UICONTROL 작업 ID]** | 평가 작업의 ID입니다. |
| **[!UICONTROL IMS 조직 ID]** | 평가 작업이 속한 IMS 조직의 ID입니다. |
| **[!UICONTROL 세그먼트 이름]** | 평가되는 세그먼트의 이름입니다. |
| **[!UICONTROL 세그먼트 ID]** | 평가되는 세그먼트의 ID입니다. |

세그먼트 섹션에서 평가 작업의 일부로 평가되는 세그먼트 목록을 볼 수 있습니다. 검색 창을 사용하여 이름별로 세그먼트 목록을 필터링할 수 있습니다.

>[!IMPORTANT]
>
>이 대시보드 보기는 현재 최대 800개의 세그먼트 지표를 지원합니다.

세그먼트 섹션에서 다음 지표를 사용할 수 있습니다.

| 지표 | 설명 |
| ------ | ----------- |
| **[!UICONTROL 이름]** | 평가되는 세그먼트의 이름입니다. |
| **[!UICONTROL 프로필 수]** | 평가되는 프로필 수입니다. |

## 세그먼트 작업 대시보드 모니터링 {#monitoring-segment-jobs-dashboard}

>[!CONTEXTUALHELP]
>id="platform_monitoring_segment_jobs"
>title="세그먼트 작업"
>abstract="세그먼트 작업 보기에는 모든 세그먼트에 대한 평가 및 내보내기 작업에 대한 정보가 포함되어 있습니다."

에 액세스하려면 **[!UICONTROL 세그먼트 작업]** 대시보드, 선택 **[!UICONTROL 모니터링]** (![모니터링 아이콘](../assets/ui/monitor-destinations/monitoring-icon.png))을 클릭하여 제품에서 사용할 수 있습니다. 에 한 번 [!UICONTROL 모니터링] 페이지를 선택하고 **[!UICONTROL 세그먼트 작업]**. 다음 [!UICONTROL 모니터링] 대시보드에는 세그먼트 평가 및 내보내기 작업에 대한 지표와 정보가 포함되어 있습니다.

>[!NOTE]
>
>전용 **세그먼트 평가 작업** 세그먼트당 모니터링에 대해 지원됩니다. 세그먼트 내보내기 작업은 조직 수준 모니터링만 지원합니다.

![세그먼트 작업 모니터링 대시보드](../assets/ui/monitor-segments/segment-jobs-dashboard.png)

를 사용하십시오 [!UICONTROL 세그먼트 작업] 대시보드 를 사용하여 프로필 평가 및 내보내기가 시간 및 예외 없이 발생하는지 파악할 수 있으므로 대상 활성화를 위한 다운스트림 서비스에서는 최신 평가 프로필 데이터를 가질 수 있습니다.

세그먼트 작업에 다음 지표를 사용할 수 있습니다.

| 지표 | 설명 |
---------|----------|
| **[!UICONTROL 세그먼트 작업]** | 세그먼트 작업의 이름을 나타냅니다. |
| **[!UICONTROL 유형]** | 세그먼트 작업 유형(내보내기 또는 평가)을 나타냅니다. 두 경우 모두 세그먼트 작업이 평가되거나 내보내집니다 **모두** 조직에 속한 세그먼트 내보내기 작업에 대한 자세한 내용은 [작업 끝점 내보내기](../../segmentation/api/export-jobs.md). 평가 작업에 대한 자세한 내용은 다음 문서를 참조하십시오 [세그먼트 평가](../../segmentation/tutorials/evaluate-a-segment.md#evaluate-a-segment). |
| **[!UICONTROL 작업 시작]** | 세그먼트 작업이 시작된 날짜 및 시간입니다. |
| **[!UICONTROL 작업 종료]** | 세그먼트 작업이 완료된 날짜 및 시간입니다. |
| **[!UICONTROL 상태]** | 완료된 작업의 상태입니다. 세그먼트 작업에 성공 또는 실패 상태가 있을 수 있습니다. |
