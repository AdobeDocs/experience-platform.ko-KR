# Experience Platform을 위한 맞춤형 접근성 솔루션

Adobe Experience Platform은 모든 유형의 사용자의 요구 사항을 충족하도록 지속적으로 향상되며 시각적, 청각적, 이동성 또는 기타 장애가 있는 개인을 포함하는 전 세계 표준을 준수합니다. 이 문서에서는 Experience Platform 사용자 인터페이스 내의 사용자 지정 접근성 솔루션에 대해 간략하게 설명합니다.

## 홈페이지 및 UI 개요

Experience Platform 사용자 인터페이스는 일반 텍스트, 그래픽 및 UI 구성 요소에 필요한 명암비를 충족합니다. 사용자 인터페이스의 색상은 시각 장애가 있는 사용자를 포함하여 모든 사용자의 접근성을 지원하도록 선택되었습니다.

Platform에서 포인터로 클릭 또는 실행할 수 있는 UI 요소는 키보드를 사용하여 결합할 수도 있습니다. 여기에는 왼쪽 내비게이션, 비디오 플레이어, 테이블 등이 포함됩니다.

Experience Platform은 웹 컨텐츠 액세스 가능성 지침 2.1 레벨 A 및 레벨 AA, 웹 액세스 가능성 이니셔티브 - WAI-ARIA(Accessible Rich Internet Applications) 웹 표준 등 국제 액세스 가능성 표준을 충족하기 위해 노력하고 있습니다.

![Adobe Experience Platform UI의 홈 페이지입니다.](images/homepage.png)

## 왼쪽 탐색

Experience Platform UI 내의 왼쪽 탐색은 키보드에 액세스할 수 있으며 액세스 가능성 표준을 충족하는 일반, 마우스로 가리키기 및 선택 상태의 색상 대비를 제공합니다.

홈 화면에서 사용자는 탭을 왼쪽 탐색에 넣을 수 있습니다. 선택 **Shift + Tab** 사용자를 홈 화면으로 돌아갑니다.

![Experience Platform이 탐색을 나갔습니다.](images/left-navigation-select.png)

왼쪽 내비게이션에 초점이 맞춰져 있으니 **탭** 는 사용자를 확장 및 축소 상호 작용으로 안내합니다. 왼쪽 탐색 영역을 확장하거나 축소하는 기능은 **입력(반환)**.

![Experience Platform 왼쪽 탐색이 축소되었습니다.](images/left-navigation-collapse.png)

왼쪽 탐색을 포커스에 두고 위쪽 및 아래쪽 화살표 키를 사용하여 탐색의 각 항목으로 이동하고 계속 순환합니다(즉, 사용자가 왼쪽 탐색에서 멀리 탭을 누를 때까지 포커스가 이동하지 않음). 선택하면 탐색 항목에 대한 포커스가 표시됩니다. 현재 선택 영역은 강조 표시와 굵은 텍스트로 표시됩니다. 왼쪽 탐색 항목을 선택할 때 **입력(반환)** 선택한 UI 항목을 오른쪽 패널에서 엽니다. 그러나 포커스는 사용자 탭이 사라질 때까지 왼쪽 탐색에 유지됩니다.

![소스가 선택된 상태로 Experience Platform 왼쪽 탐색.](images/left-navigation-sources.png)

Platform 내의 일부 기능은 모든 사용자에 대해 활성화되지 않습니다. 이러한 항목은 탐색에 표시되지만 선택할 수 없습니다. 키보드로 탐색하는 경우 이러한 항목은 화살표 탐색 중에 생략되며 다음을 사용하여 선택할 수 없습니다. **입력(반환)**.

![Experience Platform 왼쪽 탐색 중 사용자에 대해 활성화되지 않은 섹션은 선택할 수 없습니다.](images/left-navigation-sections-disabled.png)

## 포함된 비디오 대화 상자

키보드 탐색을 사용하여 사용 가능한 비디오 링크를 강조 표시하고 선택하여 Experience Platform 내에서 비디오를 볼 수 있습니다. 이렇게 하면 Platform UI 내에 포함된 비디오 대화 상자가 열립니다.

![선택한 요소 주위에 나타나는 파란색 테두리로 초점이 적용되었음을 나타냅니다.](images/profile-overview-tab.png)

## 비디오 대화 상자 키보드 접근성

키보드를 사용하여 포함된 비디오 대화 상자를 탐색할 수도 있습니다. 다음 표에는 포함된 비디오 대화 상자에서 사용할 수 있는 전체 키보드 탐색에 대한 개요가 나와 있습니다.

| 대화 상자 요소 | 키보드 접근성 | 설명 |
|---|---|---|
| 재생 및 일시 중지 | 탭<br/>스페이스바 | 사용 **탭** 재생 단추에 포커스를 설정합니다. **스페이스바** 비디오 재생을 시작하고 비디오 재생을 일시 중지합니다. |
| 스크러버 | 탭<br/>왼쪽 화살표<br/>오른쪽 화살표 | 비디오가 재생될 때 다음을 사용하십시오. **탭** 스크러버에 초점을 맞추다. 스크러버에 초점을 맞추고, **왼쪽 및 오른쪽 화살표 키** 비디오 재생을 앞으로 건너뛰고 5초 뒤로 각각 되돌아갑니다. |
| 음소거 | 탭<br/>스페이스바 | 사용 **탭** 음소거 볼륨 요소에 포커스를 둡니다. 사용 **스페이스바** 비디오 재생을 음소거 또는 음소거 해제하려면 다음을 수행하십시오. |
| 볼륨 | 탭<br/>왼쪽 화살표<br/>오른쪽 화살표 | 사용 **탭** 볼륨 요소에 초점을 맞춥니다. **왼쪽 및 오른쪽 화살표 키** 볼륨을 각각 위아래로 이동합니다. |
| [!UICONTROL 폐쇄 캡션] (&quot;cc&quot;) | 탭<br/>입력<br/>위쪽 화살표<br/>아래쪽 화살표 | **탭** 끝 [!UICONTROL 폐쇄 캡션] (&quot;cc&quot;) 요소입니다. 사용 **입력** 메뉴를 열려면 **위쪽 및 아래쪽 화살표 키** 캡션 언어를 선택합니다. **입력** 선택 항목을 확인합니다. |
| [!UICONTROL 품질] | 탭<br/>입력<br/>위쪽 화살표<br/>아래쪽 화살표 | 사용 **탭** 을(를) 집중시키려면 [!UICONTROL 품질] 요소를 생성하지 않습니다. 사용 **입력** 메뉴 및 를 열려면 **위쪽 및 아래쪽 화살표 키** 비디오 품질을 선택합니다. **입력** 선택 항목을 확인합니다. |
| 전체 화면 | 탭<br/>스페이스바 또는 Enter<br/>이스케이프 | 사용 **탭** 전체 화면 요소에 포커스를 둡니다. 사용 **스페이스바 또는 Enter** 전체 화면 보기를 활성화합니다. **이스케이프** (&quot;esc&quot;) 전체 화면 모드를 종료합니다. |
| 닫기 | 탭<br/>스페이스바 또는 Enter | 사용 **탭** 닫기 단추에 포커스를 둡니다. 사용 **스페이스바 또는 Enter** 비디오 대화 상자를 종료하는 키입니다. |

## 파일 드래그 앤 드롭

Experience Platform에서 모든 파일 선택 영역 드래그 앤 드롭 영역에 키보드로 액세스할 수 있습니다. 사용 **탭** 강조 표시 **[!UICONTROL 파일 선택]** 및 사용 **Enter 키 또는 스페이스바** 을 선택하면 운영 체제의 파일 선택 UI가 호출됩니다.

파일이 업로드되면 삭제 아이콘을 키보드로 탐색하여 선택한 파일을 제거하고 새 파일을 업로드할 수 있습니다. 사용자는 다음을 사용할 수 있습니다 **탭** 삭제 아이콘에 포커스를 맞추려면 **Enter 키 또는 스페이스바** 을 눌러 선택합니다. 파일이 제거되면 **[!UICONTROL 파일 선택]** 은(는) 자동으로 포커스에 추가되며 선택할 수 있습니다.

또는 업로드된 파일의 형식이 올바르지 않으면 오류 메시지와 함께 오류 아이콘이 표시됩니다. **[!UICONTROL 파일 선택]** 단추는 포커스에 있으며 선택할 수 있습니다.

![파일 끌어서 놓기 영역에 오류 메시지가 표시되고 파일 선택 버튼이 포커스에 표시됩니다.](images/drag-and-drop.png)

마우스를 사용하여 드래그 앤 드롭 영역을 선택하면 파일 선택 UI가 호출되거나 마우스 사용자가 파일을 선택하고 영역으로 드래그하여 업로드를 시작할 수 있습니다.

![마우스 사용자가 파일을 영역으로 드래그할 때 포커스의 파일 드래그 앤 드롭 영역](images/drag-and-drop-mouse-over.png)

## 표 찾아보기

Experience Platform 사용자 인터페이스의 모든 표는 키보드로 액세스할 수 있습니다. 일련의 키보드 단축키를 통해 테이블 행 및 열을 탐색하고 상호 작용할 수 있습니다.

* 테이블 헤더에서 **아래쪽 화살표** 테이블을 찾습니다. 탐색할 때 표 헤더를 선택할 수 있습니다. **탭**, 그리고 를 사용하여 정렬 순서를 변경할 수 있습니다 **스페이스바**.
* **위쪽 및 아래쪽 화살표 키** 표의 행을 따라 위아래로 이동합니다.
* 행이 선택되거나 포커스가 있는 경우 **입력** 행의 은 오른쪽 레일에 세부 정보를 제공합니다.
* 행을 선택하거나 포커스에 있는 경우 **화살표 키** 을 눌러 행의 각 항목을 이동합니다.
* 사용 **입력** 행에서 항목을 선택합니다. 화면 판독기가 있는 사용자는 새 창이 열려야 할 경우 알림을 받습니다.
* 200% 이상으로 확대하면 **철도 검사원** 아이콘을 클릭하면 오른쪽 레일이 축소되어 표에 더 많은 보기 공간이 제공됩니다.

![사용자가 200%로 확대하면 포커스가 있는 레일 관리자 아이콘이 표시됩니다.](images/rail-inspector.png)

### 표 키보드 접근성 찾아보기

| 키보드 접근성 | 설명 |
|---|---|
| HOME (함수 + 왼쪽 화살표) | 행에 포커스가 있을 때, 는 사용자를 행의 첫 번째 항목으로 안내합니다 |
| END (함수 + 오른쪽 화살표) | 행에 포커스가 있을 때, 는 사용자를 행의 마지막 항목으로 안내합니다 |
| 페이지 위로 | 테이블에서 위로 10개 행 트래버스(페이지당) |
| 페이지 아래로 | 테이블에서 아래로 10개 행 트래버스(페이지당) |
| 컨트롤 + 홈 | 테이블의 첫 번째 행으로 이동 |
| 컨트롤 + 끝 | 페이지당 테이블의 첫 번째 작업으로 이동 |

## 스키마 편집기 UI

스키마 편집기 UI는 다음 기능으로 액세스할 수 있습니다.

* 스키마 편집기는 를 포함하여 키보드 탐색을 지원합니다. **탭** 를 클릭하여 UI 요소를 탐색합니다.
* **탭** 검색 필드를 입력한 다음 스키마 트리에 입력합니다.
* 스키마 트리는 스키마 트리 UI를 통해 탐색하는 화살표 키 사용을 지원합니다
   * **위쪽 및 아래쪽 화살표** 를 사용하여 트리를 트래버스할 수 있습니다.
   * **왼쪽 및 오른쪽 화살표** 를 사용하여 노드를 확장 및 축소하거나 스키마 트리에서 인라인 작업 사이를 이동할 수 있습니다.
* **입력(반환)** 오른쪽의 세부 정보 패널에서 개별 노드 세부 정보를 활성화합니다.
* 다음 **홈** 키가 트리의 맨 위로 돌아갑니다.
* 다음 **종료** 키는 트리의 맨 아래로 이동합니다.
* 스키마 트리에는 화면 판독기에 대한 ARIA 레이블도 포함됩니다.

## 세그먼트 빌더 UI

Segment Builder UI를 사용하여 Experience Platform 내에서 세그먼트를 만들고 편집하고 상호 작용하는 경우 다음 기능을 사용하여 접근성을 향상시킬 수 있습니다.

* 세그먼트 빌더 UI는 키보드 탐색을 통해 액세스할 수 있습니다.
* 화면 판독기는 제목에 대한 마크업 태그를 인식해야 하며 제목 수준과 함께 제목을 알릴 수 있습니다.
* 다른 보조 기술은 적절하게 코딩된 제목을 사용하여 개요 또는 대체 보기를 표시하여 페이지의 시각적 표시를 변경할 수 있습니다.

이제 세그먼트 빌더 캔버스의 왼쪽 및 오른쪽 레일을 축소하거나 확장하여 화면 공간을 더 확보할 수 있습니다. 이 기능은 200% 확대/축소에서 전체 기능 기능을 제공하므로 특히 유용합니다.

![왼쪽 및 오른쪽 레일 공개 위젯이 강조 표시된 세그먼트 빌더 캔버스.](images/left-right-rail-expandables.png)

## 쿼리 서비스 편집기

쿼리 서비스 편집기에서 다음 접근성 기능을 사용할 수 있습니다.

* 쿼리 서비스 편집기 UI의 색상 대비는 접근성 규정 준수를 충족합니다.
* 키보드 탐색은 편집기 UI 외부에서 지원됩니다. 편집기 UI는 임베드된 코드 미러입니다.

## 소스 및 대상의 시스템 보기 탭

검색 시 **[!UICONTROL 시스템 보기]** 소스 및 대상에서 다음 기능을 사용하여 액세스 가능성을 높일 수 있습니다.

* **탭** 첫 번째 소스 연결 카드에 초점을 설정합니다.
   * **탭** 다시 카드 내부의 버튼에 초점 맞추기
   * 선택 **입력** 카드 내에서 콜 투 액션 버튼 활성화
* 선택 **입력** 연결 카드의 오른쪽 레일에서 세부 정보 를 활성화합니다
   * 오른쪽 레일이 활성화되면 해당 영역에 포커스가 설정됩니다. **탭** 다음에 중점 **닫기** 오른쪽 레일 창용. 선택 **탭** 다시 오른쪽 레일 패널을 통해 포커스를 이동합니다.
   * 소스 연결 카드가 두 개 이상인 경우 **탭** 연결을 통해 이동
   * 사용 **화살표 키(위쪽, 아래쪽, 왼쪽 및 오른쪽)** 소스 목록을 통해 이동
   * 선택 **탭** 오른쪽 레일 패널에 포커스 설정하기
