# 액세스 제어 개요

Adobe Admin Console을 통해 경험 플랫폼에 대한 액세스 제어를 [제공할 수 있습니다](https://adminconsole.adobe.com). 이 기능은 사용 권한 및 샌드박스를 사용자와 연결하는 Admin Console의 제품 프로필을 활용합니다.

## 액세스 제어 계층 구조 및 워크플로우

경험 플랫폼의 액세스 제어를 구성하려면 Experience Platform 제품 통합이 있는 조직에 대한 관리자 권한이 있어야 합니다. 권한을 부여하거나 해지하는 최소 역할은 **제품 프로필 관리자입니다**. 권한을 관리할 수 있는 다른 관리자 역할은 **제품 관리자** (제품 내의 모든 프로필을 관리할 수 있음)와 **시스템 관리자** (제한 없음)입니다. 자세한 내용은 [관리 역할에](https://helpx.adobe.com/enterprise/using/admin-roles.html) 대한 Adobe 도움말 센터 문서를 참조하십시오.

>[!NOTE] 이 시점부터 이 문서의 &quot;관리자&quot;에 대한 모든 언급은 제품 프로필 관리자 이상의(위에 설명된 대로)를 의미합니다.

액세스 권한을 획득하고 할당하는 고급 워크플로우는 다음과 같이 요약할 수 있습니다.

- Adobe Experience Platform을 구독하면 등록 양식에 지정된 관리자에게 이메일이 전송됩니다.
- 관리자는 Adobe Admin [Console에](#adobe-admin-console) 로그인하여 개요 **페이지의 제품 목록에서** Adobe Experience Platform을 선택합니다.
- 관리자는 기본 [제품 프로필을](#product-profiles) 보거나 필요에 따라 새 고객 제품 프로필을 만들 수 있습니다.
- 관리자는 기존 제품 프로필에 대한 권한 및 사용자를 편집할 수 있습니다.
- 제품 프로필을 만들거나 편집할 때 관리자는 **사용자** 탭을 사용하여 프로필에 사용자를 추가하고 권한 **** 탭에 액세스하여 이러한 사용자(&quot;데이터 집합 읽기&quot; 또는 &quot;스키마 관리&quot; 등)에게 권한을 부여합니다. 마찬가지로 관리자는 동일한 권한 탭을 사용하여 샌드박스에 대한 액세스 권한을 할당할 수 있습니다.
- 사용자가 Experience Platform 사용자 인터페이스에 로그인하면, 플랫폼 기능에 대한 액세스 권한은 2단계에서 사용자에게 부여된 권한에 의해 결정됩니다. 예를 들어 사용자에게 &quot;데이터 집합 보기&quot; 권한이 없는 경우 *사이드* 메뉴의 데이터 집합 탭이 해당 사용자에게 표시되지 않습니다.

Experience Platform에서 액세스 제어를 관리하는 방법에 대한 자세한 내용은 [액세스 제어 사용 안내서를](./ui/overview.md)참조하십시오.

Experience Platform API에 대한 모든 호출은 사용 권한에 대해 유효성이 확인되며, 현재 사용자 컨텍스트에서 해당 권한이 없으면 오류가 반환됩니다. UI 내에서 요소는 현재 사용자에게 부여된 권한에 따라 숨기거나 변경됩니다.

## Adobe Admin Console

Adobe Admin Console은 조직의 Adobe 제품 권한 및 액세스 권한을 중앙에서 관리할 수 있는 기능을 제공합니다. 콘솔을 통해 사용자 그룹에 &quot;데이터 세트 관리&quot;, &quot;데이터 세트 보기&quot; 또는 &quot;프로필 관리&quot;와 같은 다양한 플랫폼 기능에 대한 액세스 권한을 부여할 수 있습니다.

### 제품 프로필

관리 콘솔에서 **제품 프로필을**&#x200B;사용하여 사용자에게 권한이 할당됩니다. 제품 프로필을 사용하면 하나 이상의 사용자에게 권한을 부여할 수 있으며 제품 프로필을 통해 사용자에게 할당된 샌드박스의 범위에 대한 액세스 권한을 포함할 수 있습니다. 사용자는 조직에 속한 하나 또는 여러 제품 프로필에 할당할 수 있습니다.

### 기본 제품 프로필

Experience Platform은 사전 구성된 두 개의 기본 제품 프로필을 제공합니다. 다음 표에서는 액세스 권한을 부여하는 샌드박스 및 해당 샌드박스의 범위 내에서 부여하는 권한을 포함하여 각 기본 프로파일에서 제공되는 사항에 대해 간략하게 설명합니다.

| 제품 프로필 | 샌드박스 액세스 | 권한 |
| --- | --- | --- |
| 기본 프로덕션 - 모든 액세스 | 프로덕션 | 샌드박스 관리 권한을 제외하고 경험 플랫폼에 적용되는 모든 권한 |
| 기본 샌드박스 관리 | N/A | 샌드박스 관리 권한에 대한 액세스 권한만 제공합니다. |

## 샌드박스 및 권한

Experience Platform을 사용하면 하나의 프로덕션 샌드박스에 액세스할 수 있고 비프로덕션 **샌드박스를**&#x200B;만들 수 있습니다. 비프로덕션 샌드박스는 다른 샌드박스에서 데이터를 분리할 수 있도록 해주는 데이터 가상화의 한 형태이며 일반적으로 개발 실험, 테스트 또는 테스트에 사용됩니다. 제품 프로필의 **권한은** 프로필 사용자에게 액세스 권한이 부여된 샌드박스 환경 내의 플랫폼 기능에 대한 액세스 권한을 부여합니다.

경험 플랫폼의 샌드박스에 대한 자세한 내용은 [샌드박스 개요를](../sandboxes/home.md)참조하십시오.

### 샌드박스 액세스

샌드박스 액세스는 제품 프로필을 통해 관리됩니다. 제품 프로필의 샌드박스에 대한 액세스를 활성화하는 방법에 대한 자세한 내용은 [액세스 제어 사용 안내서를](./ui/overview.md)참조하십시오.

사용자는 제품 프로필 내에서 하나 이상의 샌드박스에 대한 액세스 권한을 부여할 수 있습니다. 한 사용자가 두 개 이상의 제품 프로필에 포함되어 있는 경우 해당 사용자는 해당 프로필에 포함된 모든 샌드박스에 액세스할 수 있습니다.

&quot;샌드박스 관리&quot; 권한을 사용하면 샌드박스를 관리, 보기 또는 재설정할 수 있습니다.

### 권한

제품 프로필 내의 **권한** 탭에는 해당 프로필에 대해 활성화된 샌드박스 및 권한이 표시됩니다.

![](./images/permissions-overview.png)

Admin Console을 통해 부여된 권한은 카테고리별로 정렬되며, 일부 권한은 몇 가지 하위 수준 기능에 대한 액세스 권한을 부여합니다.

다음 표에서는 액세스 권한을 부여하는 특정 플랫폼 기능에 대한 설명과 함께 관리 콘솔에서 경험 플랫폼에 대한 사용 가능한 권한에 대해 간략히 설명합니다. 제품 프로필에 권한을 추가하는 방법에 대한 자세한 내용은 [액세스 제어 사용 안내서를](./ui/overview.md)참조하십시오.

| 카테고리 | 사용 권한 | 설명 |
| --- | --- | --- |
| 데이터 모델링 | 스키마 관리 | 스키마 및 관련 리소스를 읽기, 생성, 편집 및 삭제할 수 있습니다. |
| 데이터 모델링 | 스키마 보기 | 스키마 및 관련 리소스에 대한 읽기 전용 액세스 |
| 데이터 관리 | 데이터 세트 관리 | 데이터 세트를 읽고, 만들고, 편집하고, 삭제할 수 있습니다. 스키마에 대한 읽기 전용 액세스 |
| 데이터 관리 | 데이터 집합 보기 | 데이터 집합 및 스키마에 대한 읽기 전용 액세스 |
| 데이터 관리 | 데이터 모니터링 | 데이터 세트 및 스트림에 대한 읽기 전용 액세스 |
| 프로필 관리 | 프로필 관리 | 고객 프로파일에 사용되는 데이터 세트를 읽고, 생성하고, 편집하고, 삭제할 수 있습니다. 사용 가능한 프로파일에 대한 읽기 전용 액세스 |
| 프로필 관리 | 프로필 보기 | 사용 가능한 프로파일에 대한 읽기 전용 액세스 |
| 프로필 관리 | 세그먼트에 대한 대상 내보내기 | 평가된 대상 세그먼트를 데이터 세트로 내보내는 기능 |
| ID | ID 네임스페이스 관리 | ID 네임스페이스를 읽고, 만들고, 편집하고, 삭제할 수 있습니다. |
| ID | ID 네임스페이스 보기 | ID 네임스페이스에 대한 읽기 전용 액세스 |
| 샌드박스 관리 | 샌드박스 관리 | 샌드박스를 읽고, 만들고, 편집하고, 삭제할 수 있습니다. |
| 샌드박스 관리 | 샌드박스 보기 | 조직에 속한 샌드박스에 대한 읽기 전용 액세스 |
| 샌드박스 관리 | 샌드박스 재설정 | 샌드박스를 재설정하는 기능 |
| 대상 | 대상 관리 | 대상을 읽기, 만들기, 편집 및 비활성화할 수 있습니다.* |
| 대상 | 대상 보기 | [카탈로그] 탭에서 사용 가능한 대상에 대한 읽기 *전용* 액세스 및 [검색] *탭의 인증된 대상에* 액세스할수 있습니다.* |
| 대상 | 대상 활성화 | 생성된 활성 대상에 데이터를 활성화하는 기능 이 권한을 사용하려면 대상을 활성화할 사용자에게 &quot;대상 보기&quot; 또는 &quot;대상 관리&quot;를 부여해야 합니다.* |
| 데이터 통합 | 소스 관리 | 소스를 읽고, 만들고, 편집하고, 비활성화할 수 있습니다. |
| 데이터 통합 | 소스 보기 | [카탈로그] 탭의 사용 가능한 소스 및 [ *검색* ] 탭의 인증된 소스에 대한 읽기 전용 *액세스입니다* . |
| 데이터 과학 작업 영역 | 데이터 과학 작업 공간 관리 | 데이터 과학 작업 공간에서 읽고, 만들고, 편집하고, 삭제할 수 있습니다. |

_(*) 이 권한을 사용하려면 실시간 고객 데이터 플랫폼에 대한 조항이 필요합니다. 실시간 CDP에 대한 자세한 내용은 실시간 CDP[개요를](https://docs.adobe.com/content/help/en/experience-platform/rtcdp/overview.html)읽어 보시기 바랍니다._
