# Adobe Experience Platform 용어 {#adobe-experience-platform-glossary}

## A

**액세스 제어**:역할 기반 액세스 제어를 통해 관리자는 Experience Platform 사용자에게 액세스 및 권한을 할당할 수 있습니다. 사용 권한에는 샌드박스 만들기, 스키마 정의, 데이터 집합 관리 등 Experience Platform 기능을 보거나 사용할 수 있는 기능이 포함됩니다.

**액세스 키 ID**:액세스 키 ID는  [!DNL Amazon] S3 비밀 액세스 키와 연관된 고유 식별자입니다. 액세스 키 ID와 비밀 액세스 키는 함께 사용하여 [!DNL Amazon Web Services] (AWS) 요청을 서명합니다.

**작업**:에서  [!DNL Platform Launch]작업은 이벤트가 발생하고 조건이 평가 및 전달된 후 발생할 상황을 정의하는 특정 유형의 규칙 구성 요소입니다.

**활성화**:활성화는 세그먼트 또는 프로필을 대상(예:, 또는)에 매핑하기 위해 사용자가 수행하는  [!DNL Oracle Eloqua] [!DNL Google]작업입니다 [!DNL Salesforce Marketing Cloud].

**활동**:에서 활동 [!DNL Offer Decisioning]에는 오퍼의 선택을 알리는 논리가 포함됩니다.

**관리자**:Adobe Admin Console에서 Experience Platform에 대한 권한을 구성하고 사용자 정의할 수 있는 조직의 한 명 이상의 개인 사용자.

**Adobe Admin Console**:Adobe Admin Console은 조직에 대한 Adobe 제품 권한 및 액세스를 관리할 수 있는 중앙 위치를 제공합니다. 이 콘솔을 통해 관리자는 사용자 그룹에 &quot;데이터 세트 관리&quot;, &quot;데이터 세트 보기&quot; 또는 &quot;프로필 관리&quot;와 같은 다양한 플랫폼 기능에 대한 액세스 권한을 부여할 수 있습니다.

**Adobe Experience Platform**:Adobe Experience Platform은 기업 전체의 데이터 및 컨텐츠를 표준화하며 실시간 고객 프로파일을 강화하며 데이터 과학을 지원하며 컨텐츠 속도를 가속화하여 고객 여정 전반에 걸친 경험 개인화를 유도합니다.

**Adobe Experience Platform Launch**: [!DNL Platform Launch] 는 Experience Platform 및 애플리케이션과 통합된 태그 및 SDK 관리  [!DNL Experience Cloud] 에코시스템입니다. [!DNL Platform Launch] 모든 클라이언트 디바이스에서 관련 고객 경험을 향상시키는 데 필요한 분석, 마케팅 및 광고 통합을 배포, 통합 및 관리하는 툴을 제공합니다.

**Adobe Experience Platform 쿼리 서비스**:데이터 분석가가 분석 및 머신 러닝에 사용하기 위해 이벤트와 프로파일을 쿼리할 수 있습니다. 데이터 과학자와 분석가는 쿼리 서비스를 사용하여 Experience Platform에 저장된 모든 데이터 세트(POS(Point-Of-Sale), 고객 관계 관리(CRM) 등)를 가져올 수 있고 이러한 데이터 세트를 쿼리하여 데이터에 대한 특정 질문에 답변할 수 있습니다.

**Adobe Experience Platform 세그멘테이션 서비스**:세그먼트를 작성하고 실시간 고객 프로필 데이터에서 대상을 생성할 수 있습니다. 이러한 대상은 Data Lake 내의 데이터 세트로 내보낼 수 있습니다.

**Adobe 지능형 서비스**:Attribution AI 및 고객 AI와 같은 지능형 서비스는 특정 목적을 위해 구축되어 Experience Platform이 실행 및 작동해야 하는 기계 학습 기반의 모델입니다.

**Adobe I/O**:Adobe I/O은 Experience Platform에 포함되어 있으며 API, 이벤트, 개발자 콘솔, 유용한 툴 등 개발자가 플랫폼을 통합, 확장 및 사용자 정의하는 데 필요한 모든 것을 이용할 수 있습니다.

**Adobe Sensei**:Adobe Sensei은 Experience Platform을 강화하는 인텔리전스 프레임워크입니다. 또한 브랜드 기업이 개인화된 실시간 고객 경험을 제공할 수 있는 역량을 강화할 수 있는 일련의 AI 서비스를 제공합니다.

**Amazon S3 버킷**: [!DNL Amazon S3] 버킷은 에코시스템에 저장된 데이터의 기본  [!DNL Amazon] 컨테이너입니다. 버킷에는 개체가 포함되어 있으며 각 개체는 개발자가 지정한 고유한 키를 사용하여 저장되고 검색됩니다.

**Amazon S3 커넥터**:Experience Platform 고객은  [!DNL Amazon] S3 커넥터를 사용하여  [!DNL Amazon] S3 데이터에 안전하게 연결하고 액세스할 수 있습니다.

**저장 전략 추가**:&quot;추가&quot; 저장 전략은 연결을 통해 인제스트할 제3자 데이터를 지정하고 데이터 세트 끝에 새 데이터나 행을 추가할 때 사용하는 옵션입니다. 이전에 인제스트한 행은 그대로 유지되며 마지막으로 예약된 실행 이후 만들어진 행만 Experience Platform에 인제스트됩니다. 소스 시스템에서 변경된 모든 행은 Experience Platform에서 변경되지 않습니다.

**배열**:배열은 동일한 데이터 유형의 순서가 지정된 요소에 사용됩니다.

**인공 지능**:인공 지능은 시각 인식, 음성 인식, 의사 결정, 언어 간 번역 등과 같이 인간의 지능을 필요로 하는 작업을 수행할 수 있는 컴퓨터 시스템의 이론과 개발입니다.

**속성**:속성은 프로파일을 나타내는 지정된 특성입니다.

**속성 병합**:실시간 고객 프로필 API를 사용하여 병합 정책을 정의할 때 개체는  `attributeMerge` 데이터 충돌 시 병합 정책이 프로필 속성의 우선 순위를 지정하는 방법을 나타냅니다. 플랫폼 UI에서 병합 정책을 정의할 때 [!UICONTROL Merge method]을 선택하는 것과 같습니다.

**Attribution AI**: [!DNL Attribution AI] 은 고객 라이프사이클에서 알고리즘 방식의 멀티채널 기여도 분석 기능을 제공하는 Adobe Sensei 기반의 지능형 서비스입니다.

**대상**:대상자는 세그먼트 정의 기준을 충족하는 결과 프로필 세트입니다.

**대상 크기**:대상자 크기는 세그먼트 정의 기준을 충족하고 대상 멤버십에 가입할 수 있는 총 프로필 수입니다.

**대상 스냅샷**:대상 스냅숏은 세그먼테이션 시 세그먼트 기준에 맞는 모든 프로필을 캡처합니다.

## B

**채우기**:예약된 소스의 경우 채우기 옵션을 사용하여 내역 데이터를 수집할 수 있습니다.

**채우기 기간**:채우기 기간은 소스 연결을 통해 제3자 내역 데이터를 인제하는 데 걸리는 시간을 설정하는 옵션입니다. &quot;평생&quot;의 채우기 기간을 선택하면 소스 데이터의 전체 기록이 Experience Platform에 인제스트됩니다.

**일괄** 처리:배치란 일정 기간 동안 수집되어 하나의 단위로 함께 처리되는 데이터 세트입니다. 데이터 세트는 여러 배치로 구성됩니다.

**일괄 처리 ID**:배치 ID는 데이터 일괄 처리에 대해 Adobe에서 생성된 식별자입니다.

**일괄 처리**:일괄 처리를 사용하면 데이터를 일괄 처리 파일로 Experience Platform에 인제스트할 수 있습니다. 배치는 단일 단위로 수집할 하나 이상의 파일로 구성된 데이터 단위입니다.

**일괄 세분화**:일괄적인 세그먼테이션은 지속적인 데이터 선택 프로세스에 대한 대안이며 세그먼트 정의를 통해 모든 프로필 데이터를 한 번에 이동하여 해당 고객을 생성합니다. 이 세그먼트를 만들면, 이 세그먼트가 저장되고 저장되므로 사용하도록 내보낼 수 있습니다.

**빌드**:에서  [!DNL Platform Launch]빌드는 라이브러리의 내부에 포함된 비즈니스 논리를 실행하는 데 필요한 모든 구성 및 코드를 포함하는 파일 또는 파일 집합이므로 웹 사이트 또는 모바일 앱에 해당 라이브러리를 배포할 수 있습니다.

**비즈니스 인텔리전스 툴**:비즈니스 인텔리전스(BI) 도구는 주로 통합되어  [!DNL Experience Platform Query Service]있습니다. BI 툴은 내부 및 외부 시스템에서 대량의 비정형 데이터를 수집하고 처리하는 애플리케이션 소프트웨어 유형입니다.

## C

**매핑**:에서  [!DNL Offer Decisioning]호출(빈도 설정이라고도 함)은 오퍼가 표시되는 횟수를 정의하는 의사 결정 규칙에 사용됩니다. 두 가지 유형의 대문자를 사용할 수 있습니다.결합된 타겟 대상자(&quot;전체 상한&quot;이라고 함)에서 오퍼를 제안할 수 있는 횟수와 동일한 최종 사용자에게 오퍼를 제안할 수 있는 횟수(&quot;프로필 상한&quot;이라고 함)

**카탈로그**:소스 및 대상 컨텍스트에서 카탈로그는 Adobe 애플리케이션과 타사 기술에 대한 사용 가능한 연결을 제공하는 갤러리입니다. [!DNL Catalog Service]과(와) 혼동하지 마십시오.

**[!DNL Catalog Service]**: [!DNL Catalog Service] ( [!DNL Catalog]라고도 함)는 Adobe Experience Platform 내의 데이터 위치 및 리니지에 대한 레코드 시스템입니다. Experience Platform에 인제스트된 모든 데이터는 파일 및 디렉터리로 데이터 레이크에 저장되지만 [!DNL Catalog]은 조회, 모니터링 및 데이터 거버넌스를 위해 해당 파일 및 디렉토리의 메타데이터와 설명을 보관합니다.

**클래스**:XDM(Experience Data Model)에서 클래스는 스키마를 작성하는 데 사용되는 가장 작은 필드 집합을 정의하고 스키마가 나타내는 비즈니스 개체의 기본 동작을 정의합니다.

**클라이언트**:클라이언트는 PostgreSQL 프로토콜 또는 HTTP API를  [!DNL Query Service] 통해 연결하는 외부 도구 또는 응용 프로그램입니다.

**컬렉션**:에서  [!DNL Offer Decisioning]컬렉션은 오퍼의 카테고리와 같이 마케터가 정의한 사전 정의된 조건을 기반으로 하는 오퍼의 하위 세트입니다.

**PII 마케팅 활동과 결합**:개인 식별 정보(PII)와 익명 데이터를 결합하는 마케팅 작업입니다. 광고 네트워크, 광고 서버 및 제3자 데이터 공급자로부터 제공되는 데이터에 대한 계약에는 이러한 데이터를 직접 식별할 수 있는 데이터와 함께 사용하는 것에 대한 특정 계약 금지 규정이 포함되어 있습니다.

**명령줄 인터페이스**:명령줄 인터페이스는 원시 쿼리 실행에 연결할 때 사용할 수 있는 텍스트 기반  [!DNL Query Service] 도구입니다.

**컴포지션**:컴포지션은 스키마를 작성하기 위해 함께 구성하는 구성 요소의 그룹입니다.

**조건**:에서 조건 [!DNL Platform Launch]은 반환해야 하는 논리 문을 평가하는 규칙 구성  `true` 요소입니다 `false`. 모든 조건은 `true`으로 평가되어야 하며 모든 예외 조건은 규칙에 대한 작업이 실행되기 전에 `false`으로 평가되어야 합니다.

**콘솔**:에서 콘솔 [!DNL Query Service]은 쿼리의 상태 및 작업에 대한 정보를 제공합니다. 콘솔에는 [!DNL Query Service]에 대한 연결 상태, 실행 중인 쿼리 작업 및 이러한 쿼리에서 발생한 오류 메시지가 표시됩니다.

**계약(&quot;C&quot;) 레이블**:계약(&quot;C&quot;) 데이터 사용 레이블은 계약 의무가 있거나 고객의 데이터 관리 정책과 관련된 데이터를 분류하는 데 사용됩니다.

**C1 계약 레이블**:계약  `C1` 데이터 사용 레이블은 개별 또는 장치 식별자를 포함하지 않고 Adobe Experience Cloud에서 집계된 양식으로만 데이터를 내보낼 수 있도록 지정합니다. 예를 들어 소셜 네트워크에서 가져온 데이터입니다.

**C2 계약 레이블**:계약  `C2` 데이터 사용 레이블은 제3자로 내보낼 수 없는 데이터를 지정합니다. 일부 데이터 제공업체는 원래 데이터가 수집된 위치에서 내보내기를 금지하는 계약을 가지고 있습니다. 예를 들어, 소셜 네트워크 계약에서는 수신한 데이터의 전송을 종종 제한합니다. C2는 집계 및 익명 데이터만 필요한 C1보다 더 제한적인 것입니다.

**C3 계약 레이블**:계약  `C3` 데이터 사용 레이블은 결합할 수 없거나 직접 식별 가능한 정보와 함께 사용할 수 없는 데이터를 지정합니다. 일부 데이터 제공업체는 계약에 직접 식별 가능한 정보와 함께 해당 데이터의 조합 또는 사용을 금지하는 약관을 가지고 있습니다. 예를 들어 광고 네트워크, 광고 서버 및 제3자 데이터 공급자에서 가져온 데이터에 대한 계약에는 종종 직접 식별 가능한 데이터의 사용에 대한 특정 계약 금지 사항이 포함됩니다.

**C4 계약 레이블**:계약  `C4` 데이터 사용 레이블은 온사이트 또는 크로스 사이트 광고나 컨텐트를 타게팅하는 데 데이터를 사용할 수 없음을 지정합니다. C4는 C5, C6 및 C7 레이블을 포함할 때 가장 제한적인 레이블입니다.

**C5 계약 레이블**:계약  `C5` 데이터 사용 레이블은 관심 기반 컨텐트 또는 광고의 크로스 사이트 타깃팅에 데이터를 사용할 수 없음을 지정합니다. 다음 3가지 조건이 충족되면 관심 기반 타깃팅 또는 개인화가 발생합니다.사이트에서 수집된 데이터는 사용자의 관심사에 대한 참조를 위해 사용됩니다.다른 사이트 또는 앱과 같은 다른 컨텍스트에서 사용됨를 사용하여 이러한 참조를 기반으로 제공되는 컨텐트 또는 광고를 선택합니다.

**C6 계약 레이블**:계약  `C6` 데이터 사용 레이블은 데이터를 온사이트 광고 타깃팅에 사용할 수 없음을 지정합니다. 온사이트 광고 타깃팅에는 조직의 웹 사이트 또는 앱에서 광고를 선택 및 전달하거나 이러한 광고의 전달 및 효과를 측정하는 것이 포함됩니다. 여기에는 사용자가 광고를 선택하도록 하기 위해 이전에 수집한 온사이트 데이터, 표시되는 광고의 데이터, 광고의 표시 시기 및 위치, 사용자가 광고 선택 또는 구매와 같이 광고와 관련된 조치를 취했는지 여부 등이 포함됩니다.

**C7 계약 레이블**:계약  `C7` 데이터 사용 레이블은 데이터를 컨텐츠의 온사이트 타깃팅에 사용할 수 없음을 지정합니다. 온사이트 컨텐츠 타깃팅에는 조직의 웹 사이트 또는 앱에서 컨텐츠를 선택 및 전달하거나 이러한 컨텐츠의 전달 및 효과를 측정할 수 있는 기능이 포함되어 있습니다. 여기에는 컨텐츠 선택을 위한 사용자의 관심사, 표시되는 컨텐츠에 대한 데이터 처리, 표시되는 빈도 또는 시간, 표시되는 날짜와 위치, 컨텐츠 선택 등 컨텐츠와 관련된 작업이 사용되었는지 여부에 대한 정보가 포함됩니다.

**C8 계약 레이블**:계약  `C8` 데이터 사용 레이블은 조직의 웹 사이트 또는 앱을 측정하는 데 데이터를 사용할 수 없음을 지정합니다. 여기에는 컨텐츠 및/또는 광고를 다른 컨텍스트에서 추후 개인화하기 위해 이 서비스를 사용하는 방법에 대한 정보 수집인 관심 기반 타깃팅이 포함되지 않습니다.

**C9 계약 레이블**:계약  `C9` 데이터 사용 레이블은 데이터를 데이터 과학 워크플로우에서 사용할 수 없음을 지정합니다. 일부 계약에는 데이터 과학에 사용되는 데이터에 대한 명시적 금지 사항이 포함됩니다. 인공 지능(AI), 머신 러닝(ML) 또는 모델링에 데이터 사용을 금지하는 용어로 표현되기도 합니다.

**C10 계약 레이블**:계약  `C10` 데이터 사용 레이블은 연결된 ID 활성화에 데이터를 사용할 수 없음을 지정합니다. 일부 데이터 사용 정책은 개인화를 위해 연결된 ID 데이터의 사용을 제한합니다. 병합 정책이 &quot;비공개 그래프&quot; 옵션을 사용하는 경우 `C10` 레이블이 세그먼트에 자동으로 적용됩니다.

**만든 날짜 열**:만들어진 날짜 열을 선택하는 것은 소스 연결을 통해 타사 데이터를 지정하는 옵션입니다. 추가 저장 전략을 선택하고 데이터 세트 스키마에 여러 날짜 필드가 포함되어 있는 경우 사용 가능한 스키마에서 선택하여 만든 날짜 키 열을 지정해야 합니다. 저장 전략을 덮어쓸 때 생성된 날짜 옵션을 사용할 수 없습니다.

**선택 항목으로 표 만들기**:CTAS(Create Table as Select)는 완전한 유효한 SQL 쿼리의 일부로 실행될 때 데이터 세트에 쿼리 결과 [!DNL Query Service] 를 유지하도록 지시하는 SQL 명령입니다. 새 결과 세트를 만들거나 이전 결과를 덮어쓰거나 이전 결과에 추가할 수 있습니다.

**사이트 간 데이터**:사이트 간 데이터는 사이트 내 데이터와 오프사이트 데이터의 조합 또는 여러 외부 소스의 데이터 조합을 포함하여 여러 사이트의 데이터를 조합한 것입니다.

**사이트 간 타깃팅 마케팅 작업**:크로스 사이트 광고 타깃팅에 데이터를 사용하는 마케팅 작업입니다. 사이트 내 데이터와 오프사이트 데이터의 조합 또는 여러 오프라인 소스의 데이터 조합을 포함하여 여러 사이트의 데이터 조합을 사이트 간 데이터라고 합니다. 교차 사이트 데이터는 일반적으로 고객의 관심사에 대한 참조를 위해 수집 및 처리됩니다.

**사용자 지정 ID 네임스페이스**:사용자 지정 ID 네임스페이스는 특정 조직 또는 비즈니스 사례에 대한 ID를 나타내기 위해 조직에서 만들 수 있습니다.

**사용자 지정 레이블**:사용자 지정 데이터 사용 레이블을 사용하면 특정 비즈니스 요구 사항을 충족하는 데이터 필드에 특정 레이블을 만들어 적용할 수 있습니다.

**고객 AI**:고객 AI는 AI 기반 자산으로 고객 프로파일을 강화하고 고객 세분화와 타겟팅 노력을 강화시켜주는 Adobe Sensei 기반의 인텔리전트 서비스입니다.

## D

**데이터 사전**:데이터  [!DNL Platform Launch]사전(데이터 맵이라고도 함)은 속성 내에 정의된 데이터 요소 집합입니다.

**데이터 요소**:에서  [!DNL Platform Launch]데이터 요소는 클라이언트 장치에 있는 특정 데이터 조각을 가리키기 위해 규칙 및 확장 내에서 사용되는 포인터입니다.

**데이터 통합**:데이터 수집은 소스에서 Experience Platform으로 데이터를 추가하는 프로세스입니다. 데이터를 스트리밍, 배치 또는 소스 커넥터를 통해 추가하는 등 다양한 방법으로 플랫폼에 인제스트할 수 있습니다.

**데이터 레이어**:에서  [!DNL Platform Launch]데이터 레이어는 페이지나 화면을 보는 컨텍스트에 대한 메타데이터를 포함하는 클라이언트 장치에 있는 데이터 구조입니다.

**데이터 거버넌스**:데이터 거버넌스는 데이터 사용과 관련된 규정 및 조직 정책을 준수하는 데이터를 보장하는 데 사용되는 전략과 기술을 포함합니다.

**데이터 통합 파트너**:데이터 통합 파트너는 코드를 작성하지 않고도 200개 이상의 소스에서 Experience Platform으로 대용량 데이터의 로드 및 변환을 간소화하고 자동화합니다.

**데이터 세트 레이블**:데이터 사용 레이블을 데이터 세트에 추가할 수 있습니다. 해당 데이터 집합 내의 모든 필드는 데이터 집합의 레이블을 상속합니다.

**데이터 과학 작업 공간**: [!DNL Data Science Workspace] Experience Platform을 통해 고객은 플랫폼 및 Adobe 애플리케이션에서 데이터를 활용하는 머신 러닝 모델을 작성하여 지능적인 세그먼트를 생성하고 인사이트를 생성하며 예측을 제공하여 최종 사용자 디지털 경험을 대폭 향상시킬 수 있습니다.

**데이터 소스**:데이터 소스는 사용자가 지정한 데이터의 소스입니다. 데이터 소스의 예는 모바일 앱, 프로필 및/또는 경험 이벤트, 웹 사이트 프로필 이벤트 또는 CRM입니다.

**데이터 관리자**:데이터 관리자는 조직의 데이터 자산을 관리, 감독 및 관리하는 담당자입니다. 또한 데이터 관리자는 데이터 거버넌스 정책을 안전하게 보호하고 정부 규정 및 조직 정책을 준수하도록 유지 관리할 수 있습니다.

**데이터 스트림**:데이터 스트림은 동일한 스키마를 공유하고 동일한 소스에서 보내는 메시지 집합 또는 모음입니다.

**데이터 유형**:데이터 유형은 계층적 표현으로 여러 속성이 포함된 객체 유형 필드를 정의하는 재사용 가능한 XDM 리소스입니다.

**데이터 사용 레이블**:데이터 사용 레이블을 사용하면 개인 정보 관련 고려 사항 및 계약 조건을 반영하는 데이터를 분류하여 규정 및 기업 정책을 준수할 수 있습니다. 데이터 세트에 추가된 데이터 사용 레이블은 상속되거나 해당 데이터 세트 내의 모든 필드에 적용됩니다. 데이터 사용 레이블을 필드에 직접 적용할 수도 있습니다.

**데이터 흐름**:데이터 흐름(Dataflow)은 소스에서 대상으로 또는 플랫폼으로 이동하는 데이터의 가상 파이프라인입니다.

**데이터 흐름 실행**:데이터 흐름 실행은 사용자가 지정한 일정에 따라 Experience Platform으로 이동하는 데이터 흐름

**데이터 세트**:데이터 집합은 스키마(열) 및 필드(행)를 포함하는 데이터 수집을 위한 저장소 및 관리 구조입니다.

**데이터 집합 ID**:인제스트된 데이터 세트에 대한 Adobe 생성 식별자입니다.

**데이터 세트 출력**:데이터 집합 출력에서는 특정 실행에 사용할 &quot;테이블을 선택하여 만들기&quot; 옵션을 결정하는 메커니즘을  [!DNL Query Service] 제공합니다.

**델타 열**:델타 열을 사용하면 증분 수집에 대한 타임스탬프를 나타내는 소스 데이터 필드를 선택할 수 있습니다.

**델타 저장 전략**:델타 저장 전략은 소스 연결을 통해 타사 데이터를 인제스트하는 옵션입니다. 이 옵션을 사용하면 소스 데이터의 새 행 또는 변경된 행을 Experience Platform으로 인제스트하도록 지정할 수 있습니다. 새 행이 데이터 세트 끝에 추가되고 변경된 행이 Experience Platform의 데이터 세트에 업데이트됩니다.

**설명자**:XDM(경험 데이터 모델)에서 설명자는 필드의 특정 동작을 설명하는 추가 스키마 관련 메타데이터 세트입니다. Experience Platform에서 설명자를 사용하여 두 스키마 간의 관계와 같은 의도한 스키마 동작을 이해할 수 있습니다.

**대상**:대상은 대상이 활성화되고 전달되는 Adobe 애플리케이션, 광고 플랫폼, 클라우드 스토리지 서비스 또는 마케팅 서비스와 같은 모든 종단점에 대한 일반적인 용어입니다.

**대상 범주**:대상 카테고리는 유사한 특성을 가진 대상의 그룹핑입니다.

**대상 카탈로그**:대상 카탈로그는 Experience Platform에서 사용할 수 있는 대상 목록입니다.

**직접 호출 규칙**:에서 직접 호출 규칙 [!DNL Platform Launch]은 이벤트 감지 및 조회 시스템을 무시하고 페이지에서 직접 호출할 때 실행하는 규칙입니다.

**표시 이름**:XDM(경험 데이터 모델)에서 표시 이름은 UI에 표시되는 필드의 사용자에게 친숙한 이름입니다.

## E

**적격한 혜택**:적격한 오퍼는 업스트림에 정의된 제한 조건을 충족하므로 프로파일에 일관성 있게 제공할 수 있습니다.

**자격 조건 규칙**:에서  [!DNL Offer Decisioning]자격 조건 규칙은 달력, 예약 및 제한 조건과 관련된 프로필에 적용됩니다.

**이메일 타깃팅 마케팅 작업**:이메일 타깃팅 캠페인의 데이터를 사용하는 마케팅 작업입니다.

**포함 코드**:에서  [!DNL Platform Launch]내장 코드는 사이트 또는 환경의 HTML 내에 배치된 스크립트 태그입니다. 포함 코드는 브라우저가 빌드를 검색할 위치에 지시합니다.

**열거형**:열거형(열거형)은 미리 정의된 값 세트로 제한되는 XDM 필드입니다.

**환경**:에서  [!DNL Platform Launch]환경은 빌드의 호스트 전달 및 파일 형식을 지정하는 배포 지침 세트입니다. 라이브러리를 빌드하려면 먼저 환경과 쌍을 이루어야 합니다.

**진단 오류**:오류 진단을 통해 인제스트된 배치에 대한 자세한 오류 메시지를 생성할 수 있습니다. 오류 임계값을 사용하면 일괄 처리에 실패하기 전에 허용되는 오류 비율을 구성할 수 있습니다.

**이벤트**:에서  [!DNL Platform Launch]이벤트는 규칙 구성 요소의 특정 유형이며 클라이언트 장치에서 규칙 실행을 시작하는 트리거입니다.

**XDM 결정 이벤트**:XDM 결정 이벤트는 의사 결정 활동의 결과와 컨텍스트에 대한 관찰을 캡처하는 데 사용되는 시간 시리즈 기반 클래스입니다. 여기에는 의사 결정 방법, 결정 발생 시 어떤 옵션이 제안(및 선택)되었는지, 의사 결정 과정에 영향을 미쳤거나 의사 결정 과정에서 관찰될 수 있었던 컨텍스트 상태가 존재하는지에 대한 정보가 포함됩니다.

**XDM 시스템**:XDM 시스템은 다운스트림 Experience Platform 서비스에서 사용할 XDM 스키마를 운영하는 프레임워크입니다.
