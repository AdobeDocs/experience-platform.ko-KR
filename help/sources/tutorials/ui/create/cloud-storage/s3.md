# 만들기 [!DNL Amazon S3] UI의 소스 연결

Adobe Experience Platform의 소스 커넥터는 예약된 방식으로 외부 소스 데이터를 수집할 수 있습니다. 이 자습서에서는 을(를) 만드는 단계를 제공합니다 [!DNL Amazon S3]  (이하 &quot;라 한다)[!DNL S3]&quot;) Platform 사용자 인터페이스를 사용한 소스 커넥터입니다.

## 시작하기

이 자습서에서는 Adobe Experience Platform의 다음 구성 요소를 이해하고 있어야 합니다.

- [[!DNL Experience Data Model (XDM)] 시스템](../../../../../xdm/home.md): Experience Platform이 고객 경험 데이터를 구성하는 표준화된 프레임워크입니다.
   - [스키마 작성 기본 사항](../../../../../xdm/schema/composition.md): 스키마 컴포지션의 주요 원칙 및 모범 사례를 포함하여 XDM 스키마의 기본 빌딩 블록에 대해 알아봅니다.
   - [스키마 편집기 자습서](../../../../../xdm/tutorials/create-schema-ui.md): 스키마 편집기 UI를 사용하여 사용자 지정 스키마를 만드는 방법을 알아보십시오.
- [[!DNL Real-Time Customer Profile]](../../../../../profile/home.md): 여러 소스에서 집계된 데이터를 기반으로 통합된 실시간 소비자 프로필을 제공합니다.

이미 유효한 [!DNL S3] 연결을 통해 이 문서의 나머지 부분을 건너뛰고 다음 자습서를 진행할 수 있습니다. [데이터 흐름 구성](../../dataflow/batch/cloud-storage.md).

### 필요한 자격 증명 수집

에 액세스하려면 [!DNL S3] 플랫폼의 버킷에서는 다음 자격 증명에 유효한 값을 제공해야 합니다.

| 자격 증명 | 설명 |
| ---------- | ----------- |
| `s3AccessKey` | 사용자의 액세스 키 ID입니다 [!DNL S3] 버킷. |
| `s3SecretKey` | 사용자의 암호 키 ID [!DNL S3] 버킷. |
| `serviceUrl` | (선택 사항) 사용자 지정 [!DNL S3] 연결할 끝점입니다. 이 필드는 [!DNL S3] 버킷은 영역별로 다릅니다. 의 형식 `serviceUrl` is: `https://s3.{REGION}.amazonaws.com/)`. |
| `bucketName` | 다음 [!DNL S3] 버킷에는 데이터와 해당 설명 메타데이터가 포함되어 있습니다. 사용자 [!DNL S3] 버킷 이름은 3자에서 63자 사이여야 하며 문자 또는 숫자로 시작하고 끝나야 합니다. 버킷 이름은 소문자, 숫자 또는 하이픈(`-`)이고 IP 주소로 형식을 지정할 수 없습니다. |
| `folderPath` | 폴더의 경로 [!DNL S3] 데이터가 저장되는 버킷. 이 자격 증명은 사용자가 액세스를 제한한 경우에 필요합니다. |

시작하는 방법에 대한 자세한 내용은 [이 AWS 문서](https://aws.amazon.com/blogs/security/wheres-my-secret-access-key/).

## 연결 [!DNL S3] account

플랫폼 UI에서 **[!UICONTROL 소스]** 왼쪽 탐색 모음에서 를 클릭하여 [!UICONTROL 소스] 작업 공간. 다음 [!UICONTROL 카탈로그] 화면에는 계정을 만들 수 있는 다양한 소스가 표시됩니다.

화면 왼쪽에 있는 카탈로그에서 적절한 카테고리를 선택할 수 있습니다. 또는 검색 옵션을 사용하여 작업할 특정 소스를 찾을 수 있습니다.

아래에 **[!UICONTROL 클라우드 스토리지]** 카테고리, 선택 **[!UICONTROL Amazon S3]** 그런 다음 **[!UICONTROL 데이터 추가]**.

![카탈로그](../../../../images/tutorials/create/s3/catalog.png)

다음 **[!UICONTROL Amazon S3에 연결]** 페이지가 나타납니다. 이 페이지에서 새 자격 증명 또는 기존 자격 증명을 사용할 수 있습니다.

### 새 계정

새 자격 증명을 사용하는 경우 **[!UICONTROL 새 계정]**. 표시되는 입력 양식에서 이름, 선택적 설명 및 [!DNL S3] 자격 증명. 완료되면 을 선택합니다 **[!UICONTROL 소스에 연결]** 그런 다음 새 연결이 설정될 시간을 허용합니다.

![connect](../../../../images/tutorials/create/s3/connect.png)

### 기존 계정

기존 계정을 연결하려면 [!DNL S3] 연결할 계정을 선택한 다음 **[!UICONTROL 다음]** 계속 진행합니다.

![기존](../../../../images/tutorials/create/s3/existing.png)
