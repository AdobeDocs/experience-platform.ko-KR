# 쿼리 서비스 세션 관리

이 안내서를 사용하여 Adobe Experience Platform 사용자 인터페이스에서 활성 쿼리 서비스 세션을 관리합니다. 세션 관리는 관리자가 샌드박스 간 동시 쿼리 편집기 세션을 모니터링하고 사용자가 세션을 열어 놓을 때 사용 가능한 용량을 모니터링하는 데 도움이 됩니다.

## 세션 관리에 필요한 권한 {#permissions}

>[!AVAILABILITY]
>
>세션 관리는 Data Distiller 권한이 있는 조직만 사용할 수 있습니다.

>[!IMPORTANT]
>
>이 기능은 관리자를 위한 것입니다. 쿼리를 실행하는 최종 사용자는 세션을 관리할 수 없습니다.

세션을 보고 종료하려면 Data Distiller 액세스 권한이 있는 조직에 속해 있어야 하며 **[!UICONTROL Manage Query Session]** 권한이 할당되어 있어야 합니다. 필요한 권한이 없는 사용자는 쿼리 서비스에 액세스할 수 있지만 활성 세션을 보거나 관리할 수 없습니다.

## 활성 세션 보기 {#view-active-sessions}

관리자는 조직의 샌드박스에 있는 모든 활성 Query Service 세션을 볼 수 있습니다. Experience Platform의 왼쪽 탐색에서 **[!UICONTROL Queries]**&#x200B;을(를) 선택하여 쿼리 서비스 작업 영역을 연 다음 **[!UICONTROL Admin]** 탭을 선택하여 세션 관리에 액세스합니다.

![관리자 탭을 선택한 쿼리 서비스 작업 영역입니다. [세션 관리] 테이블이 표시되고 조직의 여러 샌드박스에 걸쳐 활성 및 비활성 세션이 나열됩니다.](../images/ui/session-management/session-management-admin-tab.png)

세션 관리 테이블은 실시간으로 자동으로 업데이트되며 현재 조직에 지정된 쿼리 서비스 동시 세션 용량을 사용하는 모든 세션을 나열합니다. 각 행은 쿼리 편집기에서 열린 단일 세션을 나타냅니다.

## 세션 상태 및 유휴 시간 {#session-status}

세션 테이블은 세션을 안전하게 종료할 수 있는지 여부를 결정하는 데 도움이 되는 정보를 제공합니다.

| 열 | 설명 |
| --- | --- |
| 사용자 ID | 세션을 소유하는 사용자의 Adobe ID |
| 사용자 이름 | Adobe ID과 연결된 이름 |
| 샌드박스 | 세션이 실행 중인 샌드박스를 나타냅니다. |
| 세션 상태 | 세션이 **[!UICONTROL Active]**&#x200B;인지 **[!UICONTROL Inactive]**&#x200B;인지를 표시합니다. |
| 유휴 시간 | 상호 작용 없이 세션이 열린 기간을 표시합니다. |
| 남은 세션 시간 | 자동 만료되기 전에 세션이 열린 상태로 유지될 수 있는 기간을 나타냅니다. |

### 세션 상태

**[!UICONTROL Inactive]**&#x200B;은(는) 사용자가 현재 쿼리를 실행하고 있지 않음을 나타냅니다. 이러한 세션은 종료될 수 있습니다. **[!UICONTROL Active]**&#x200B;은(는) 쿼리가 현재 실행 중임을 나타냅니다. 쿼리 실행이 완료될 때까지 **[!UICONTROL End session]** 컨트롤을 사용할 수 없습니다.

### 유휴 시간 및 남은 세션 시간

유휴 시간은 사용자 상호 작용 없이 세션이 열린 시간을 보여 줍니다. 남은 세션 시간은 세션이 열린 상태로 유지된 후 시스템에서 자동으로 닫힙니다. 최대 허용 기간(2시간 동안 활동이 없음) 후에 세션이 자동으로 만료됩니다. 이 기간은 시스템 정의이며 구성할 수 없습니다.

## 유휴 세션 종료 {#end-idle-sessions}

유휴 세션을 종료하여 다른 사용자의 동시 세션 용량을 확보할 수 있습니다. 사용자가 더 이상 적극적으로 작업하지 않을 때 유휴 시간이 높은 세션을 종료하는 것이 좋습니다.

세션 관리 테이블에서 **[!UICONTROL End session]**&#x200B;을(를) 선택하여 종료하려는 비활성 세션을 선택합니다.

![비활성 세션을 보여주는 세션 관리 테이블(종료 세션이 강조 표시됨).](../images/ui/session-management/end-session.png)

실수로 종료되지 않도록 확인 대화 상자가 나타납니다. 대화 상자에서 **[!UICONTROL End session]**&#x200B;을(를) 선택하여 작업을 확인합니다.

![경고 메시지를 표시하는 세션 종료 확인 대화 상자와 강조 표시된 세션 종료입니다.](../images/ui/session-management/end-session-confirmation-dialog.png)

세션이 종료되면 테이블에서 세션이 제거되고, 즉시 용량을 사용할 수 있게 되며, 작업을 감사용으로 기록합니다.

>[!NOTE]
>
>상태가 **[!UICONTROL Active]**&#x200B;인 세션을 종료할 수 없습니다. 이 안전 조치는 진행 중인 워크로드의 중단을 방지합니다.

## 종료 후 세션 동작 {#session-behavior-after-termination}

관리자가 세션을 종료하면 영향을 받는 사용자의 코드는 작업을 손실하지 않고 편집기에 남아 있습니다. 사용자가 종료 후 쿼리를 실행하려고 하면 시스템에서 종료된 세션을 감지하고 자동으로 연결을 재설정한 다음 쿼리 편집기 콘텐츠를 그대로 유지합니다.

이 비헤이비어는 사용자가 편집기에서 작성된 작업을 잃지 않도록 하며 새 세션이 설정되면 계속할 수 있습니다.

## 세션 관리에 대한 감사 로그 {#audit-logs}

시스템은 세션 관리 작업을 로그하여 가시성과 신뢰성을 제공합니다. 감사 로그는 세션 ID, 세션이 종료된 사용자, 작업을 수행한 관리자 및 작업 시간을 기록합니다.

감사 로그를 사용하여 세션 종료 기록을 검토하고 예기치 않은 연결 해제를 조사합니다.

감사 로그를 보는 방법에 대한 자세한 내용은 [쿼리 서비스 감사 로그 안내서](../data-governance/audit-log-guide.md)를 참조하십시오.
