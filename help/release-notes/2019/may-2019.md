## [!DNL Data Ingestion] {#ingestion}

Adobe Experience Platform은 모든 유형의 데이터와 지연 시간을 수집하는 풍부한 기능을 제공합니다. Adobe 내장 커넥터, 데이터 통합 파트너 또는 Adobe Experience Platform UI를 사용하여 일괄 처리 또는 스트리밍 API를 사용하여 수집할 수 있습니다.

제품 설명서의 경우 다음을 참조하십시오. [데이터 수집 개요](../../ingestion/home.md).

### 일괄 처리 수집

배치 수집 API를 사용하여 모든 유형의 데이터를 Adobe Experience Platform에 수집할 수 있습니다. 대량 수집 API는 XDM 또는 비XDM 스키마를 준수하는 데이터를 수집하기 위한 안전하고 확장 가능한 메커니즘을 제공합니다.

| 기능 | 설명 |
| -----------| ---------- |
| 다양한 형식 지원 | JSON, Parquet, CSV 수집 지원 |
| 데이터 유효성 검사 | 데이터에 대해 세 가지 수준의 유효성 검사가 수행됩니다. 스키마, 데이터 유형 및 제한 |
| 데이터 암호화 작동 중 | 데이터 네트워크를 통해 전송되는 동안 데이터를 보호하고 보호합니다. |
| 다중 부분 업로드 | 256MB 이상의 파일에 대해 지원되는 대용량 파일 업로드 |
| 자동 분할됨 | 데이터는 자동으로 분할되므로 확장성을 향상시키고 대규모 데이터 세트의 성능을 최적화할 수 있습니다. |

**알려진 문제**

None.

제품 설명서의 경우 다음을 참조하십시오. [배치 수집 개요](../../ingestion/batch-ingestion/overview.md).

| 기능 | 설명 |

| 클라우드 기반 Storage Connectors | 데이터 수집 위치 [!DNL Microsoft Azure Blob] 및 [!DNL Amazon S3] Adobe Experience Platform에 클라우드 스토리지 |
| [!DNL Microsoft Dynamics CRM] &amp; [!DNL Salesforce CRM] | 에서 사용할 수 있는 모든 데이터 검색 [!DNL Microsoft Dynamics] 또는 [!DNL Salesforce] CRM. 모든 개체, 표준 또는 사용자 지정 데이터 수집 [!DNL Microsoft Dynamics] 또는 [!DNL Salesforce] CRM. 증분 데이터를 시간별 또는 일별로 수집할 수 있습니다. |  |

## [!DNL Experience Data Model] (XDM) {#xdm}

표준화와 상호 운용성은 이면의 주요 개념입니다 [!DNL Experience Platform]. [!DNL Experience Data Model] Adobe 기반의 XDM(Customer Experience Management)은 고객 경험 데이터를 표준화하고 고객 경험 관리를 위한 스키마를 정의하려는 노력입니다.

XDM은 디지털 경험의 힘을 향상시키기 위해 설계된 문서화된 사양입니다. Adobe Experience Platform의 서비스와 통신할 수 있도록 모든 애플리케이션에 대한 공통 구조 및 정의를 제공합니다. XDM 표준을 준수함으로써 모든 고객 경험 데이터를 보다 빠르고 통합된 방식으로 통찰력을 제공하는 공통 표현에 통합할 수 있습니다. 고객 작업을 통해 유용한 통찰력을 얻을 수 있고, 세그먼트를 통해 고객 대상을 정의하고, 개인화를 위해 고객 속성을 사용할 수 있습니다.

XDM은 [!DNL Experience Clou]d. Adobe Experience Platform에서 제공하는 최적의 메시지를 적합한 사람에게 적절한 시점에 제공할 수 있습니다.

그 방법론 [!DNL Experience Platform] 빌드됨, [!DNL XDM System] 운영 [!DNL Experience Data Model] 스키마에서 사용 [!DNL Experience Platform] 구성 요소.

| 기능 | 설명 |
| -----------| ---------- |
| [!DNL Schema Registry] | 다음 [!DNL Schema Registry] 사용자 인터페이스([!DNL Schema Editor]) 및 RESTful API로, 모든 스키마 관련 리소스를 보고 관리할 수 있습니다. 다음 [!DNL Schema Editor] 및 [!DNL Schema Registry] API를 사용하여 각 IMS 조직에 고유한 새 스키마와 리소스를 만들고 관리할 수도 있습니다. |
| [!DNL Schema Library] | 다음 [!DNL Schema Library] 모두 포함 [!DNL Schema Registry] 개별 IMS 조직에서 정의한 리소스 및 Adobe을 통해 모든 사용자가 사용할 수 있도록 만들어진 리소스 등 [!DNL Experience Platform] 애플리케이션이 통합된 파트너 및 공급업체 [!DNL Experience Platform]. |

**알려진 문제**

없음.

를 사용하여 XDM을 사용하는 방법에 대해 자세히 알아보려면 [!DNL Schema Registry] API 및 [!DNL Schema Editor]을(를) 참조하십시오. [XDM 시스템 설명서](../../xdm/home.md).

## [!DNL Real-time Customer Profile] {#profile}

[!DNL Profile] 는 다양한 엔터프라이즈 데이터 자산의 데이터를 병합하고 실시간 소비자 프로필 및 관련 시계열 이벤트라고 하는 형식의 해당 데이터에 대한 액세스를 제공하는 범용 조회 엔티티 저장소입니다. 통합 보기는 여러 채널에서 작업을 알리고 권한을 제공하는 데 도움이 됩니다. [!DNL Platform], 및 Adobe 솔루션 통합. 또한, [!DNL Profile] 는 기계 학습 및 Sensei의 동력을 제공하는 데 사용됩니다. [!DNL Profile] API는 타사 솔루션, CRM 시스템 및 독점 솔루션의 기능을 보강하는 데 직접 사용할 수도 있습니다.

| 기능 | 설명 |
| -----------| ---------- |
| [!DNL Real-time customer profile] | [!DNL Profile] 데이터 inlet을 통해 데이터를 가져오고, 프로필 저장소의 기존 데이터와 병합하고, 거의 실시간으로 통합 보기를 만들거나 수정합니다. 고객의 경험에 따라 시간 내에 진화하는 이 데이터를 사용하여 개인화된 추천을 만들 수 있습니다. |
| 결합 보기 | 의 주요 기능 중 하나 [!DNL Profile] 은 다중 채널 데이터를 통합하는 기능입니다. [!DNL Profile] 는 소비자 베이스, 데이터 세트 간 및 연결된 ID에 있는 개인을 결합 보기라고 제공할 수 있습니다. |
| Edge | Edge Service는 모든 대응 가능 채널에서 지연 시간이 짧은 데이터 수집, 플러그인 가능한 컴퓨팅 및 신속한 데이터 활성화를 위한 프레임워크 역할을 합니다. Edge Service를 통해 실시간 개인화를 간편하게 수행할 수 있습니다. |

## ID 서비스 {#identity}

Identity Service는 조직의 생태계에서 재생되는 각 서로 다른 시스템에서 데이터의 단편적인 특성으로 인해 고객의 단절된 ID로 인해 야기되는 근본적인 문제를 해결합니다. 이는 여러 장치 및 소비자가 브랜드와 함께 참여하는 데 사용하는 다양한 시스템에서 ID를 브리징하여 수행합니다. 이러한 방식으로 ID 서비스를 통해 고객 및 고객 행동을 완벽하게 파악할 수 있으므로 효과적인 디지털 경험을 실시간으로 제공할 수 있습니다.

| 기능 | 설명 |
| -----------| ---------- |
| ID 그래프 | ID 그래프는 고객 활동과 거의 실시간으로 업데이트되는 ID 간의 관계 맵입니다. [!DNL Identity Service] 조직만 볼 수 있고 비공개 그래프라고 하는 데이터를 기반으로 구축된 id 그래프를 관리합니다. [!DNL Identity Service] 수집된 데이터 레코드에 둘 이상의 ID가 포함되어 이러한 ID 간에 관계를 추가하는 경우 개인 그래프를 늘립니다. |

**알려진 문제**

없음.

제품 설명서의 경우 [ID 서비스 개요](../../identity-service/home.md).

## [!DNL Segmentation Service] {#segmentation}

[!DNL Segmentation Service] 프로필 저장소 내의 마케팅 가능한 사람 그룹을 구분하는 기준을 설명하는 프로필 저장소의 특정 프로필 하위 집합을 정의합니다. 세그먼트는 브랜드에서 고객 터치 포인트를 나타내는 레코드 데이터(예: 인구 통계 정보) 또는 시계열 이벤트를 기반으로 할 수 있습니다.

예를 들어 &quot;운동화 구입을 잊으셨습니까?&quot;라는 이메일 캠페인에서 지난 30일 이내에 운동화를 검색했지만 구매를 완료하지 않은 모든 사용자의 대상 세그먼트를 원할 수 있습니다. 또는 보상 프로그램에서 특정 등급을 가진 사용자만 표시하도록 웹 콘텐츠를 타깃팅할 수 있습니다.

| 기능 | 설명 |
| -----------| ---------- |
| 세그멘테이션 API | RESTful 서비스를 통해 고객 세그먼트를 빌드하기 위해 프로그래밍 방식으로 액세스할 수 있습니다. |
| 세그먼트 빌더 | 세그먼트 빌더는 고객이 [!DNL Profile] 데이터. |

## 데이터 거버넌스 {#governance}

Adobe Experience Platform을 사용하면 여러 엔터프라이즈 시스템의 데이터를 함께 가져와서 마케터가 고객을 식별, 이해 및 참여시킬 수 있습니다. [!DNL Experience Platform] 에는 내에서 데이터를 적절하게 사용할 수 있도록 하는 완벽한 데이터 거버넌스 인프라가 포함되어 있습니다 [!DNL Platform] 시스템 간에 공유할 때 사용할 수 있습니다.

데이터 거버넌스는 고객 데이터를 관리하고 데이터 사용량에 적용되는 규정, 제한 및 정책을 준수하는 데 사용되는 일련의 전략과 기술입니다. 이 구성 요소는 [!DNL Experience Platform] 카탈로그 작성, 데이터 계보, 데이터 사용 레이블 지정, 데이터 액세스 정책, 마케팅 작업을 위한 데이터 액세스 제어 등 다양한 수준에서 사용할 수 있습니다.

데이터 거버넌스를 시작하려면 고객 데이터에 적용되는 규정, 계약 의무 및 기업 정책을 완벽하게 이해해야 합니다. 여기서, 데이터는 적절한 데이터 사용 레이블을 적용하여 분류할 수 있으며, 데이터 사용 정책의 정의를 통해 데이터 사용을 제어할 수 있습니다.

Data Governance 프레임워크는 [!DNL Experience Platform] 사용자 인터페이스 및 [!DNL Policy Service] API.

| 기능 | 설명 |
| -----------| ---------- |
| 레이블 [!DNL Experience Platform] 개체 | 사전 정의된 분류 레이블을 기반으로 연결, 데이터 세트 및 데이터 세트 필드 수준에서 데이터 사용 레이블을 적용하고 관리합니다. |
| 데이터 사용 정책 만들기 | 레이블 및 요청된 마케팅 작업이 있는지에 따라 데이터 사용 정책을 정의하고 관리합니다. |
| 정책 위반 확인 | 에서 마케팅 작업을 수행할 때 정의된 사용 정책을 위반하는지 평가 [!DNL Platform] 데이터. |
