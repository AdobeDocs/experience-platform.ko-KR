## [!DNL Data Ingestion] {#ingestion}

Adobe Experience Platform은 모든 유형의 데이터 및 지연을 인제스트하는 다양한 기능을 제공합니다. Adobe 내장 커넥터, 데이터 통합 파트너 또는 Adobe Experience Platform UI를 사용하여 일괄 처리 또는 스트리밍 API를 사용하여 인제스트할 수 있습니다.

제품 설명서의 경우 [데이터 수집 개요를 참조하십시오](../../ingestion/home.md).

### 일괄 처리

일괄 처리 통합 API를 사용하여 모든 유형의 데이터를 Adobe Experience Platform으로 인제스트합니다. 벌크 통합 API는 XDM 또는 비-XDM 스키마를 준수하는 데이터를 인제스트하는 안전하고 확장 가능한 메커니즘을 제공합니다.

| 기능 | 설명 |
| -----------| ---------- |
| 다양한 포맷 지원 | JSON, Parentheet, CSV 통합 지원 |
| 데이터 유효성 검사 | 데이터에 대해 세 가지 수준의 유효성 검사가 수행됩니다.스키마, 데이터 유형 및 제한 |
| 데이터 암호화 활용 | 데이터 네트워크를 통해 전송되는 동안 데이터를 보호하고 보호합니다. |
| 다중 부분 업로드 | 256MB 이상의 파일에 대해 대용량 파일 업로드 지원 |
| 자동 분할 | 데이터를 자동으로 분할하여 대용량 데이터 세트의 확장성을 향상시키고 성능을 최적화합니다. |

**알려진 문제**

None.

제품 설명서의 경우 [일괄 처리 통합 개요를 참조하십시오](../../ingestion/batch-ingestion/overview.md).

### 커넥터

Adobe Experience Platform 커넥터를 사용하면 여러 소스에서 데이터를 손쉽게 인제스트할 수 있으므로 [!DNL Experience Platform] 서비스를 사용하여 데이터를 구조화하고 레이블을 지정하고 향상시킬 수 있습니다. [!DNL Platform] connectors를 사용하면 클라우드 기반 스토리지 시스템 및 CRM 서비스에 인증하고, 통합 실행에 대한 시간을 설정하고, 데이터 통합 처리량을 관리할 수 있습니다.

| 기능 | 설명 |
| -----------| ---------- |
| 클라우드 기반 스토리지 커넥터 | Adobe Experience Platform으로 클라우드 [!DNL Microsoft Azure Blob] 및 [!DNL Amazon S3] 클라우드 스토리지 데이터 인제스트 |
| [!DNL Microsoft Dynamics CRM] &amp; [!DNL Salesforce CRM] | CRM [!DNL Microsoft Dynamics] 또는 CRM에서 사용할 수 있는 모든 데이터를 [!DNL Salesforce] 검색합니다. CRM 또는 CRM에서 모든 객체, 표준 또는 사용자 정의 [!DNL Microsoft Dynamics] 를 [!DNL Salesforce] 인제스트합니다. 증분 데이터를 시간 또는 매일 수집할 수 있습니다. |  |

**알려진 문제**

* 클라우드 기반 스토리지 커넥터는 API 호출을 통해서만 사용할 수 있습니다.

| 데이터를 실시간으로 전송 [!DNL Launch] | 의 Adobe Experience Platform 익스텐션을 사용하여 클라이언트 디바이스에서 XDM 형식 데이터 [!DNL Launch] 를 실시간으로 손쉽게 전송할 수 있습니다. |

## [!DNL Experience Data Model] (XDM) {#xdm}

표준화와 상호 운용성은 그 이면의 핵심 개념입니다 [!DNL Experience Platform]. [!DNL Experience Data Model] (XDM)은 Adobe을 기반으로 고객 경험 데이터를 표준화하고 고객 경험 관리를 위한 스키마를 정의하는 것입니다.

XDM은 디지털 경험의 강력함을 향상시키도록 고안된 공개적으로 문서화된 사양입니다. 이 소프트웨어는 Adobe Experience Platform의 서비스와 통신하기 위해 모든 애플리케이션에 대한 공통 구조와 정의를 제공합니다. 모든 고객 경험 데이터는 XDM 표준을 준수하여 보다 빠르고 통합된 방식으로 인사이트를 제공하는 공통 표현에 통합할 수 있습니다. 고객 행동을 통해 유용한 인사이트를 얻고 세그먼트를 통해 고객 고객을 정의하며 개인화를 위해 고객 속성을 사용할 수 있습니다.

XDM은 [!DNL Experience Clou]d를 Adobe Experience Platform이 제공한 메커니즘으로 정확한 타이밍에 올바른 고객에게 올바른 메시지를 전달할 수 있습니다.

구성 요소 [!DNL Experience Platform] 에서 사용할 수 있도록 스키마 [!DNL XDM System] 를 [!DNL Experience Data Model] 빌드하는 방법론 [!DNL Experience Platform] .

| 기능 | 설명 |
| -----------| ---------- |
| [!DNL Schema Registry] | 이 [!DNL Schema Registry] 는 모든 스키마 관련 리소스를 보고 관리할 수 있는 사용자 인터페이스([!DNL Schema Editor]) 및 RESTful API를 제공합니다. 또한 [!DNL Schema Editor] 및 [!DNL Schema Registry] API를 사용하여 각 IMS 조직에 고유한 새 스키마 및 리소스를 만들고 관리할 수 있습니다. |
| [!DNL Schema Library] | 여기에는 개별 IMS 조직에서 정의한 리소스, 애플리케이션이 통합된 Adobe, 파트너 및 공급업체가 모든 사용자가 사용할 수 있도록 만들어진 리소스 등 모든 [!DNL Schema Library] 리소스가 [!DNL Schema Registry] 포함되어 있습니다 [!DNL Experience Platform] [!DNL Experience Platform]. |

**알려진 문제**

None.

API를 사용한 XDM 작업에 대한 자세한 내용 [!DNL Schema Registry] 은 [!DNL Schema Editor]XDM 시스템 설명서를 참조하십시오 [](../../xdm/home.md).

## [!DNL Real-time Customer Profile] {#profile}

[!DNL Profile] 은 다양한 엔터프라이즈 데이터 자산 간의 데이터를 통합하고 실시간 소비자 프로필 및 관련 시계열 이벤트의 형태로 해당 데이터에 대한 액세스를 제공하는 범용 조회 엔티티 스토어입니다. 통합 뷰를 통해 다양한 채널, Adobe 솔루션 통합 [!DNL Platform]에 걸쳐 동작을 효과적으로 전달하고 제어할 수 있습니다. 또한 머신 러닝 [!DNL Profile] 과 Sensei를 강화하는 데 사용됩니다. [!DNL Profile] 또한 API를 사용하여 타사 솔루션, CRM 시스템 및 독점 솔루션의 기능을 강화할 수 있습니다.

| 기능 | 설명 |
| -----------| ---------- |
| [!DNL Real-time customer profile] | [!DNL Profile] 데이터 인렛을 통해 데이터를 인제스트하고 프로필 저장소의 기존 데이터와 병합하며 거의 실시간으로 통합 보기를 만들거나 수정합니다. 고객 경험에 따라 시간 경과에 따라 진화하는 이 데이터를 사용하여 개인화된 추천을 만들 수 있습니다. |
| 조합 보기 | 멀티 채널 데이터 통합 [!DNL Profile] 의 주요 기능 중 하나는 [!DNL Profile] 조합 보기라고 하는 소비자 기반, 데이터 세트 간 및 연결된 ID에 대한 개별 보기를 병합하여 제공할 수 있습니다. |
| Edge | Edge Service는 모든 대응 가능 채널에서 낮은 지연 시간 데이터 수집, 플러그 가능 컴퓨팅 및 신속한 데이터 활성화를 위한 프레임워크로 사용됩니다. Edge Service는 개인화를 용이하게 합니다. |

**알려진 문제**

None.

제품 설명서의 경우 프로필 [개요를 참조하십시오](../../profile/home.md).

## ID 서비스 {#identity}

ID 서비스는 조직의 에코시스템에서 각 서로 다른 시스템에서 분산된 데이터의 특성을 초래하여 단절된 ID로 인해 발생하는 기본적인 문제를 해결합니다. 이는 소비자가 브랜드 참여를 유도하기 위해 사용하는 다양한 시스템과 다양한 디바이스에 걸쳐 ID를 결합함으로써 이루어집니다. 이러한 방식으로 ID 서비스는 고객 및 고객의 행동을 완벽하게 파악하여 효과적인 디지털 경험을 실시간으로 제공할 수 있도록 합니다.

| 기능 | 설명 |
| -----------| ---------- |
| ID 그래프 | ID 그래프는 고객 활동과 거의 실시간으로 업데이트되는 ID 간의 관계 맵입니다. [!DNL Identity Service] 조직만이 볼 수 있고 데이터를 기반으로 개인 그래프라고 하는 ID 그래프를 관리합니다. [!DNL Identity Service] 인제스트된 데이터 레코드에 둘 이상의 ID가 포함된 경우 개인 그래프를 늘려 이러한 ID 간의 관계를 추가합니다. |

**알려진 문제**

None.

제품 설명서는 ID 서비스 [개요를 참조하십시오](../../identity-service/home.md).

## [!DNL Segmentation Service] {#segmentation}

[!DNL Segmentation Service] 프로필 저장소 내의 마케팅 가능한 사람들을 구분하는 기준을 설명하면서 프로필 스토어와 특정 프로필 하위 집합을 정의합니다. 세그먼트는 브랜드 고객 접점을 나타내는 레코드 데이터(예: 인구 통계 정보) 또는 시간 시리즈 이벤트를 기반으로 할 수 있습니다.

예를 들어 &quot;운동화 구입을 잊으셨습니까?&quot;라는 이메일 캠페인에서 지난 30일 이내에 운동화를 검색했지만 구매를 완료하지 않은 모든 사용자의 고객 세그먼트를 원할 수 있습니다. 또는 보상 프로그램에서 특정 등급을 받은 사용자만 표시하도록 웹 컨텐츠를 타깃팅할 수 있습니다.

| 기능 | 설명 |
| -----------| ---------- |
| 세그멘테이션 API | RESTful 서비스를 사용하면 프로그래머틱 방식으로 고객 세그먼트를 구축할 수 있습니다. |
| 세그먼트 빌더 | 세그먼트 빌더는 [!DNL Profile] 데이터에서 고객 세그먼트를 구축하기 위한 강력하고 직관적인 작업 공간입니다. |

**알려진 문제**

None.

## [!DNL Data Governance] {#governance}

Adobe Experience Platform을 사용하면 여러 엔터프라이즈 시스템의 데이터를 통합하여 마케터가 고객을 식별하고 이해하고 고객의 참여를 유도할 수 있습니다. [!DNL Experience Platform] 시스템 간에 데이터를 공유할 때 적절한 데이터 사용 [!DNL Platform] 을 보장하는 엔드 투 엔드 데이터 관리 인프라를 포함합니다.

데이터 거버넌스는 고객 데이터를 관리하고 데이터 사용에 적용되는 규정, 제한 사항 및 정책을 준수하는 데 사용되는 일련의 전략과 기술입니다. 카탈로그 작성, 데이터 계보, 데이터 사용 표시, 데이터 액세스 정책, 마케팅 작업을 위한 데이터 액세스 제어 등 다양한 [!DNL Experience Platform] 수준에서 핵심적인 역할을 합니다.

데이터 거버넌스를 시작하려면 고객 데이터에 적용되는 규정, 계약 의무 및 기업 정책을 철저하게 이해해야 합니다. 여기에서, 데이터는 적절한 데이터 사용 레이블을 적용하여 분류할 수 있으며, 데이터 사용 정책의 정의를 통해 데이터를 사용할 수 있습니다.

이 [!DNL Data Governance] 프레임워크는 [!DNL Experience Platform] 사용자 인터페이스 및 [!DNL Policy Service] API를 통해 데이터를 분류하고 데이터 사용 정책을 만드는 과정을 간소화하고 간소화합니다.

| 기능 | 설명 |
| -----------| ---------- |
| 레이블 [!DNL Experience Platform] 엔티티 | 사전 정의된 분류 레이블을 기반으로 연결, 데이터 세트 및 데이터 세트 필드 수준에서 데이터 사용 레이블을 적용하고 관리할 수 있습니다. |
| 데이터 사용 정책 만들기 | 레이블 및 요청된 마케팅 작업을 기반으로 데이터 사용 정책을 정의하고 관리할 수 있습니다. |
| 정책 위반 확인 | 데이터에 대한 마케팅 작업을 수행할 때 정의된 사용 정책이 위반되는지 여부를 [!DNL Platform] 평가합니다. |

**알려진 문제**

* None.
