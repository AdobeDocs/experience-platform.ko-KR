## 데이터 통합 {#ingestion}

Adobe Experience Platform은 모든 유형의 데이터와 지연 시간을 인제스트할 수 있는 다양한 기능을 제공합니다. Adobe 내장 커넥터, 데이터 통합 파트너 또는 Adobe Experience Platform UI를 사용하여 일괄 처리 또는 스트리밍 API를 사용하여 인제스트할 수 있습니다.

제품 설명서는 데이터 통합 [개요를](../../ingestion/home.md)참조하십시오.

### 일괄 처리

일괄 처리 통합 API를 사용하여 모든 유형의 데이터를 Adobe Experience Platform으로 인제스트할 수 있습니다. 일괄 처리 API는 XDM 또는 XDM 이외의 스키마를 따르는 데이터를 인제스트하는 안전하고 확장 가능한 메커니즘을 제공합니다.

| 기능 | 설명 |
| -----------| ---------- |
| 다양한 포맷 지원 | JSON, 쪽모이 세공, CSV 통합 지원 |
| 데이터 유효성 검사 | 데이터에 대해 세 가지 수준의 유효성 검사가 수행됩니다.스키마, 데이터 유형 및 제한 |
| 데이터 암호화 동작 | 데이터 네트워크를 통해 데이터를 전송하는 동안 데이터를 보호하고 보호합니다. |
| 다중 부분 업로드 | 256MB 이상의 파일에 대한 대용량 파일 업로드 지원 |
| 자동 분할됨 | 데이터를 자동으로 분할하여 확장성을 향상시키고 대용량 데이터 세트의 성능을 최적화할 수 있습니다. |

**알려진 문제**

제품 설명서의 경우 [일괄 처리 통합 개요를](../../ingestion/batch-ingestion/overview.md)참조하십시오.

### 커넥터

Adobe Experience Platform Connectors를 사용하면 여러 소스에서 데이터를 손쉽게 인제스트할 수 있으므로 Experience Platform 서비스를 사용하여 데이터를 구조화, 레이블 지정 및 향상시킬 수 있습니다. 플랫폼 커넥터를 사용하면 클라우드 기반 스토리지 시스템과 CRM 서비스를 인증하고, 통합 실행 시간을 설정하고, 데이터 통합 처리량을 관리할 수 있습니다.

| 기능 | 설명 |
| -----------| ---------- |
| 클라우드 기반 스토리지 커넥터 | Microsoft Azure Blob 및 Amazon S3 클라우드 저장소의 데이터를 Adobe Experience Platform으로 인제스트 |
| Microsoft Dynamics CRM 및 Salesforce CRM | Microsoft Dynamics 또는 Salesforce CRM 파섹 Microsoft Dynamics 또는 Salesforce CRM 파섹 증분 데이터는 시간별 또는 일별로 인제스트할 수 있습니다. |  |

### 스트리밍 통합

스트리밍 인제스트는 클라이언트 및 서버측 디바이스에서 실시간으로 Experience Platform으로 데이터를 전송합니다. 인증된 데이터 수집 및 다중 레코드 페이로드에 대한 지원을 통해 1초 이내에 실시간 고객 프로필을 업데이트할 수 있습니다.

| 기능 | 설명 |
| -----------| ---------- |
| 동기 유효성 검사 | JSON 페이로드에서 오류를 즉시 검색할 수 있으므로 개발자 속도가 향상됩니다. |
| 엔드 투 엔드 모니터링 | 데이터 레이크와 실시간 고객 프로필에서 인제스트에서 소비에 이르는 스트리밍 데이터를 모니터링합니다. |
| 인증된 데이터 수집 | 클라이언트의 인증 토큰을 요구하도록 옵션을 켜거나 끕니다. Adobe Experience Platform 서비스를 사용하면 신뢰할 수 있는 출처의 기록과 신뢰할 수 없는 출처의 레코드를 구별할 수 있습니다. |
| Launch를 통해 실시간으로 데이터 전송 | Launch의 Adobe Experience Platform 익스텐션을 사용하여 클라이언트 디바이스에서 실시간으로 XDM 형식 데이터를 손쉽게 전송할 수 있습니다. |

### ETL

| ETL 통합 | Experience Platform은 데이터를 추출, 변환 및 플랫폼으로 로드하기 위한 보안 커넥터를 제작하려는 사용자를 위한 ETL 통합 설명서를 제공합니다. |

## XDM(Experience Data Model) {#xdm}

Adobe Experience Platform의 주요 개념은 표준화 및 상호 운용성 Adobe 기반의 XDM(Experience Data Model)은 고객 경험 데이터를 표준화하고 고객 경험 관리를 위한 스키마를 정의하려는 노력의 일환입니다.

XDM은 디지털 경험의 성능을 개선하기 위해 고안된 문서화된 사양입니다. Adobe Experience Platform에서 제공하는 서비스와 커뮤니케이션할 수 있는 모든 애플리케이션에 대한 공통 구조와 정의를 제공합니다. XDM 표준을 준수하여 모든 고객 경험 데이터를 보다 신속하고 통합된 방식으로 인사이트를 제공하는 일반적인 표현으로 통합할 수 있습니다. 고객 행동을 통해 유용한 인사이트를 얻고 세그먼트를 통해 고객 고객을 정의하며 개인화를 위해 고객 속성을 사용할 수 있습니다.

XDM은 Adobe Experience Platform을 기반으로 하는 Experience Cloud를 통해 정확한 타이밍에 적합한 고객에게 올바른 메시지를 전달할 수 있는 메커니즘입니다.

XDM 시스템은 경험 플랫폼을 구축하는 방법론을 통해 경험 플랫폼 구성 요소에서 사용하기 위해 경험 데이터 모델 스키마를 관리합니다.

| 기능 | 설명 |
| -----------| ---------- |
| 스키마 레지스트리 | 스키마 레지스트리는 모든 스키마 관련 리소스를 보고 관리할 수 있는 사용자 인터페이스(스키마 편집기) 및 RESTful API를 제공합니다. 스키마 편집기 및 스키마 레지스트리 API를 사용하여 각 IMS 조직에 고유한 새 스키마 및 리소스를 만들고 관리할 수도 있습니다. |
| 스키마 라이브러리 | 스키마 라이브러리에는 개별 IMS 조직에 의해 정의된 리소스, Adobe, Experience Platform 파트너 및 응용 프로그램이 Experience Platform과 통합된 제공업체가 모든 사용자에게 제공하는 리소스 등 모든 스키마 레지스트리 리소스가 포함되어 있습니다. |

**알려진 문제**

없음.

스키마 레지스트리 API 및 스키마 편집기를 사용하여 XDM을 사용하는 방법에 대한 자세한 내용은 XDM [시스템 설명서를](../../xdm/home.md)참조하십시오.

## 실시간 고객 프로필 {#profile}

프로파일은 다양한 엔터프라이즈 데이터 에셋 간의 데이터를 통합하고 실시간 소비자 프로필 및 관련 시계열 이벤트 형태로 해당 데이터에 대한 액세스를 제공하는 범용 조회 엔티티 스토어입니다. 통합된 뷰를 통해 채널, 플랫폼 및 Adobe 솔루션 통합 전반에 걸쳐 동작을 전달하고 역량을 강화할 수 있습니다. 또한 Profile은 머신 러닝 및 Sensei를 강화하는 데 사용됩니다. 또한 프로필 API를 사용하여 타사 솔루션, CRM 시스템 및 독점 솔루션의 기능을 강화할 수 있습니다.

| 기능 | 설명 |
| -----------| ---------- |
| 실시간 고객 프로필 | 데이터 인제트를 통해 데이터를 프로파일링하고 프로파일 저장소의 기존 데이터와 병합하며 거의 실시간으로 통합 뷰를 만들거나 수정합니다. 고객 경험에 따라 시간 경과에 따라 진화하는 이 데이터를 사용하여 개인화된 추천을 만들 수 있습니다. |
| 조합 보기 | 프로필의 주요 기능 중 하나는 멀티채널 데이터를 통합하는 기능입니다. 프로파일은 소비자 기반, 데이터 세트 간 및 연결된 ID에 대한 병합된 뷰를 조합 보기라고 할 수 있습니다. |
| Edge | Edge Service는 주소 지정이 가능한 모든 채널에서 데이터 수집, 플러그 방식의 컴퓨팅 및 신속한 데이터 활성화를 위한 프레임워크로 사용됩니다. Edge Service를 통해 개인화된 실시간 경험을 제공할 수 있습니다. |

**알려진 문제**

없음.

제품 설명서의 경우 프로필 [개요를 참조하십시오](../../profile/home.md).

## ID 서비스 {#identity}

Identity Service는 단절된 고객 ID로 인해 발생하는 기본적인 문제를 해결합니다. 이러한 ID는 조직의 에코시스템에서 작동하는 서로 다른 각 시스템에서 분산된 데이터의 특성을 초래했습니다. 이를 위해서는 다양한 디바이스와 소비자가 브랜드 참여를 유도하기 위해 사용하는 다양한 시스템에 ID를 결합해야 합니다. 이러한 방식으로 Identity Service는 고객 및 고객 행동에 대한 완전한 이해를 용이하게 해주므로 효과적인 디지털 경험을 실시간으로 제공할 수 있습니다.

| 기능 | 설명 |
| -----------| ---------- |
| ID 그래프 | ID 그래프는 고객 활동과 거의 실시간으로 업데이트되는 ID 간의 관계 맵입니다. Identity Service는 조직만 볼 수 있고 데이터를 기반으로 하여 개인 그래프라고 하는 ID 그래프를 관리합니다. Identity Service는 인제스트된 데이터 레코드에 둘 이상의 ID가 포함되어 있을 때 개인 그래프를 늘립니다. 이러한 ID 간의 관계를 추가합니다. |

**알려진 문제**

없음.

## 세그멘테이션 서비스 {#segmentation}

세그멘테이션 서비스는 프로필 스토어의 특정 프로필 하위 집합을 정의하며 프로필 스토어 내의 마케팅 가능한 사용자 그룹을 구분하는 기준을 설명합니다. 세그먼트는 레코드 데이터(예: 인구 통계 정보) 또는 브랜드에 고객 접점을 나타내는 시계열 이벤트를 기반으로 할 수 있습니다.

예를 들어 &quot;운동화 구입을 잊으셨습니까?&quot;라는 이메일 캠페인에서 지난 30일 이내에 운동화를 검색했지만 구매를 완료하지 않은 모든 사용자의 고객 세그먼트를 원할 수 있습니다. 또는 보상 프로그램에서 특정 등급을 가진 사용자만 표시할 웹 컨텐츠를 타깃팅할 수 있습니다.

| 기능 | 설명 |
| -----------| ---------- |
| 세그멘테이션 API | RESTful 서비스를 통해 프로그래머틱 방식으로 고객 세그먼트를 구축할 수 있습니다. |
| 세그먼트 빌더 | 세그먼트 빌더는 프로필 데이터에서 고객 세그먼트를 작성하기 위한 강력하고 직관적인 작업 공간입니다. |

## 데이터 거버넌스 {#governance}

기업은 Adobe Experience Platform을 사용하여 여러 엔터프라이즈 시스템의 데이터를 통합하여 마케터가 고객을 식별하고 이해하고 고객의 참여를 유도할 수 있습니다. 경험 플랫폼은 데이터 사용 레이블 지정 및 실행(DULE)을 비롯한 엔드 투 엔드 데이터 거버넌스 인프라를 포함하여 플랫폼 내에서 그리고 시스템 간에 데이터를 공유할 때 데이터를 적절하게 사용합니다.

데이터 거버넌스는 고객 데이터를 관리하고 데이터 사용에 적용되는 규정, 제한 및 정책을 준수하는 데 사용되는 일련의 전략과 기술입니다. Adobe Experience Platform은 카탈로그 작성, 데이터 계보, 데이터 사용 레이블 지정, 데이터 액세스 정책, 마케팅 활동을 위한 데이터 액세스 제어 등 다양한 수준에서 핵심적인 역할을 합니다.

데이터 거버넌스를 시작하려면 고객 데이터에 적용되는 규정, 계약 의무 및 기업 정책을 철저하게 이해해야 합니다. 여기에서 데이터는 적절한 데이터 사용 레이블을 적용하여 분류할 수 있으며 데이터 사용 정책의 정의를 통해 데이터를 사용할 수 있습니다.

DULE 프레임워크는 Experience Platform 사용자 인터페이스 및 DULE Policy Service API를 통해 데이터를 분류하고 데이터 사용 정책을 만드는 과정을 간소화하고 간소화합니다.

| 기능 | 설명 |
| -----------| ---------- |
| 경험 플랫폼 엔티티에 레이블 지정 | 사전 정의된 분류 레이블을 기반으로 연결, 데이터 세트 및 데이터 세트 필드 수준에서 데이터 사용 레이블을 적용하고 관리할 수 있습니다. |
| 데이터 사용 정책 만들기 | 레이블 및 요청된 마케팅 작업을 기반으로 데이터 사용 정책을 정의하고 관리합니다. |
| 정책 위반 확인 | 플랫폼 데이터에 대한 마케팅 작업을 수행할 때 정의된 사용 정책이 위반되는지 여부를 평가합니다. |
