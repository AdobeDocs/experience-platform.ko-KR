## 사용자 인터페이스 업데이트 {#ux}

Adobe Experience Platform은 사용자 경험을 향상시키고 다른 [!DNL Experience Cloud] 응용 프로그램과 통합하기 위해 도메인 및 헤더 막대에 대한 업데이트를 발표했습니다.

- 조직 간 또는 다른 애플리케이션으로 쉽게 전환할 수 있습니다
- 도움말 메뉴의 주요 문서 및 컨텍스트 관련 설명서를 비롯한 사용자 도움말이 개선되었습니다
- [!DNL Experience Platform] 및 파일 지원 티켓에 대한 피드백을 제공하는 기능

새로운 경험에서 점차 벗어나고 있다. [https://experience.adobe.com/platform](https://experience.adobe.com/platform)에서 경험을 볼 수 있습니다.

## [!DNL Data Science Workspace] {#dsw}

[!DNL Data Science Workspace] 머신 러닝과 인공 지능을 사용하여 데이터에서 인사이트를 도출합니다. Adobe Experience Platform에 통합된 [!DNL Data Science Workspace]은(는) Adobe 솔루션에서 컨텐츠와 데이터 자산을 사용하여 예측하는 데 도움이 됩니다. [!DNL Data Science Workspace]이(가) 수행하는 방법 중 하나는 [!DNL JupyterLab]을 사용하는 것입니다. [!DNL JupyterLab] 는 Adobe Experience Platform에 긴밀하게 통합되어  [[!DNL Project Jupyter]](https://jupyter.org/) 있고 웹 기반의 유저 인터페이스입니다. 데이터 과학자들이 [!DNL Jupyter] 노트북, 코드 및 데이터를 사용하여 작업할 수 있는 대화형 개발 환경을 제공합니다.

**새로운 기능**

| 기능 | 설명 |
|--- | ---|
| [!DNL JupyterLab Launcher] | 이제 [!DNL JupyterLab Launcher]에 [!DNL Spark] 2.4 노트북의 시작자가 포함됩니다. [!DNL Spark] 2.3 노트북 시작자는 이제 더 이상 사용되지 않는 것으로 표시되며 이후 릴리스에서 제거되도록 설정됩니다. |
| [!DNL Spark] 2.4 | 이제 New Scala([!DNL Spark]) 및 PySpark 레서피에서 [!DNL Spark] 2.4를 사용합니다. |
| 커널 | 이제 스칼라([!DNL Spark]) 전자 필기장이 Scala 커널을 통해 작성되었습니다. 이제 PySpark 노트북이 [!DNL Python] 커널을 통해 제작됩니다. [!DNL Spark] 및 PySpark 커널은 더 이상 사용되지 않으며 이후 릴리스에서 제거되도록 설정됩니다. |
| 레서피 | 이제 새 PySpark 및 [!DNL Spark] 레서피가 [!DNL Python] 및 R 레서피와 유사한 [!DNL Docker] 워크플로우를 따릅니다. |

전자 필기장 및 레서피가 [!DNL Spark] 2.4를 사용하도록 마이그레이션하는 방법에 대한 자세한 내용은 [전자 필기장 데이터 액세스 안내서](../../data-science-workspace/jupyterlab/access-notebook-data.md)를 참조하십시오. 데이터 과학 작업 공간에 대한 자세한 일반 정보는 [개요 설명서](../../data-science-workspace/home.md)를 참조하십시오.

## [!DNL Destinations] {#destinations}

[실시간 고객 데이터 플랫폼](../../rtcdp/overview.md)에서 대상은 대상 플랫폼과 사전 구축된 통합으로, 데이터를 해당 파트너에게 원활하게 활성화할 수 있습니다.

**[!DNL Facebook]**

실시간 CDP는 이제 [!DNL Facebook]에 대한 데이터 활성화를 지원하므로, 해시 이메일을 기반으로 고객 타깃팅, 개인화 및 억제를 위해 [!DNL Facebook] 캠페인에 대한 프로파일을 활성화할 수 있습니다.

새 기능에 대한 자세한 내용은 [Facebook 대상](../../destinations/catalog/social/facebook.md) 페이지를 참조하십시오.

**[!DNL Amazon Kinesis]및  [!DNL Azure Event Hubs] 스트리밍 클라우드 스토리지 대상**

실시간 CDP는 스트리밍 클라우드 스토리지 대상에 대한 데이터 활성화를 지원하므로 JSON 포맷으로 대상 데이터와 이벤트를 이러한 대상으로 내보낼 수 있습니다. 그런 다음 대상에 있는 이러한 이벤트 위에 비즈니스 논리를 설명할 수 있습니다. 자세한 내용은 아래를 참조하십시오.

>[!NOTE]
>
>실시간 CDP의 [!DNL Amazon Kinesis] 및 [!DNL Azure Event Hubs] 대상은 현재 베타 버전입니다. 설명서 및 기능은 변경될 수 있습니다.

| 설명서 | 설명 |
|--- | ---|
| [(베타) Amazon Kinesis 대상](../../destinations/catalog/cloud-storage/amazon-kinesis.md) | 이 문서에서는 Adobe Experience Platform의 데이터를 스트리밍하기 위해 [!DNL Amazon Kinesis] 스토리지에 대한 실시간 아웃바운드 연결을 만드는 방법에 대해 설명합니다. |
| [(베타) Azure 이벤트 허브 대상](../../destinations/catalog/cloud-storage/azure-event-hubs.md) | 이 문서에서는 Adobe Experience Platform의 데이터를 스트리밍하기 위해 [!DNL Azure Event Hubs] 스토리지에 대한 실시간 아웃바운드 연결을 만드는 방법에 대해 설명합니다. |
| [API 자습서 - 스트리밍 대상에 연결 및 데이터 활성화](../../destinations/api/streaming-destinations.md) | 이 자습서에서는 API 호출을 사용하여 Adobe Experience Platform 데이터에 연결하고, 스트리밍 클라우드 저장소 대상([!DNL Amazon Kinesis] 또는 [!DNL Azure Event Hubs])에 연결을 만들고, 데이터 흐름을 새로 만든 대상에 만들고, 데이터를 새로 만든 대상에 활성화하는 방법을 보여 줍니다. |

자세한 내용은 [대상 개요](../../destinations/home.md)를 참조하십시오.

## [!DNL Experience Platform Web SDK] 및 [!DNL Experience Platform Edge Network] {#edge}

[!DNL Experience Platform Web SDK] 및 [!DNL Experience Platform Edge Network]을 사용하면 최종 사용자 디바이스 및 브라우저에서 실시간으로 Adobe Experience Platform 및 기타 Adobe 솔루션에 데이터를 보낼 수 있습니다. 가장 최근의 사용 사례 목록은 자주 업데이트되는 [공개 로드맵](https://github.com/adobe/alloy/projects/5)에서 확인할 수 있습니다.

**새로운 기능**

| 기능 | 설명 |
|--- | ---|
| ECID 지원 | SDK는 설치할 추가 라이브러리 또는 정보 없이 즉시 사용 가능한 ECID를 지원합니다. |
| 구성 UI | [!DNL Launch]의 새 에지 구성 UI를 사용하여 구성 ID 설정을 관리합니다. 액세스하려면 허용 목록에 있어야 합니다. |
| Adobe Experience Platform Web SDK Mixin | 지원되는 모든 필드를 포함하는 [!DNL Experience Platform web SDK]에 사용할 믹스인입니다. |
| 강좌 동의 제어 | 회사에 [!DNL Experience Platform Web SDK]의 옵트인 및 옵트아웃을 제어합니다. |
| 새 [!DNL Experience Cloud Debugger Extension]의 클라이언트측 디버깅 지원 | 시스템을 통해 데이터가 이동하는 방식을 보려면 [!DNL Experience Platform web SDK]의 요청과 가장자리 추적을 참조하십시오. |
| Adobe Analytics | 에지 구성을 통해 [!DNL Analytics] 보고서 세트로 데이터를 보냅니다. XDM은 컨텍스트 데이터로 병합되며 다중 세트 태깅을 지원합니다. |
| Adobe Target | Adobe Target 지원. VEC, 양식 기반 컴포저, A/B, XT, Automated Personalization, MVT 포함 |
| Adobe Audience Manager 지원 | [!DNL Audience Manager] ID 동기화, URL 대상 및 쿠키 대상에 대한 지원. |
| ID 동기화 | 좀 더 명확하게 하기 위해 `setCustomersIds`을(를) `syncIdentity`(으)로 변경했습니다. |
| XDM 개체 빌더 | 이제 실행 확장 기능에서 XDM 개체를 데이터 요소로 만들 수 있습니다. |

[!DNL Platform Web SDK] 및 [!DNL Edge Network]에 대한 자세한 내용은 [설명서](../../edge/home.md)를 참조하십시오.

## [!DNL Real-time Customer Profile] {#profile}

Adobe Experience Platform을 사용하면 고객이 브랜드와 상호 작용하는 장소 또는 시기와 상관없이 고객의 관심사와 연관성 있는 경험을 일관되게 전달할 수 있습니다. [!DNL Real-time Customer Profile]을 사용하면 온라인, 오프라인, CRM 및 제3자 데이터를 비롯한 여러 채널의 데이터를 결합하는 각 개별 고객의 전체 보기를 확인할 수 있습니다. [!DNL Profile] 서로 다른 고객 데이터를 하나의 통합 뷰로 통합하여 고객 상호 작용에 대해 실행 가능하고 타임스탬프가 지정된 계정을 제공할 수 있습니다.

**새로운 기능**

| 기능 | 설명 |
| -----------| ---------- |
| 신규 프로필 내보내기 지표 | 내보낸 프로필의 총 수와 각 네임스페이스의 프로필 수를 보여주는 프로필 내보내기 작업에 지표를 추가했습니다. |
| 새로운 통찰력 통찰력 지표 | Observability Insights API에는 이제 [!DNL Profile]에 대한 인제스트를 스트리밍하기 위한 다음 지표가 있습니다.들어오는 요청 비율, 성공한 섭취 비율, 인제스트된 레코드 크기. |
| 벌크 GET 끝점 | 단일 API 호출에서 여러 결과를 검색할 수 있도록 벌크 GET 끝점을 [!DNL Real-time Customer Profile] API에 추가했습니다. 이제 세그먼트 정의, 세그먼트 작업 및 병합 정책에 대해 최대 100개의 ID를 일괄 처리할 수 있습니다. |
| ID로 프로필 검색 | 이제 [!DNL Platform] UI에서 ID 네임스페이스를 선택하고 ID 값을 제공하여 프로필을 검색할 수 있습니다. |

**버그 수정**

- None.

**알려진 문제**

- 없음.

[!DNL Profile] 데이터 작업에 대한 자습서 및 모범 사례를 포함하여 [!DNL Real-time Customer Profile]에 대한 자세한 내용은 [실시간 고객 프로필 개요](../../profile/home.md)를 참조하십시오.

## 소스 {#sources}

Adobe Experience Platform은 외부 소스에서 데이터를 인제스트할 수 있는 동시에 [!DNL Platform] 서비스를 사용하여 해당 데이터를 구조화, 레이블 지정 및 개선할 수 있습니다. Adobe 애플리케이션, 클라우드 기반 스토리지, 제3자 소프트웨어 및 CRM 시스템과 같은 다양한 소스의 데이터를 인제스트할 수 있습니다.

[!DNL Experience Platform] 은 다양한 데이터 공급자에 대한 소스 연결을 쉽게 설정할 수 있는 RESTful API와 대화형 UI를 제공합니다. 이러한 소스 연결을 통해 외부 스토리지 시스템 및 CRM 서비스에 인증 및 연결할 수 있고 통합 실행에 대한 시간을 설정할 수 있으며 데이터 통합 처리량을 관리할 수 있습니다.

**새로운 기능**

| 기능 | 설명 |
| ------- | ----------- |
| 클라우드 스토리지 시스템에 대한 추가 API 및 UI 지원 | [!DNL Azure File Storage]에 대한 새 소스 커넥터입니다. |
| 데이터베이스에 대한 추가 API 및 UI 지원 | [!DNL Azure Data Explorer], IBM DB2 및 [!DNL Oracle DB]에 대한 새 소스 커넥터입니다. |
| Adobe Audience Manager에서 [!DNL Experience Platform] 데이터 공유 | [!DNL Audience Manager] 커넥터에 대한 프로비저닝 프로세스가 업데이트되었습니다. [!DNL Audience Manager] 이제 데이터 세트 [!DNL Real-time Customer Profile] 는 기본적으로 비활성화됩니다. 홍보할 데이터 세트를 수동으로 [!DNL Profile]으로 선택할 수 있습니다. 새 기본 설정은 소급 적용되지 않으며 새 [!DNL Audience Manager] 커넥터에 대한 프로비저닝에만 영향을 줍니다. [데이터 집합 사용자 안내서](../../catalog/datasets/user-guide.md)에서 자세한 내용을 참조하십시오. |

**알려진 문제**

- 없음.
