# Adobe Experience Platform의 데이터 품질

Adobe Experience Platform은 일괄 처리 또는 스트리밍 통합 중 하나를 통해 업로드된 모든 데이터의 완벽성, 정확성 및 일관성을 보장하는 세분화된 지원을 제공합니다. 다음 문서에서는 배치 및 스트리밍 통합에 대해 지원되는 검사 및 유효성 검사 동작에 대한 요약을 제공합니다 [!DNL Experience Platform].

## 지원되는 검사

|   | 배치 수집 | 스트리밍 수집 |
| ------ | --------------- | ------------------- |
| 데이터 유형 확인 | 예 | 예 |
| 열거형 확인 | 예 | 예 |
| 범위 확인(최소, 최대) | 예 | 예 |
| 필수 필드 확인 | 예 | 예 |
| 패턴 확인 | 아니요 | 예 |
| 형식 확인 | 아니요 | 예 |

## 지원되는 유효성 검사 동작

배치 및 스트리밍 수집 모두 의 검색 및 분석을 위해 잘못된 데이터를 이동함으로써 실패한 데이터가 다운스트림으로 이동하는 것을 방지합니다 [!DNL Data Lake]. 데이터 수집은 일괄 처리 및 스트리밍 통합에 대한 다음 유효성 검사를 제공합니다.

### 일괄 처리 수집

일괄 처리에 대해 다음 유효성 검사가 수행됩니다.

| 유효성 검사 영역 | 설명 |
| --------------- | ----------- |
| 스키마 | 스키마가 **not** empty 및 는 다음과 같이 결합 스키마에 대한 참조를 포함합니다. `"meta:immutableTags": ["union"]` |
| `identityField` | 모든 유효한 ID 설명자가 정의되었는지 확인합니다. |
| `createdUser` | 배치를 수집하는 사용자가 배치를 수집할 수 있는지 확인합니다. |

### 스트리밍 수집

스트리밍 수집에는 다음 유효성 검사가 수행됩니다.

| 유효성 검사 영역 | 설명 |
| --------------- | ----------- |
| 스키마 | 스키마가 **not** empty 및 는 다음과 같이 결합 스키마에 대한 참조를 포함합니다. `"meta:immutableTags": ["union"]` |
| `identityField` | 모든 유효한 ID 설명자가 정의되었는지 확인합니다. |
| JSON | JSON이 유효한지 확인합니다. |
| 조직 | 나열된 조직이 유효한 조직인지 확인합니다. |
| 소스 이름 | 데이터 소스의 이름이 지정되었는지 확인합니다. |
| 데이터 세트 | 데이터 세트가 지정, 활성화 및 제거되지 않았는지 확인합니다. |
| Header | 헤더가 지정되고 유효한지 확인합니다. |

방법에 대한 자세한 정보 [!DNL Platform] 데이터를 모니터링하고 검증하는 방법은 [데이터 흐름 설명서 모니터링](./monitor-data-ingestion.md).

## ID 값 유효성 검사

다음 표에서는 ID 값의 성공적인 유효성 검사를 위해 따라야 하는 기존 규칙에 대해 설명합니다.

| 네임스페이스 | 유효성 검사 규칙 | 규칙을 위반할 때 시스템 동작 |
| --- | --- | --- |
| ECID | <ul><li>ECID의 ID 값은 정확히 38자여야 합니다.</li><li>ECID의 ID 값은 숫자로만 구성되어야 합니다.</li></ul> | <ul><li>ECID의 ID 값이 정확히 38자가 아닌 경우 레코드를 건너뜁니다.</li><li>ECID의 ID 값에 숫자가 아닌 문자가 포함되어 있으면 레코드를 건너뜁니다.</li></ul> |
| 비ECID | ID 값은 1024자를 초과할 수 없습니다. | ID 값이 1024자를 초과하는 경우 레코드를 건너뜁니다. |

자세한 내용은 [!DNL Identity Service] 가드레일, [[!DNL Identity Service] 가드 레일 개요](../../identity-service/guardrails.md).
