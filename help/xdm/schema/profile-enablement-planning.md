# 실시간 고객 프로필 활성화 계획

실시간 고객 프로필에 대해 활성화하기 전에 스키마 및 데이터 세트가 준비되었는지 확인할 수 있습니다. 스키마 필드를 디자인한 후 프로필에 대한 스키마를 활성화하기 전에 이 계획 검토를 완료합니다. 프로필 활성화는 데이터 모델에 영구적인 동작 변경 사항을 적용합니다. 스키마 활성화는 되돌릴 수 없으며, 데이터 세트를 활성화하면 실시간 고객 프로필 내에서 해당 레코드가 처리되는 방식에 영향을 줍니다. 스키마 디자인에서 의도하지 않은 활성화, 데이터 품질 문제 또는 장기적인 제한을 방지하려면 이 지침을 검토하십시오.

프로필 활성화는 Experience Platform에서 데이터가 결합, 병합 및 활성화되는 방법을 결정합니다. Planning에서는 변경하기 전에 스키마 구조, ID 구성 및 데이터 세트 목적이 올바른지 확인합니다. 이 계획 검토를 완료한 후 **[!UICONTROL Schema Editor]** 또는 **[!UICONTROL Dataset]** UI에서 프로필에 대한 데이터를 사용하도록 설정할 수 있습니다.

## 전제 조건

이 계획 안내서를 사용하기 전에 다음을 확인하십시오.

* **[!UICONTROL Schema Editor]** 또는 스키마 레지스트리 API를 사용하여 스키마를 디자인했습니다. 시작하려면 [스키마 만들기 튜토리얼](../tutorials/create-schema-ui.md)을 참조하세요.
* 스키마에 ID 필드가 하나 이상 구성되었습니다. 지침은 [ID 필드 구성 가이드](../ui/fields/identity.md)를 검토하십시오.
* [실시간 고객 프로필](../../profile/home.md)에 대한 기본 이해 및 스키마를 사용하여 통합 고객 보기를 작성하는 방법.
* 프로필에 대해 스키마 및 데이터 세트를 활성화하는 데 적절한 권한. 프로필 활성화 옵션에 대한 액세스 권한이 없는 경우 시스템 관리자에게 문의하십시오.

이 필수 구성 요소를 완료하지 않은 경우 이 계획 안내서를 계속 진행하기 전에 [스키마 만들기 자습서](../tutorials/create-schema-ui.md)로 시작하십시오.

## 계획이 중요한 이유 {#why-planning-matters}

프로필 활성화는 Experience Platform에서 데이터를 처리하는 방식을 영구적으로 변경합니다. 다음 영구 변경 사항은 스키마 및 데이터 세트에 적용됩니다.

**스키마**: 프로필에 대해 스키마를 활성화하면 비활성화하거나 삭제할 수 없습니다. 또한 데이터가 수집된 후에는 스키마에서 필드를 제거하거나 이름을 변경할 수 없습니다. 이 영속성은 나중에 결정을 번복하거나 구조를 단순화할 수 없으므로 프로필을 활성화하기 전에 스키마 디자인이 완전하고 안정적이어야 함을 의미합니다.

**데이터 세트**: 프로필에 대한 데이터 세트를 활성화하면 실시간 고객 프로필이 해당 레코드를 사용하여 프로필을 만들고 업데이트합니다. [데이터 세트 사용 안내서](../../catalog/datasets/enable-for-profile.md)에서 데이터 세트 사용 동작을 검토하십시오. 스키마와 달리 나중에 데이터 세트를 비활성화하거나 삭제할 수 있지만 이렇게 하면 연결된 프로필 레코드가 제거되고 세그멘테이션 또는 활성화 워크플로에 영향을 줄 수 있습니다. 활성화된 데이터 세트를 변경하기 전에 다운스트림의 영향을 고려하십시오.

이러한 변경 사항은 다운스트림 프로세스에 영향을 주므로 활성화하기 전에 스키마와 해당 데이터 세트가 프로필에 적합한지 확인하십시오.

### 지원 워크플로우 이해

프로필에 대해 해당 스키마를 사용하는 스키마 및 데이터 세트를 모두 활성화해야 합니다. 다음 순서로 리소스를 활성화합니다.

1. **프로필에 대한 스키마를 사용하도록 설정**: 먼저 **[!UICONTROL Schema Editor]**&#x200B;의 스키마에서 프로필을 사용하도록 설정합니다. 이렇게 하면 이 스키마를 사용하는 모든 데이터 세트를 프로필에 대해 활성화할 수 있습니다.
2. **프로필에 대해 개별 데이터 세트 활성화**: 스키마를 사용하도록 설정한 후 통합 고객 프로필에 기여해야 하는 각 데이터 세트에 대해 프로필을 활성화하십시오.

스키마가 아직 활성화되지 않은 프로필에 대해 데이터 세트를 활성화할 수 없습니다. 스키마는 데이터 세트를 사용하기 위한 사전 요구 사항 역할을 합니다. 이 2단계 프로세스에서는 실시간 고객 프로필이 레코드 처리를 시작하기 전에 데이터 모델의 유효성을 검사합니다.

## 프로필에 대해 스키마 또는 데이터 세트를 활성화해야 하는 경우 {#when-to-enable}

아래 기준을 검토하여 프로필 활성화가 데이터에 적합한지 확인하십시오.

다음 상황에서 프로필 활성화:

* 데이터는 통합 고객 프로필에 기여합니다.
* 데이터는 세그먼테이션 또는 활성화 워크플로우에 필요합니다.
* 스키마에는 개인 또는 고객 수준 키를 나타내는 ID 필드가 포함되어 있습니다.
* 데이터 세트에는 여러 채널에서 결합해야 하는 경험 이벤트 또는 프로필 속성이 포함되어 있습니다. 이벤트 요구 사항을 확인하려면 [XDM ExperienceEvent 클래스](../classes/experienceevent.md)을(를) 검토하십시오.

## 프로필에 대해 스키마 또는 데이터 세트를 활성화하지 않을 경우 {#when-not-to-enable}

다음과 같은 경우에는 프로필을 활성화하지 마십시오.

* 스키마는 조회 또는 참조 전용 데이터를 나타냅니다.
* 데이터 세트에는 테스트, 임시 또는 비프로덕션 데이터가 포함됩니다.
* 이 데이터는 개인을 식별하지 않거나 보고에만 사용됩니다.
* 스키마가 실험적이거나 구조적으로 불완전합니다.

이러한 시나리오에서 프로필을 활성화하면 불필요한 프로필이 생성되거나 라이선스 사용이 증가하거나 장기 스키마 제약 조건이 발생할 수 있습니다.

## 프로필 활성화 전 주요 고려 사항 {#key-considerations}

이 섹션에서 고려 사항을 검토하여 스키마와 데이터 세트가 프로필 사용 사례 및 장기 데이터 거버넌스를 지원하는지 확인하십시오. 프로필을 활성화하기 전에 스키마 구조, ID 구성 및 데이터 세트 목적을 확인하십시오.

### 스키마 준비

스키마 구조를 검토하여 프로필 요구 사항을 지원하는지 확인하십시오. 스키마에는 세분화 및 활성화에 필요한 필드가 포함되어야 하며 실험적이거나 장기간 필요하지 않은 필드는 제외해야 합니다. 활성화 후에 추가하는 모든 추가 필드는 추가적이어야 합니다(자세한 내용은 [스키마 불변성 제약 조건](#why-planning-matters) 참조). 이 제한 사항은 프로필을 활성화하기 전에 필드 선택 사항의 유효성을 신중하게 검사해야 함을 의미합니다. 허용되는 업데이트에 대한 자세한 내용은 [스키마 진화 규칙](./composition.md#evolution)을 참조하세요.

### ID 구성

ID 구성은 프로필이 데이터 세트 간에 레코드를 결합하는 방법을 결정합니다. 먼저 유효한 기본 ID가 선택되었는지 확인합니다. 이 필드는 안정적이고 고유하며 모든 레코드에서 일관되게 채워져야 합니다. 결합 오류를 방지하기 위해 ID 네임스페이스가 올바르게 할당되었는지 확인합니다. 보조 ID를 사용하는 경우 다른 개인이 동일한 ID 값을 공유할 때 발생할 수 있는 프로필 충돌을 일으키지 않고 사용 사례를 지원하는지 확인하십시오. 실시간 고객 프로필은 충돌하는 레코드를 함께 결합할 때 우선하는 데이터를 결정하는 병합 정책을 적용하여 충돌을 해결합니다.

### 데이터 세트 목적

다운스트림 워크플로에서 사용된 프로필 속성 또는 경험 이벤트에 직접 기여하는 경우에만 프로필에 대한 데이터 세트를 활성화합니다. 세그먼테이션에서 사용되지 않는 조회 또는 참조 데이터, 테스트 또는 샘플 데이터 또는 활성화용으로 사용되지 않는 운영 체제 생성 레코드가 포함된 데이터 세트를 활성화하지 마십시오. 이러한 데이터 유형은 통합 고객 프로필에 기여하지 않으며 불필요한 스토리지 오버헤드를 만듭니다. 데이터 세트에 세그먼테이션 및 활성화를 지원하는 ID 필드 또는 고객 행동 데이터가 포함되어 있지 않은 경우 프로필에 대해 활성화하지 마십시오.

**예**:

고객 ID가 있는 거래 데이터가 포함된 &quot;고객 구매 이벤트&quot; 데이터 세트를 활성화합니다. 실시간 고객 프로필은 이러한 이벤트를 사용하여 고객 타임라인을 구축하고 구매 행동에 따라 세그멘테이션을 활성화합니다.

고객 식별자 없이 SKU 참조 데이터만 포함하는 &quot;제품 카탈로그&quot; 데이터 세트는 활성화하지 않습니다. 이러한 유형의 데이터 세트를 활성화하면 통합 고객 프로필에 영향을 주지 않고 불필요한 스토리지 오버헤드가 발생합니다.

## 사전 지원 체크리스트 {#pre-enablement-checklist}

프로필에 대한 스키마 또는 데이터 세트를 활성화하기 전에 이 체크리스트를 사용하여 준비 상태를 확인하십시오. 프로필을 활성화하기 전에 각 항목을 완료합니다.

### 스키마 수준 검사

먼저 스키마 디자인이 완전하고 안정적인지 확인합니다. 스키마를 검토하여 사용 사례의 모든 필수 필드가 있는지 확인하고 실험 또는 임시 필드가 포함되어 있지 않은지 확인하십시오. 스키마가 권장 패턴을 따르는지 확인하려면 [스키마 디자인 모범 사례](./best-practices.md)를 참조하세요. 최종 필드 목록에서 팀의 승인을 얻습니다([스키마 불변성 제약 조건](#why-planning-matters) 참조).

그런 다음 기본 ID 구성이 올바른지 확인합니다. **[!UICONTROL Schema Editor]**&#x200B;에서 스키마를 열고 ID 아이콘으로 표시된 필드를 찾습니다. 이 필드가 소스 데이터에서 일관되게 채워지고 ID 네임스페이스가 사용 사례에 적합한지 확인합니다. 기본 ID는 모든 레코드에서 안정적이고 고유하며 안정적으로 제공되어야 올바른 프로필 결합이 보장됩니다.

마지막으로, 스키마 구조의 이름을 바꾸거나 재구성할 필요가 없는지 확인합니다. 스키마 구조 변경은 추가 업데이트로만 제한됩니다([스키마 불변성 제약 조건](#why-planning-matters) 참조). 이름 지정 또는 조직의 장기적 모호성은 나중에 수정할 수 없으므로 활성화하기 전에 이러한 문제를 해결하십시오.

### 데이터 세트 수준 검사

활성화하려는 각 데이터 세트에 대해 프로필 관련 데이터가 포함되어 있는지 확인하는 것부터 시작합니다. 샘플 레코드를 검토하여 단순히 운영되거나 참조되는 정보가 아닌 고객 또는 이벤트 데이터가 포함되어 있는지 확인합니다. 레코드에 고객 프로필에 연결되는 ID 값이 포함되어 있는지 확인합니다. ID 필드 또는 고객 동작 데이터가 없는 데이터 세트는 프로필에 대해 활성화하면 안 됩니다.

데이터 세트의 ID 값이 프로필 활성화 환경의 다른 데이터 세트와 어떻게 관련되는지 파악하여 데이터 세트가 ID 결합 또는 세그멘테이션에 기여해야 하는지 여부를 결정합니다. 이 데이터 세트의 레코드를 기존 프로필과 연결할지 또는 새 프로필 조각을 만들지 여부를 고려합니다. 실시간 고객 프로필이 데이터 세트 간에 레코드를 결합하는 방법과 이 데이터 세트가 전체 ID 전략에 어떻게 적합한지 이해하려면 [병합 정책 설명서](../../profile/merge-policies/overview.md)를 검토하십시오.

데이터 세트를 활성화하기 전에 데이터 세트에 포함된 고유한 ID 값의 수를 예측하고 이러한 ID 값이 테스트 계정이나 시스템 식별자가 아닌 실제 고객을 나타내는지 확인하십시오. 각 고유 ID가 주소 지정 가능한 대상자 규모에 기여하므로 이 데이터 세트를 활성화하면 라이선스 권한에 맞게 조정되는지 확인하십시오. 프로필을 사용하면 스토리지 및 처리 비용이 증가하므로 데이터 세트가 이러한 투자를 정당화하는 가치를 제공하도록 할 수 있습니다.

이 체크리스트를 완료하면 활성화 후 되돌릴 수 없는 문제를 방지할 수 있습니다.

## 스키마 및 데이터 세트에 대한 프로필 활성화 {#enable-profile}

사전 활성화 체크리스트를 완료한 후 다음 단계에 따라 프로필을 활성화하십시오. [지원 워크플로 이해](#why-planning-matters)에서 설명한 대로 해당 스키마를 사용하는 데이터 세트를 활성화하려면 먼저 스키마를 활성화해야 합니다.

### 프로필에 대한 스키마 활성화

먼저 스키마에서 프로필 활성화:

1. Experience Platform UI에서 **[!UICONTROL Schemas]**(으)로 이동합니다.
2. 목록에서 스키마를 선택하여 **[!UICONTROL Schema Editor]**&#x200B;에서 엽니다.
3. 오른쪽 레일에서 **[!UICONTROL Profile]** 토글을 선택합니다. 스키마 속성 패널에 확인 대화 상자가 표시됩니다.
4. **[!UICONTROL Enable]**&#x200B;을(를) 선택하여 확인하십시오. 이제 프로필에 대해 스키마를 사용할 수 있습니다.

자세한 지침은 스키마 편집기 설명서의 [스키마 사용 안내서](../ui/resources/schemas.md#profile)를 참조하십시오.

### 프로필용 데이터 세트 활성화

프로필에 대해 스키마를 활성화한 후 통합 프로필에 기여해야 하는 각 데이터 세트를 활성화합니다.

1. Experience Platform UI에서 **[!UICONTROL Datasets]**(으)로 이동합니다.
2. 목록에서 데이터 세트를 선택하여 데이터 세트 세부 사항 페이지를 엽니다.
3. 오른쪽 레일에서 **[!UICONTROL Profile]** 토글을 선택합니다. 프로필을 표시하기 위해 데이터 세트 속성 패널이 업데이트됩니다.

실시간 고객 프로필에 기여해야 하는 각 데이터 세트에 대해 이 프로세스를 반복합니다. 자세한 지침 및 API 지원 옵션은 사전 요구 사항 섹션에서 참조된 데이터 세트 사용 안내서를 참조하십시오.

### 주문이 중요한 이유

[지원 워크플로 이해](#why-planning-matters)에서 설명한 대로 데이터 세트를 활성화하기 전에 스키마를 활성화해야 합니다. 이렇게 하면 실시간 고객 프로필에서 데이터 세트 활성화를 허용하기 전에 스키마 구조가 프로필 작업을 지원하는지 확인하고 스키마를 사용하는 모든 데이터 세트는 세그멘테이션 및 ID 결합에 대한 올바른 필드 정의를 상속합니다.

스키마와 데이터 세트를 모두 활성화하면 실시간 고객 프로필이 레코드 처리 및 통합 고객 프로필 작성을 시작합니다. 활성화 전에 수집된 레코드는 데이터를 다시 수집하지 않는 한 프로필에 포함되지 않습니다.

## 다음 단계 {#next-steps}

프로필 활성화의 영구적인 효과를 검토하고, 스키마와 데이터 세트가 준비되었는지 확인하고, ID 구성이 사용 사례를 지원하는지 확인했습니다. 스키마 구조 및 필드 관계를 더 깊이 이해하려면 스키마 진화 규칙과 데이터 모델 내에서 필드가 상호 작용하는 방법에 대해 설명하는 [스키마 구성 기본 사항](../schema/composition.md)을 검토하십시오. 활성화 중이나 후에 문제가 발생하면 일반적인 문제 및 해결 방법에 대해서는 [XDM 문제 해결 안내서](../troubleshooting-guide.md)를 참조하십시오. ID 네임스페이스가 프로필 결합 및 해결에 미치는 영향에 대해 자세히 알아보려면 [ID 네임스페이스 개요](../../identity-service/features/namespaces.md)를 검토하십시오.
