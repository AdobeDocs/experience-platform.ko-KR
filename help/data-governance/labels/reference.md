# 데이터 사용 레이블 용어집

데이터 사용 레이블을 사용하면 해당 데이터에 적용되는 사용 정책에 따라 데이터 세트와 필드를 분류할 수 있습니다. Adobe Experience Platform 데이터 거버넌스는 데이터 카테고리를 시작하는 데 사용할 수 있는 몇 가지 핵심 데이터 사용 레이블을 즉시 제공합니다.

이 문서에서는 현재 [!DNL Experience Platform]에서 제공하는 핵심 데이터 사용 레이블에 대해 간략히 설명합니다. [!DNL Data Governance]에 대한 자세한 내용은 [데이터 거버넌스 개요](../home.md)에서 확인할 수 있습니다.

## 계약 레이블

계약 &quot;C&quot; 레이블은 계약 의무가 있거나 조직의 데이터 관리 정책과 관련된 데이터를 분류하는 데 사용됩니다.

| 레이블 | 정의 |
|---|---|
| **C1** | 데이터는 개별 또는 장치 식별자를 포함하지 않고 Adobe Experience Cloud에서 집계된 형식으로만 내보낼 수 있습니다. [추가 정보...](#c1) |
| **C2** | 데이터를 제3자로 내보낼 수 없습니다. [추가 정보...](#c2) |
| **C3** | 데이터는 결합하거나 직접 식별 가능한 정보와 함께 사용할 수 없습니다. [추가 정보...](#c3) |
| **C4** | 온사이트 또는 사이트 간 광고나 컨텐트를 타게팅하는 데 데이터를 사용할 수 없습니다. [추가 정보...](#c4) |
| **C5** | 데이터를 컨텐츠 또는 광고의 관심 기반 크로스 사이트 타깃팅에 사용할 수 없습니다. [추가 정보...](#c5) |
| **C6** | 데이터는 온사이트 광고 타깃팅에 사용할 수 없습니다. [추가 정보...](#c6) |
| **C7** | 데이터를 컨텐츠의 온사이트 타깃팅에 사용할 수 없습니다. [추가 정보...](#c7) |
| **C8** | 조직의 웹 사이트 또는 앱을 측정하는 데 데이터를 사용할 수 없습니다. [추가 정보...](#c8) |
| **C9** | 데이터는 데이터 과학 워크플로우에서 사용할 수 없습니다. [추가 정보...](#c9) |
| **C10** | 연결된 ID 활성화에는 데이터를 사용할 수 없습니다. [추가 정보...](#c10) |

## ID 레이블

ID &quot;I&quot; 레이블은 특정 개인을 식별하거나 연락할 수 있는 데이터를 분류하는 데 사용됩니다.

| 레이블 | 정의 |
|---|---|
| **I1** | 장치가 아닌 특정 사람을 식별하거나 접촉할 수 있는 직접 식별 가능한 데이터. |
| **I2** | 특정 개인을 식별하거나 접촉하기 위해 다른 데이터와 함께 사용할 수 있는 간접적으로 식별 가능한 데이터입니다. |

## 민감한 레이블

민감한 &quot;S&quot; 레이블은 사용자와 조직에서 민감한 데이터를 분류하는 데 사용됩니다.

중요한 유형의 데이터는 지리적인 데이터의 서로 다른 유형일 수 있습니다.하지만 이 카테고리는 지리적 데이터에만 국한되지 않습니다.

| 레이블 | 정의 |
|---|---|
| **S1** | 장치의 정확한 위치를 결정하는 데 사용할 수 있는 위도 및 경도를 지정하는 데이터. |
| **S2** | 광범위하게 정의된 위치 영역을 결정하는 데 사용할 수 있는 데이터입니다. |

## 부록

아래 섹션에서는 사용 가능한 데이터 사용 레이블에 대한 추가 정보를 제공합니다.

### 계약 레이블 세부 사항

다음 섹션에서는 특정 계약 &quot;C&quot; 레이블의 구현과 관련된 세부 정보를 제공합니다.

#### C1 {#c1}

일부 데이터는 개별 또는 장치 식별자를 포함하지 않고 Adobe Experience Cloud에서 집계된 형식으로만 내보낼 수 있습니다. 예를 들어 소셜 네트워크에서 가져온 데이터입니다.

#### C2 {#c2}

일부 데이터 제공업체는 원래 데이터가 수집된 위치에서 내보내기를 금지하는 계약을 가지고 있습니다. 예를 들어, 소셜 네트워크 계약에서는 종종 해당 계약으로부터 받는 데이터의 전송을 제한합니다. C2 레이블은 집계 및 익명 데이터만 필요한 [C1](#c1)보다 더 제한적인 것입니다.

#### C3 {#c3}

일부 데이터 제공업체는 계약에 직접 식별 가능한 정보와 함께 해당 데이터의 조합 또는 사용을 금지하는 약관을 가지고 있습니다. 예를 들어 광고 네트워크, 광고 서버 및 제3자 데이터 공급자로부터 제공되는 데이터에 대한 계약에는 종종 직접 식별 가능한 데이터와 함께 이러한 데이터의 사용에 대한 특정 계약 금지 사항이 포함됩니다.

#### C4 {#c4}

C4는 가장 제한적인 레이블입니다. 레이블 [C5](#c5), [C6](#c6) 및 [C7](#c7)이 포함됩니다.

#### C5 {#c5}

다음 3가지 조건이 충족되면 관심 기반 타깃팅 또는 개인화가 발생합니다.온사이트에서 수집된 데이터는 (1) 사용자의 관심사에 대한 참조를 위해 사용되고, (2) 다른 사이트나 앱(오프라인) 등의 다른 컨텍스트에서 사용되며 (3) 이러한 참조를 기반으로 제공되는 컨텐트 또는 광고를 선택하는 데 사용됩니다.

사이트 내 데이터와 오프사이트 데이터의 조합 또는 여러 오프라인 소스의 데이터 조합을 포함하여 여러 사이트의 데이터 조합을 사이트 간 데이터라고 합니다. 사이트마다 컨텍스트가 다르기 때문에 모든 컨텍스트에서 크로스 사이트 데이터의 사용이 원본과 다를 수 있습니다. 사이트 간 데이터는 일반적으로 사용자 관심사에 대한 참조를 위해 수집 및 처리됩니다. 그 결과, 광고나 컨텐츠가 사이트 내 또는 외부 사이트에 표시되는지 여부에 관계없이 광고 또는 컨텐츠를 타깃팅하기 위해 사이트 간 데이터를 사용하는 것은 일반적으로 관심 기반 타깃팅으로 간주됩니다. 예를 들어 온사이트 데이터가 오프사이트 데이터와 함께 사용되어 조직의 고유 사이트에 사용자를 표시하는 광고를 선택하면 관심 기반 타깃팅으로 자격이 부여됩니다. 또 다른 예로, 오프사이트 사용자에 대한 광고를 다시 타깃팅하는 것도 관심 기반 타깃팅으로 간주될 수 있습니다.

오프사이트 데이터를 타깃팅하기 위해 단독으로 사용하면 외부 데이터를 일반적으로 수집 및 처리하여 사용자의 관심사를 파악하므로 관심 기반 타깃팅으로 간주할 수 있습니다.

그러나 온사이트 데이터만 사용하여 컨텐트 또는 광고를 타깃팅하는 것은 일반적으로 관심 기반 타깃팅으로 간주되지 않습니다. 그렇지 않으면 관심 기반 타깃팅으로 간주되지 않는 온사이트 타깃팅은 두 개의 서로 다른 레이블로 취급됩니다. 특히 Label C6는 사이트 내 광고 타깃팅 및 보고를 해결하고, 광고 선택, 전달 및 보고를 구체적으로 설명하며, Label C7은 사이트 내 컨텐츠 선택, 전달 및 보고(사이트 내 컨텐츠 타깃팅) 문제를 해결합니다.

궁극적으로 레이블의 해석 및 해당 레이블이 있는 데이터의 사용 방법이 적용되는 방식은 사용자에게 달려 있습니다. 참조용으로 IAB 및 DAA 프레임워크는 아래에 제공됩니다.

IAB:개인화. 나중에 다른 웹사이트 또는 응용 프로그램과 같은 다른 컨텍스트에서 나중에 귀하를 위해 광고 및/또는 컨텐트를 개인화하기 위해 이 서비스를 사용하는 것에 대한 정보를 수집하고 처리합니다. 일반적으로 사이트 또는 앱의 컨텐츠는 향후 선택된 광고 및/또는 컨텐트에 대한 정보를 제공하는 귀하의 관심사에 대해 추론하는 데 사용됩니다.

DAA:온라인 행동 광고 이러한 데이터를 사용하여 사용자의 선호도 또는 관심사를 예측하여 그러한 웹 보기 비헤이비어에서 유추된 환경 설정 또는 관심사를 기반으로 해당 컴퓨터 또는 장치에 광고를 전달하기 위한 목적으로 시간 경과에 따른 웹 보기 비헤이비어와 관련된 특정 컴퓨터 또는 장치의 데이터를 수집합니다.

#### C6 {#c6}

광고는 주로 상품 또는 서비스 판매를 촉진하려는 웹사이트 또는 앱에 나타나는 텍스트 및 이미지를 포함한 메시지 또는 알림입니다. 이러한 메시지 또는 알림의 용도를 결정하는 것은 전적으로 사용자의 책임입니다. 광고는 사이트 내 컨텐츠와 별개입니다. 레이블 [C7](#c7)이 표시됩니다. C6 레이블이 있는 데이터는 조직의 웹 사이트 또는 앱에서 광고를 선택 및 배달하거나 이러한 광고의 전달 및 효과를 측정하는 등 현장 광고 타깃팅에 사용할 수 없습니다. 여기에는 사용자가 광고를 선택하는 데 사용자의 관심사에 대한 이전에 수집된 사이트 내 데이터를 사용하고, 광고가 표시되는 내용과 장소, 사용자가 광고 선택 또는 구매와 같이 광고와 관련된 조치를 취했는지 여부 등이 포함됩니다. 일반적으로 사용자의 온사이트 활동을 기반으로 하고 사이트 내 광고 타깃팅에서 이러한 기본 설정을 사용하는 사용자의 선호 사항을 참조하면 관심 기반 타깃팅에 필요한 세 가지 요구 사항을 모두 충족하지는 않으므로 관심 기반 타깃팅으로 간주되지 않습니다(개인화라고도 함). *[이러한 요구 사항은 레이블 C5를 참조하십시오.](#c5)*

궁극적으로 레이블의 해석 및 해당 레이블이 있는 데이터의 사용 방법이 적용되는 방식은 사용자에게 달려 있습니다. 참조용으로 IAB 및 DAA 프레임워크는 아래에 제공됩니다.

IAB:3. 광고 선택, 전달, 보고:정보를 수집하고 이전에 수집한 정보와 결합하여 광고를 선택 및 전달하고 이러한 광고의 제공 및 효과를 측정합니다. 여기에는 광고를 선택하기 위해 이전에 수집한 정보, 표시되는 광고에 대한 데이터 처리, 광고의 표시 빈도, 광고의 표시 시기 및 위치, 광고 선택 또는 구매와 같은 광고와 관련된 조치를 취했는지 여부 등이 포함됩니다. 이러한 정보에는 개인화가 포함되지 않습니다. 개인화는 시간에 따라 웹사이트 또는 앱과 같은 다른 컨텍스트에서 나중에 귀하를 위해 광고 및/또는 컨텐트를 개인화하기 위해 이 서비스를 사용하는 것에 대한 정보를 수집하고 처리하는 것입니다.

DAA:온라인 행동 광고에는 퍼스트 파티, 광고 전달 또는 광고 보고, 또는 컨텍스트 광고(즉, 방문하는 웹 페이지의 컨텐츠, 소비자의 현재 웹 페이지 방문 또는 검색 쿼리 기반 광고)가 포함되지 않습니다.

#### C7 {#c7}

사이트 내 컨텐츠는 상품 또는 서비스의 판매를 촉진하기 위해 만들어진 것이 아니라 정보, 교육 또는 오락 등의 목적으로 만들어진 텍스트와 이미지입니다. 컨텐츠가 네이티브 광고로 자격이 되는지 여부를 포함하여 컨텐츠의 용도를 결정하는 것은 전적으로 귀하에게 달려 있습니다. C7 레이블은 레이블 [C6](#c6)에 적용되는 온사이트 광고를 덮기 위한 것이 아닙니다. C7 레이블이 있는 데이터는 조직의 웹 사이트 또는 앱에서 컨텐츠를 선택 및 제공하는 등 온사이트 컨텐츠 타깃팅에 사용하거나 이러한 컨텐츠의 전달 및 효과를 측정할 수 없습니다. 여기에는 선택한 컨텐츠에 대한 사용자의 관심사, 표시되는 컨텐츠에 대한 데이터 처리, 표시되는 빈도 또는 시간, 표시되는 시기 및 위치, 사용 시 컨텐츠 선택 등 컨텐츠와 관련된 작업이 수행되었는지 여부에 대한 정보가 포함됩니다. 일반적으로, 사용자의 온사이트 활동을 기반으로 하고 사이트 내 컨텐츠 타깃팅에서 이러한 기본 설정을 사용하는 사용자의 선호 사항을 참조하면 관심 기반 타깃팅에 필요한 세 가지 요구 사항을 모두 충족하지는 않으므로 관심 기반 타깃팅으로 간주되지 않습니다(개인화라고도 함). *[이러한 요구 사항은 레이블 C5를 참조하십시오.](#c5)*

궁극적으로 레이블의 해석 및 해당 레이블이 있는 데이터의 사용 방법이 적용되는 방식은 사용자에게 달려 있습니다. 참조용으로 IAB 및 DAA 프레임워크는 아래에 제공됩니다.

IAB:4. 컨텐츠 선택, 전달, 보고:정보 수집 및 이전에 수집한 정보와 조합하여 사용자를 위해 컨텐츠를 선택 및 전달하고 이러한 컨텐츠의 전달 및 효과를 측정할 수 있습니다. 여기에는 컨텐츠 선택을 위해 이전에 수집한 정보, 표시되는 컨텐츠에 대한 데이터 처리, 표시되는 빈도 또는 시간, 표시되는 시기 및 위치, 컨텐츠 선택 등 컨텐츠와 관련된 작업을 수행했는지 여부 등이 포함됩니다. 이러한 정보에는 개인화가 포함되지 않습니다. 개인화는 시간이 지남에 따라 웹사이트 또는 앱과 같은 다른 컨텍스트에서 나중에 컨텐츠 및/또는 광고를 개인화하기 위해 이 서비스를 사용하는 방법에 대한 정보를 수집하고 처리하는 것입니다.

DAA:온라인 행동 광고에는 퍼스트 파티, 광고 전달 또는 광고 보고, 또는 컨텍스트 광고(즉, 방문하는 웹 페이지의 컨텐츠, 소비자의 현재 웹 페이지 방문 또는 검색 쿼리 기반 광고)가 포함되지 않습니다.

#### C8 {#c8}

데이터를 사용하여 조직의 사이트 또는 앱의 사용자 사용을 측정, 이해 및 보고할 수 없습니다. 여기에는 특정 시간에 걸쳐 웹 사이트 또는 앱과 같은 다른 서비스 등 다른 컨텍스트에서 나중에 컨텐츠 및/또는 광고를 개인화하기 위해 이 서비스를 사용하는 것에 대한 정보를 수집하는 관심 기반 타깃팅(교차 사이트 타깃팅)이 포함되지 않습니다.

#### C9 {#c9}

일부 계약에는 데이터 과학을 위한 데이터 사용에 대한 명시적 금지 사항이 포함되어 있습니다. 경우에 따라 AI(인위적 Intelligence), 머신 러닝(ML) 또는 모델링에 대한 데이터 사용을 금지하는 용어로 표현됩니다.

#### C10 {#c10}

일부 데이터 사용 정책은 개인화를 위해 연결된 ID 데이터의 사용을 제한합니다. C10 레이블은 병합 정책이 &quot;비공개 그래프&quot; 옵션을 사용하는 경우 세그먼트에 자동으로 적용됩니다.
