# 데이터 사용 레이블 용어집 {#data-usage-labels-glossary}

>[!CONTEXTUALHELP]
>id="platform_policies_labeltype"
>title="레이블 유형"
>abstract="데이터 사용 레이블에는 몇 가지 카테고리가 있습니다. Adobe 정의 레이블에는 계약 레이블, ID 레이블 및 중요 레이블이 포함됩니다. 조직에서 정의한 레이블은 사용자 지정 레이블로 분류됩니다."
>text="See the data usage labels glossary for more information on these label types."

데이터 사용 레이블을 사용하여 데이터 세트와 필드를 [거버넌스 정책](../policies/overview.md) 및 [액세스 제어 정책](../../access-control/abac/overview.md) 해당 데이터에 적용됩니다. Adobe Experience Platform에서는 데이터 분류를 시작하는 데 사용할 수 있는 몇 가지 핵심 데이터 사용 레이블을 제공합니다.

이 문서에서는 Experience Platform이 현재 제공하는 핵심 데이터 사용 레이블에 대해 설명합니다.

## 계약 레이블

계약 &quot;C&quot; 레이블은 계약 의무가 있거나 조직의 데이터 거버넌스 정책과 관련된 데이터를 분류하는 데 사용됩니다.

| 레이블 | 정의 |
| --- | --- |
| [C1](#c1) | 데이터는 개별 또는 장치 식별자를 포함하지 않고 집계된 양식으로 Adobe Experience Cloud에서만 내보낼 수 있습니다. |
| [C2](#c2) | 데이터를 타사 시스템으로 내보낼 수 없습니다. |
| [C3](#c3) | 데이터는 결합하거나 직접 식별 가능한 정보와 함께 사용할 수 없습니다. |
| [C4](#c4) | 데이터는 온사이트 또는 사이트 간 광고 또는 컨텐츠를 타깃팅하는 데 사용할 수 없습니다. |
| [C5](#c5) | 데이터는 컨텐츠 또는 광고에 대한 관심사 기반의 사이트 간 타깃팅에 사용할 수 없습니다. |
| [C6](#c6) | 온사이트 광고 타깃팅에는 데이터를 사용할 수 없습니다. |
| [C7](#c7) | 데이터는 컨텐츠를 온사이트 타깃팅하는 데 사용할 수 없습니다. |
| [C8](#c8) | 조직의 웹 사이트 또는 앱을 측정하는 데 데이터를 사용할 수 없습니다. |
| [C9](#c9) | 데이터는 데이터 과학 워크플로우에서 사용할 수 없습니다. |
| [C10](#c10) | 결합된 ID 활성화에는 데이터를 사용할 수 없습니다. |
| [C11](#c11) | 데이터를 세그먼트 일치 파트너와 공유할 수 없습니다. |
| [C12](#c12) | 데이터를 어떤 식으로든 내보낼 수 없습니다. |

## ID 레이블

ID &quot;I&quot; 레이블은 특정 개인을 식별하거나 연락할 수 있는 데이터를 분류하는 데 사용됩니다.

| 레이블 | 정의 |
| --- | --- |
| **I1** | 장치가 아닌 특정 개인을 식별하거나 연락할 수 있는 직접 식별 가능한 데이터. |
| **I2** | 특정 개인을 식별하거나 연락하기 위해 다른 데이터와 함께 사용할 수 있는 간접적으로 식별 가능한 데이터. |

## 중요 레이블 {#sensitive}

중요 &quot;S&quot; 레이블은 사용자와 조직이 민감하도록 데이터를 분류하는 데 사용됩니다.

민감하다고 고려할 수 있는 데이터 유형은 서로 다른 지리적 데이터 유형일 수 있습니다. 하지만 이 카테고리는 지리적 데이터에만 국한되지 않습니다.

| 레이블 | 정의 |
| --- | --- |
| **S1** | 장치의 정확한 위치를 결정하는 데 사용할 수 있는 위도와 경도를 지정하는 데이터. |
| **S2** | 광범위하게 정의된 지오펜스 영역을 결정하는 데 사용할 수 있는 데이터입니다. |
| **PSPD** | 허용된 중요 개인 데이터(PSPD)는 Adobe이 계약상 &quot;중요&quot;, &quot;특수 데이터 카테고리&quot; 또는 해당 법률에서 사용하는 유사한 용어로 간주하는 업로드를 허용하는 데이터를 의미합니다. 특히 PHI(Protected Health Information) 및 기타 제어 상태 데이터는 제외됩니다. |
| **RHD** | PHI(Protected Health Information)를 참조하는 데이터 또는 Adobe이 수동으로 업로드할 수 있는 환자의 정보를 참조합니다. |

## 부록

아래 섹션에서는 사용 가능한 데이터 사용 레이블에 대한 추가 정보를 제공합니다.

### 계약 레이블 세부 정보

다음 섹션에서는 특정 계약 &quot;C&quot; 레이블 구현과 관련된 자세한 정보를 제공합니다.

#### C1 {#c1}

일부 데이터는 개별 또는 장치 식별자를 포함하지 않고 집계된 양식으로 Adobe Experience Cloud에서만 내보낼 수 있습니다. 예를 들어, 소셜 네트워크에서 가져온 데이터입니다.

#### C2 {#c2}

일부 데이터 제공업체는 원래 수집된 위치에서 데이터를 내보내는 것을 금지하는 계약 조건을 가지고 있습니다. 예를 들어 소셜 네트워크 계약은 종종 그로부터 받는 데이터 전송을 제한합니다. C2 레이블은 보다 더 제한적입니다 [C1](#c1)- 집계 및 익명 데이터만 필요하지만 보다 덜 제한적입니다. [C12](#c12): 대상에 관계없이 데이터 내보내기를 완전히 방지합니다.

#### C3 {#c3}

일부 데이터 제공업체는 계약에 직접 식별 가능한 정보와 함께 해당 데이터의 결합 또는 사용을 금지하는 약관을 가지고 있습니다. 예를 들어, 광고 네트워크, 광고 서버 및 타사 데이터 공급자로부터 가져온 데이터에 대한 계약에는 직접 식별 가능한 데이터로 이러한 데이터를 사용하는 것에 대한 특정 계약 금지 사항이 포함되어 있는 경우가 있습니다.

#### C4 {#c4}

C4는 레이블을 포함합니다 [C5](#c5), [C6](#c6), 및 [C7](#c7). 이 레이블은 다음으로 가장 제한적인 레이블 중 하나이며, [C12](#c12).

#### C5 {#c5}

다음 세 가지 조건이 충족되면 관심사 기반 타깃팅 또는 개인화가 발생합니다. 사이트에서 수집된 데이터는 (1) 사용자의 관심사에 대한 추론하는 데 사용되고, (2) 다른 사이트 또는 앱(오프사이트) 등 다른 컨텍스트에서 사용되며 (3) 이 이러한 참조를 기반으로 제공되는 컨텐츠나 광고를 선택하는 데 사용됩니다.

온사이트 데이터와 오프사이트 데이터의 조합 또는 여러 오프사이트 소스의 데이터 조합을 포함하여 여러 사이트의 데이터 조합을 교차 사이트 데이터라고 합니다. 서로 다른 사이트는 모든 컨텍스트에서 교차 사이트 데이터의 사용이 원본과 다르도록 서로 다른 컨텍스트를 나타냅니다. 사이트 간 데이터는 일반적으로 수집 및 처리되어 사용자의 관심사에 대한 참조를 만듭니다. 그 결과, 광고나 컨텐츠가 사이트에 표시되는지 또는 오프사이트 위치에 표시되는지에 관계없이 일반적으로 광고 또는 컨텐츠를 타깃팅할 사이트 간 데이터를 사용하는 것은 관심 기반 타깃팅으로 분류됩니다. 예를 들어, 온사이트 데이터가 오프사이트 데이터와 결합하여 조직의 사이트에 사용자를 표시할 광고를 선택한 경우 이 데이터를 사용하면 관심 기반 타깃팅으로 분류됩니다. 다른 예로, 오프사이트 사용자에게 광고를 다시 타겟팅하는 것도 관심사 기반 타겟팅으로 사용할 수 있습니다.

오프사이트 데이터를 타깃팅하기 위해 오프사이트 데이터만 사용하면 사용자의 관심사에 대해 추론하기 위해 일반적으로 수집 및 처리되므로 관심사 기반 타깃팅으로 분류될 수도 있습니다.

그러나 온사이트 데이터만 사용하여 콘텐츠 또는 광고를 타겟팅하는 것은 일반적으로 관심사 기반 타겟팅으로 분류되지 않습니다. 관심사 기반 타깃팅으로 적합하지 않은 온사이트 타깃팅은 두 개의 개별 레이블로 처리됩니다. 특히, Label C6는 온사이트 광고 타깃팅 및 보고를 해결하며, 특히 광고 선택, 전달 및 보고, Label C7 주소를 온사이트 컨텐츠 선택, 전달 및 보고(온사이트 컨텐츠 타깃팅)에 대해 설명합니다.

궁극적으로, 레이블의 해석 및 해당 레이블이 있는 데이터의 사용이 적용되는 방식은 사용자에게 달려 있습니다. 참조용으로 IAB 및 DAA 프레임워크는 아래에 제공됩니다.

IAB: 개인화. 이 서비스의 사용에 대한 정보를 수집하고 처리하여 시간이 지남에 따라 다른 웹 사이트나 앱과 같은 다른 컨텍스트에서 광고 및/또는 콘텐츠를 나중에 개인화할 수 있습니다. 일반적으로 사이트 또는 앱의 컨텐츠는 사용자의 관심사에 대해 추론하는 데 사용되며, 이런 맥락으로 인해 광고 및/또는 컨텐츠의 향후 선택을 알려줍니다.

데이터: 온라인 행동 광고. 특정 컴퓨터 또는 장치에서 시간 및 비제휴 웹 사이트에서 웹 보기 동작에 대한 데이터를 수집하여 이러한 데이터를 사용하여 사용자 환경 설정 또는 관심사를 예측하여 그러한 웹 보기 동작에서 추론된 환경 설정 또는 관심사를 기반으로 해당 컴퓨터 또는 장치에 광고를 전달할 수 있습니다.

#### C6 {#c6}

광고는 주로 제품이나 서비스 판매를 홍보하기 위한 웹 사이트 또는 앱에 표시되는 텍스트 및 이미지를 포함한 메시지 또는 알림입니다. 이러한 메시지 또는 알림의 용도를 결정하는 것은 귀하의 책임입니다. 광고는 온사이트 콘텐츠와 별도로 레이블로 덮여 있습니다 [C7](#c7). C6 레이블이 있는 데이터는 조직의 웹 사이트 또는 앱에서 광고를 선택 및 게재하거나 그러한 광고의 게재 및 효과를 측정하는 등 온사이트 광고 타깃팅에 사용할 수 없습니다. 여기에는 사용자의 관심사에 대해 이전에 수집된 온사이트 데이터를 사용하여 광고를 선택하고, 표시된 광고에 대한 데이터를 처리하며, 광고가 표시되는 위치와 광고 선택 또는 구매 등과 같은 광고와 관련된 작업을 사용자가 수행했는지 여부가 포함됩니다. 일반적으로 해당 사용자의 온사이트 활동을 기반으로 사용자의 환경 설정을 변경한 다음 온사이트 및 타깃팅에서 이러한 환경 설정을 사용하는 것은 관심사 기반 타깃팅에 필요한 세 가지 요구 사항을 모두 충족하지 않으므로 관심사 기반 타깃팅(개인화라고도 함)으로 적합하지 않습니다. *[이러한 요구 사항에 대해서는 레이블 C5 를 참조하십시오.](#c5)*

궁극적으로, 레이블의 해석 및 해당 레이블이 있는 데이터의 사용이 적용되는 방식은 사용자에게 달려 있습니다. 참조용으로 IAB 및 DAA 프레임워크는 아래에 제공됩니다.

IAB: 3. 광고 선택, 전달, 보고: 정보 수집 및 이전에 수집된 정보와 결합하여 광고를 선택 및 제공하고 그러한 광고의 제공 및 효과를 측정합니다. 여기에는 사용자의 관심사에 대한 정보를 사용하여 광고를 선택하거나, 광고가 표시되는 빈도를 처리하는 데이터, 광고가 표시되는 시간 및 위치, 광고 선택 또는 구매와 같은 광고와 관련된 작업을 수행했는지 여부 등이 포함됩니다. 개인화는 이 서비스를 사용하여 시간에 따라 웹 사이트나 앱과 같은 다른 컨텍스트에서 나중에 광고 및/또는 콘텐츠를 개인화하기 위해 이 서비스를 사용하여 정보를 수집하고 처리하는 데 포함되지 않습니다.

데이터: 온라인 행동 광고에는 자사, 광고 게재 또는 광고 보고 또는 컨텍스트 광고(즉, 방문 중인 웹 페이지의 컨텐츠, 소비자의 현재 웹 페이지 방문 또는 검색 쿼리 기반 광고)의 활동이 포함되지 않습니다.

#### C7 {#c7}

온사이트 컨텐츠는 알림, 교육 또는 오락 기능을 위해 디자인된 텍스트 및 이미지만 제품 또는 서비스 판매를 홍보하도록 만들어지지 않습니다. 컨텐츠가 기본 광고로 자격이 되는지 여부를 포함하여 컨텐츠의 목적을 결정하는 것은 귀하의 책임입니다. C7 레이블은 레이블로 적용되는 온사이트 광고를 표시하지 않습니다 [C6](#c6). C7 레이블이 있는 데이터는 조직의 웹 사이트 또는 앱에서 컨텐츠 선택 및 전달을 포함하여 온사이트 컨텐츠 타깃팅에 사용하거나 그러한 컨텐츠의 전달 및 효과를 측정할 수 없습니다. 여기에는 선택 컨텐츠에서 사용자의 관심사에 대해 이전에 수집된 정보, 표시되는 컨텐츠에 대한 데이터 처리, 표시되는 컨텐츠 빈도 또는 시간, 표시되는 시기 및 위치, 사용자가 컨텐츠 선택 등 컨텐츠와 관련된 작업을 수행했는지 여부 등이 포함됩니다. 일반적으로 해당 사용자의 온사이트 활동을 기반으로 사용자의 환경 설정을 변경한 다음 온사이트 컨텐츠 타깃팅에서 이러한 환경 설정을 사용하는 것은 관심사 기반 타깃팅에 필요한 세 가지 요구 사항을 모두 충족하지 않으므로 관심사 기반 타깃팅(개인화라고도 함)으로 적합하지 않습니다. *[이러한 요구 사항에 대해서는 레이블 C5 를 참조하십시오.](#c5)*

궁극적으로, 레이블의 해석 및 해당 레이블이 있는 데이터의 사용이 적용되는 방식은 사용자에게 달려 있습니다. 참조용으로 IAB 및 DAA 프레임워크는 아래에 제공됩니다.

IAB: 4. 컨텐츠 선택, 전달, 보고: 정보를 수집하고 이전에 수집한 정보와 결합하여 콘텐츠를 선택하여 전달하고, 해당 콘텐츠의 배달과 효과를 측정합니다. 여기에는 사용자의 관심사에 대해 이전에 수집된 정보를 사용하여 컨텐츠를 선택하거나, 컨텐츠가 표시되는 시간 또는 시간, 컨텐츠가 표시되는 시기 및 위치, 컨텐츠 선택 등과 같은 컨텐츠와 관련된 작업을 수행했는지 여부에 대한 데이터 처리가 포함됩니다. 개인화는 포함되지 않습니다. 개인화는 이 서비스를 통해 이후에 웹 사이트나 앱과 같은 다른 컨텍스트에서 콘텐츠를 개인화하고 광고를 개인화하기 위한 정보를 수집하고 처리하는 것입니다.

데이터: 온라인 행동 광고에는 자사, 광고 게재 또는 광고 보고 또는 컨텍스트 광고(즉, 방문 중인 웹 페이지의 컨텐츠, 소비자의 현재 웹 페이지 방문 또는 검색 쿼리 기반 광고)의 활동이 포함되지 않습니다.

#### C8 {#c8}

데이터를 사용하여 조직의 사이트 또는 앱 사용을 측정, 이해 및 보고할 수 없습니다. 여기에는 관심사 기반 타깃팅(사이트 간 타깃팅)이 포함되지 않습니다. 이 타겟팅은 시간이 지남에 따라 웹 사이트나 앱과 같은 다른 서비스에서 콘텐츠를 나중에 개인화하고/또는 광고를 개인화하기 위해 이 서비스를 사용하여 정보를 수집하고 있습니다.

#### C9 {#c9}

일부 계약에는 데이터 과학을 위한 데이터 사용에 대한 명시적 금지 사항이 포함됩니다. 경우에 따라 AI(인공 지능), ML(기계 학습) 또는 모델링을 위해 데이터를 사용할 수 없다는 용어가 있습니다.

#### C10 {#c10}

일부 데이터 거버넌스 정책은 개인화를 위해 결합된 ID 데이터의 사용을 제한합니다. C10 레이블은 병합 정책이 &quot;개인 그래프&quot; 옵션을 사용하는 경우 세그먼트에 자동으로 적용됩니다.

#### C11 {#c11}

Adobe Experience Platform 세그먼트 일치 기능을 사용하면 자사 세그먼트를 개인 정보 및 동의 환경 설정과 일치시켜 보강된 프로파일링 및 다운스트림 인사이트를 제공할 수 있습니다. C11 레이블은 [!DNL Segment Match] 프로세스. 세그먼트 일치에서 제외할 데이터 세트 및/또는 필드를 결정하고 그에 따라 C11 레이블을 추가한 후에는 세그먼트 일치 워크플로우에 의해 레이블이 자동으로 적용됩니다.

#### C12 {#c12}

이 레이블이 있는 데이터는 어떤 식으로든 Platform에서 내보낼 수 없습니다. C12 레이블이 지정된 필드는 CSV 다운로드, API 사용 및 활성화 워크플로우에서 제외됩니다.
