# 지원되는 데이터 사용 레이블

Adobe Experience Platform에는 데이터 사용 레이블 지정 및 실행(DULE)을 핵심 조건으로 하는 데이터 거버넌스 인프라를 포함합니다.  DULE 기능을 사용하면 데이터 세트 및 필드에 데이터 사용 레이블을 적용하여 해당 데이터에 적용되는 사용 정책 유형에 따라 데이터를 분류할 수 있습니다.

이 문서에서는 현재 지원되는 모든 데이터 사용 레이블에 대해 간략하게 설명합니다 [!DNL Experience Platform]. 데이터 거버넌스 개요 [!DNL Data Governance] 에서 [및 DULE에 대한 자세한 내용을 확인할 수 있습니다](../home.md).

## 계약 레이블

계약 &quot;C&quot; 레이블은 계약 의무가 있거나 조직의 데이터 관리 정책과 관련된 데이터를 분류하는 데 사용됩니다.

| 레이블 | 정의 |
|---|---|
| **C1** | 데이터는 개인 또는 장치 식별자를 포함하지 않고 Adobe Experience Cloud에서 집계된 양식으로만 내보낼 수 있습니다. [추가 정보...](#c1) |
| **C2** | 데이터를 타사 파일로 내보낼 수 없습니다. [추가 정보...](#c2) |
| **C3** | 데이터를 결합하거나 직접 식별 가능한 정보와 함께 사용할 수 없습니다. [추가 정보...](#c3) |
| **C4** | 온사이트 또는 사이트 간 광고 또는 컨텐츠를 타깃팅하는 데 데이터를 사용할 수 없습니다. [추가 정보...](#c4) |
| **C5** | 컨텐츠 또는 광고의 관심 기반 크로스 사이트 타깃팅에는 데이터를 사용할 수 없습니다. [추가 정보...](#c5) |
| **C6** | 온사이트 광고 타깃팅에는 데이터를 사용할 수 없습니다. [추가 정보...](#c6) |
| **C7** | 컨텐츠의 온사이트 타깃팅에는 데이터를 사용할 수 없습니다. [추가 정보...](#c7) |
| **C8** | 조직의 웹 사이트 또는 앱을 측정하는 데 데이터를 사용할 수 없습니다. [추가 정보...](#c8) |
| **C9** | 데이터 과학 워크플로우에서는 데이터를 사용할 수 없습니다. [추가 정보...](#c9) |
| **C10** | 연결된 ID 활성화에는 데이터를 사용할 수 없습니다. [추가 정보...](#c10) |

## ID 레이블

ID &quot;I&quot; 레이블은 특정 개인을 식별하거나 연락할 수 있는 데이터를 분류하는 데 사용됩니다.

| 레이블 | 정의 |
|---|---|
| **I1** | 장치가 아닌 특정 사용자를 식별하거나 연락할 수 있는 직접 식별 가능한 데이터. |
| **I2** | 특정 개인을 식별하거나 접촉하기 위해 다른 데이터와 함께 사용할 수 있는 간접적으로 식별 가능한 데이터입니다. |

## 민감한 레이블

민감한 &quot;S&quot; 레이블은 사용자와 조직에서 민감한 데이터를 분류하는 데 사용됩니다.

민감한 데이터라고 간주할 수 있는 데이터 유형은 서로 다를 수 있습니다. 하지만 이 카테고리는 지리적 데이터에만 국한되지 않습니다.

| 레이블 | 정의 |
|---|---|
| **S1** | 장치의 정확한 위치를 결정하는 데 사용할 수 있는 위도 및 경도를 지정하는 데이터. |
| **S2** | 광범위하게 정의된 위치 영역을 결정하는 데 사용할 수 있는 데이터 |

## 부록

아래 섹션에서는 사용 가능한 데이터 사용 레이블에 대한 추가 정보를 제공합니다.

### 계약 레이블 세부 정보

다음 섹션에서는 특정 계약 &quot;C&quot; 레이블의 구현과 관련된 세부 정보를 제공합니다.

#### C1 {#c1}

일부 데이터는 Adobe Experience Cloud에서 개인 또는 장치 식별자를 포함하지 않고 집계된 양식으로만 내보낼 수 있습니다. 예를 들어 소셜 네트워크에서 가져온 데이터입니다.

#### C2 {#c2}

일부 데이터 제공업체는 원래 데이터가 수집된 위치에서 내보내는 것을 금지하는 계약을 가지고 있습니다. 예를 들어, 소셜 네트워크 계약에서는 사용자가 얻은 데이터의 전송을 종종 제한합니다. C2 레이블은 [C1보다](#c1)더 제한적이며, 이 경우 집계 및 익명 데이터만 필요합니다.

#### C3 {#c3}

일부 데이터 제공업체는 계약에 직접 식별 가능한 정보로 해당 데이터를 조합하거나 사용하는 것을 금지하는 약관을 가지고 있습니다. 예를 들어, 광고 네트워크, 광고 서버 및 타사 데이터 제공업체에서 가져온 데이터에 대한 계약에는 이러한 데이터를 직접 식별할 수 있는 데이터와 함께 사용하는 특정 계약 금지 규정이 포함되어 있습니다.

#### C4 {#c4}

C4는 레이블 [C5](#c5), [C6](#c6)및 [C7을 포함하는 가장 제한적인 레이블입니다](#c7).

#### C5 {#c5}

다음 세 가지 조건이 충족되면 관심 기반 타깃팅 또는 개인화가 발생합니다. 온사이트에서 수집된 데이터는 (1) 사용자의 관심사에 대한 참조를 위해 사용되고, (2) 다른 사이트나 앱(오프라인) 등의 다른 컨텍스트에서 사용되고, (3) 이러한 참조를 기반으로 제공되는 컨텐츠나 광고를 선택하는 데 사용됩니다.

사이트 내 데이터와 외부 데이터의 조합 또는 여러 외부 소스의 데이터 조합을 포함하여 여러 사이트의 데이터를 결합하는 것을 사이트 간 데이터라고 합니다. 사이트마다 컨텍스트가 다르기 때문에 모든 컨텍스트에서 사이트 간 데이터 사용이 원본과 다를 수 있습니다. 사이트 간 데이터는 일반적으로 사용자 관심사에 대한 참조를 위해 수집 및 처리됩니다. 따라서 광고 또는 컨텐츠가 사이트 내 또는 외부 사이트에 표시되는지 여부에 관계없이 일반적으로 광고 또는 컨텐츠를 타깃팅하기 위해 사이트 간 데이터를 사용하는 것은 관심 기반 타깃팅으로 간주됩니다. 예를 들어, 온사이트 데이터가 오프사이트 데이터와 함께 사용되어 조직의 고유 사이트에 사용자를 표시할 광고를 선택한 경우 이를 사용하면 관심 기반 타깃팅으로 자격을 얻게 됩니다. 또 다른 예로, 오프사이트 사용자에게 광고를 다시 타깃팅하면 관심 기반 타깃팅으로 간주될 수 있습니다.

오프사이트 데이터를 타깃팅하기 위해 단독으로 사용하는 경우에도 사용자의 관심사에 대한 참조를 위해 일반적으로 수집 및 처리되므로 관심 기반 타깃팅으로 간주할 수 있습니다.

그러나 사이트 내 데이터만 사용하여 컨텐츠 또는 광고를 타깃팅하면 일반적으로 관심 기반 타깃팅으로 간주되지 않습니다. 그렇지 않으면 관심 기반 타깃팅으로 적합하지 않은 온사이트 타깃팅은 두 개의 서로 다른 레이블로 처리됩니다. 특히 Label C6는 사이트 내 광고 타깃팅 및 보고를 해결하고, 광고 선택, 전달 및 보고에 대해 구체적으로 설명하며, Label C7은 사이트 내 컨텐츠 선택, 전달 및 보고(사이트 내 컨텐츠 타깃팅)를 해결합니다.

궁극적으로 레이블의 해석 및 해당 레이블에 있는 데이터의 사용 방법은 사용자에게 달려 있습니다. 참조용으로 IAB 및 DAA 프레임워크는 아래에 제공됩니다.

IAB: 개인화 귀하가 서비스를 사용하는 것에 대한 정보를 수집하고 처리하여 다른 웹사이트 또는 응용 프로그램과 같은 다른 컨텍스트에서 나중에 귀하를 위해 광고 및/또는 컨텐트를 개인화합니다. 일반적으로 사이트 또는 앱의 컨텐츠는 향후 광고 및/또는 컨텐츠의 선택을 알리게 하는 귀하의 관심사에 대해 추론하는 데 사용됩니다.

DAA: 온라인 행동 광고 이러한 데이터를 사용하여 사용자의 선호도 또는 관심사를 예측하여 이러한 웹 보기 동작에서 유추된 기본 설정 또는 관심사를 기반으로 해당 컴퓨터 또는 장치에 광고를 전달하기 위한 목적으로 특정 컴퓨터 또는 장치에서 시간의 경과에 따른 웹 보기 동작과 관련된 데이터를 수집합니다.

#### C6 {#c6}

광고는 주로 상품이나 서비스의 판매를 촉진하려는 웹사이트 또는 앱에 나타나는 텍스트 및 이미지를 포함한 메시지 또는 알림입니다. 이러한 메시지 또는 알림의 용도를 결정하는 것은 전적으로 사용자의 책임입니다. 광고는 사이트 내 컨텐츠와 별개이며 레이블 [C7이 포함됩니다](#c7). C6 레이블이 있는 데이터는 조직의 웹사이트 또는 앱에서 광고를 선택 및 배달하거나 이러한 광고의 전달 및 효과를 측정하는 등 현장 광고 타깃팅에 사용할 수 없습니다. 여기에는 사용자의 관심사에 대한 이전에 수집된 사이트 내 데이터를 사용하여 광고를 선택하거나, 광고가 표시된 시기 및 위치에 대한 데이터를 처리하고, 사용자가 광고를 클릭하거나 구매하는 등 광고와 관련된 조치를 취했는지 여부가 포함됩니다. 일반적으로 사용자의 온사이트 활동을 기반으로 사용자의 선호도를 파악한 다음 사이트 내 광고 타깃팅에서 이러한 기본 설정을 사용하는 경우 관심 기반 타깃팅에 필요한 세 가지 요구 사항을 모두 충족하지는 않기 때문에 관심 기반 타깃팅(개인화라고도 함)으로 적합하지 않습니다. _[이러한 요구 사항은 레이블 C5를 참조하십시오.](#c5)_

궁극적으로 레이블의 해석 및 해당 레이블에 있는 데이터의 사용 방법은 사용자에게 달려 있습니다. 참조용으로 IAB 및 DAA 프레임워크는 아래에 제공됩니다.

IAB: 3. 광고 선택, 전달, 보고: 정보를 수집하고 이전에 수집한 정보와 결합하여 광고를 선택 및 전달하고 이러한 광고의 전달 및 효과를 측정할 수 있습니다. 여기에는 광고를 선택하기 위해 이전에 수집한 귀하의 관심 사항에 대한 정보를 사용하는 것, 표시되는 광고에 대한 데이터 처리, 표시되는 빈도, 광고 표시 시기 및 위치, 광고 클릭 또는 구매 등과 같이 광고와 관련된 조치를 취했는지 여부 등이 포함됩니다. 여기에는 개인화가 포함되어 있지 않습니다. 개인화는 귀하가 해당 서비스를 사용하여 웹사이트 또는 앱과 같은 다른 컨텍스트에서 나중에 귀하를 위해 광고 및/또는 컨텐트를 개인화하기 위한 정보를 수집하고 처리하는 것입니다.

DAA: 온라인 행동 광고는 제1자, 광고 전달 또는 광고 보고의 활동이나 컨텍스트 광고(즉, 방문하는 웹 페이지의 컨텐츠, 소비자의 현재 웹 페이지 방문 또는 검색 쿼리 기반 광고)를 포함하지 않습니다.

#### C7 {#c7}

사이트 내 컨텐츠는 상품 또는 서비스의 판매를 촉진하기 위해 만들어진 것이 아니라, 정보, 교육 또는 오락 목적으로 만들어진 텍스트와 이미지입니다. 컨텐츠가 네이티브 광고로 자격이 되는지 여부를 포함하여 컨텐츠의 용도를 결정하는 것은 전적으로 귀하에게 달려 있습니다. C7 레이블은 레이블 [C6로 구성된 온사이트 광고를 덮기 위한 것이 아닙니다](#c6). C7 레이블이 있는 데이터는 조직의 웹 사이트 또는 앱에서 컨텐츠를 선택 및 배달하거나 이러한 컨텐츠의 전달 및 효과를 측정하는 등 온사이트 컨텐츠 타깃팅에 사용할 수 없습니다. 여기에는 선택한 컨텐츠에 대한 사용자의 관심사, 표시되는 컨텐츠에 대한 데이터 처리, 표시되는 빈도 또는 시간, 표시되는 시간과 장소, 사용 시 컨텐츠 클릭 등 컨텐츠와 관련된 작업이 수행되었는지 등이 포함됩니다. 일반적으로 사용자의 온사이트 활동을 기반으로 사용자의 선호도를 파악한 다음 사이트 내 컨텐츠 타깃팅에서 이러한 기본 설정을 사용하는 경우 관심 기반 타깃팅에 필요한 세 가지 요구 사항을 모두 충족하지는 않기 때문에 관심 기반 타깃팅(개인화라고도 함)으로 적합하지 않습니다. _[이러한 요구 사항은 레이블 C5를 참조하십시오.](#c5)_

궁극적으로 레이블의 해석 및 해당 레이블에 있는 데이터의 사용 방법은 사용자에게 달려 있습니다. 참조용으로 IAB 및 DAA 프레임워크는 아래에 제공됩니다.

IAB: 4. 컨텐츠 선택, 전달, 보고: 정보를 수집하고 이전에 수집한 정보와 결합하여 사용자를 위해 컨텐츠를 선택 및 전달하고 이러한 컨텐츠의 전달 및 효과를 측정할 수 있습니다. 여기에는 귀하의 관심사에 대해 이전에 수집한 정보를 사용하여 컨텐트를 선택하거나, 표시되는 컨텐트에 대한 데이터 처리, 표시되는 빈도 또는 시간, 컨텐트가 표시된 시기 및 위치, 컨텐츠 클릭 등 컨텐츠와 관련된 작업을 수행했는지 여부 등이 포함됩니다. 여기에는 개인화가 포함되어 있지 않습니다. 개인화는 귀하가 서비스를 사용해서 컨텐트 및/또는 광고를 차후에 웹사이트 또는 앱과 같은 다른 컨텍스트에서 귀하에게 개인화하기 위한 정보를 수집하고 처리하는 것입니다.

DAA: 온라인 행동 광고는 제1자, 광고 전달 또는 광고 보고의 활동이나 컨텍스트 광고(즉, 방문 중인 웹 페이지의 컨텐츠, 소비자의 현재 웹 페이지 방문 또는 검색 쿼리)를 포함하지 않습니다.

#### C8 {#c8}

조직의 사이트 또는 앱의 사용자 사용을 측정, 이해 및 보고하는 데 데이터를 사용할 수 없습니다. 여기에는 시간 경과에 따라 컨텐트 및/또는 광고를 다른 컨텍스트(예: 웹사이트 또는 앱)에서 나중에 개인화하기 위해 이 서비스를 사용하는 것에 대한 정보를 수집하는 관심 기반 타깃팅(교차 사이트 타깃팅)이 포함되지 않습니다.

#### C9 {#c9}

일부 계약에는 데이터 과학에 대한 데이터 사용에 대한 명시적 금지 사항이 포함되어 있습니다. 경우에 따라 AI(인공 지능), 머신 러닝(ML) 또는 모델링에 대한 데이터 사용을 금지하는 용어로 표현됩니다.

#### C10 {#c10}

일부 데이터 사용 정책은 개인화를 위해 연결된 ID 데이터의 사용을 제한합니다. 병합 정책이 &quot;비공개 그래프&quot; 옵션을 사용하는 경우 C10 레이블이 세그먼트에 자동으로 적용됩니다.
