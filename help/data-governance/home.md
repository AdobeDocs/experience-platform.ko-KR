# 데이터 거버넌스 개요

Adobe Experience Platform의 핵심 기능 중 하나는 여러 엔터프라이즈 시스템의 데이터를 통합하여 마케터가 고객을 식별하고 이해하고 참여를 유도할 수 있도록 지원하는 것입니다. 이 데이터는 조직 또는 법률 규정에 의해 정의된 사용 제한 사항의 적용을 받을 수 있습니다. 따라서 [!DNL Platform] 내의 데이터 작업이 데이터 사용 정책을 준수하도록 하는 것이 중요합니다.

Adobe Experience Platform [!DNL Data Governance]을 사용하면 고객 데이터를 관리하고 데이터 사용에 적용되는 규정, 제한 사항 및 정책을 준수할 수 있습니다. 카탈로그 작성, 데이터 계보, 데이터 사용 표시, 데이터 사용 정책, 데이터 사용 정책, 마케팅 작업에 대한 데이터 사용 제어 등 다양한 수준에서 [!DNL Experience Platform] 내에서 주요 역할을 합니다.

## 데이터 거버넌스 역할

개념 측면에서, 데이터 거버넌스는 자동도 아니며 공백에서 발생하지도 않습니다. 데이터 거버넌스 에코시스템이 확장됨에 따라 일반적으로 데이터 관리자로 인식되는 한 개인의 역할에서 시작된 것이 크게 증가했습니다. 오늘날 데이터 거버넌스는 성공적인 데이터 관리를 위해 지속적인 관리 및 모니터링이 필요하며, 데이터 레이블을 적절하게 지정할 수 있고 사용 정책을 만들 수 있으며 이러한 정책을 준수할 수 있는 툴을 데이터 관리자에 의존합니다.

데이터 거버넌스는 조직 내 모든 개인의 책임이어야 하지만, 다음은 데이터 거버넌스 주기 내에서 중요한 역할들입니다.

![데이터 거버넌스 역할](./images/overview/roles.png)

### 데이터 관리자

데이터 프로세스는 데이터 거버넌스의 핵심입니다. 이 역할은 규정, 계약 제한 및 정책을 해석하고 이를 데이터에 직접 적용하는 책임을 집니다. 이러한 규정, 제한 사항 및 정책을 이해함으로써 데이터 관리자의 역할은 다음과 같습니다.

* 데이터, 데이터 세트 및 데이터 샘플을 검토하여 메타데이터 사용 레이블 지정을 적용하고 관리합니다.
* 데이터 정책을 만들어 데이터 세트 및 필드에 적용합니다.
* 데이터 정책을 조직에 통신

### 마케터

마케터는 데이터 거버넌스의 종단점입니다. 데이터 관리자, 과학자 및 엔지니어가 만든 데이터 거버넌스 인프라에서 데이터를 요청합니다. 마케터는 다음을 포함하여 마케팅 우산 아래 다양한 전문 분야를 포함합니다.

* 마케팅 분석가는 개인 및 그룹(세그먼트라고도 함)으로 고객을 이해할 수 있도록 데이터를 요청합니다.
* 마케팅 전문가 및 경험 디자이너는 데이터를 사용하여 새로운 고객 경험을 디자인합니다.


## [!DNL Data Governance] 프레임워크

[!DNL Data Governance] 프레임워크는 데이터를 분류하고 데이터 사용 정책을 만드는 과정을 간소화하고 능률화합니다. 데이터 레이블이 적용되고 데이터 사용 정책이 적용되면 마케팅 작업을 평가하여 데이터를 올바르게 사용할 수 있습니다.

[!DNL Data Governance] 프레임워크에는 다음 세 가지 주요 요소가 있습니다.레이블, 정책 및 적용.

1. **레이블:** 규정 및 조직 정책을 준수하려면 개인 정보 관련 고려 사항 및 계약 조건을 반영하는 데이터를 분류합니다.
1. **정책:** 특정 데이터에 대해 허용되거나 허용되지 않는 마케팅 작업의 종류를 설명합니다.
1. **적용:** 정책 프레임워크를 사용하여 다양한 데이터 액세스 패턴에 대한 정책을 조언하고 적용합니다.

## 데이터 사용 레이블

[!DNL Data Governance] 데이터 관리자가 데이터 세트 및 필드 수준에서 사용 레이블을 적용하여 적용되는 정책 유형에 따라 데이터를 분류할 수 있습니다.

[!DNL Data Governance] 프레임워크에는 데이터를 3가지 방법으로 분류하는 데 사용할 수 있는 사전 정의된 데이터 사용 레이블이 포함되어 있습니다.

![데이터 사용 레이블 범주](./images/overview/label-categories.png)

* **계약 &quot;C&quot; 데이터 레이블: 계약** 의무가 있거나 고객 데이터 관리 정책과 관련된 데이터를 레이블을 지정하고 분류합니다.
* **ID &quot;I&quot; 데이터 레이블:** 특정 개인을 식별하거나 연락할 수 있는 데이터를 레이블을 지정하고 분류합니다.
* **민감한 &quot;S&quot; 데이터 레이블:** 지리적 데이터와 같은 민감한 데이터와 관련된 데이터를 레이블 및 분류합니다.

>[!NOTE]
>
>사용 가능한 레이블의 전체 목록과 각 레이블 유형에 대한 정의는 [지원되는 데이터 사용 레이블](labels/reference.md)의 안내서를 참조하십시오.

레이블은 언제든지 적용할 수 있으므로 데이터 관리 방식을 유연하게 선택할 수 있습니다. 데이터를 [!DNL Experience Platform]으로 인제스트되거나 [!DNL Platform]에서 데이터를 사용할 수 있게 되는 즉시 레이블 지정 데이터를 권장합니다.

자세한 내용은 [데이터 사용 레이블](./labels/overview.md)에 대한 개요를 참조하십시오.

## 데이터 사용 정책

데이터 사용 레이블이 데이터 규정 준수를 효과적으로 지원하기 위해서는 데이터 사용 정책을 구현해야 합니다. 데이터 사용 정책은 [!DNL Experience Platform] 내의 데이터에 대해 수행하도록 허용되거나 제한된 마케팅 작업의 종류를 설명하는 규칙입니다.

마케팅 작업의 예는 데이터 세트를 제3자 서비스로 내보내려는 것일 수 있습니다. PII(개인 식별 정보)와 같은 특정 유형의 데이터를 내보낼 수 없고 데이터 세트에 &quot;I&quot; 레이블(ID 데이터)이 적용된 경우 데이터 사용 정책을 위반했음을 알리는 [!DNL Policy Service]의 응답을 받게 됩니다.

데이터 사용 레이블이 적용되면 데이터 관리자는 [!DNL Policy Service] API 또는 [!DNL Experience Platform] 사용자 인터페이스를 사용하여 정책을 만들 수 있습니다.

>[!IMPORTANT]
>
>모든 데이터 사용 정책(Adobe에서 제공하는 핵심 정책 포함)은 기본적으로 비활성화됩니다. 개별 정책을 적용하려면 해당 정책을 수동으로 활성화해야 합니다.

데이터 사용 정책 및 마케팅 작업에 대한 자세한 내용은 [정책 개요](./policies/overview.md)를 참조하십시오.

### [!DNL Data Governance] 용어

다음 표에서는 [!DNL Data Governance] 및[!DNL Data Governance] 프레임워크와 관련된 주요 용어에 대해 개괄적으로 설명합니다.

| 용어 | 정의 |
|---|---|
| **계약 레이블** | 계약 &quot;C&quot; 레이블은 계약 의무가 있거나 조직의 데이터 관리 정책과 관련된 데이터를 분류하는 데 사용됩니다. |
| **사이트 간 데이터** | 사이트 간 데이터는 사이트 내 데이터와 외부 데이터의 조합 또는 여러 외부 소스의 데이터 조합을 포함하여 여러 사이트의 데이터를 조합하는 것입니다. |
| **데이터 거버넌스** | 데이터 거버넌스는 데이터 사용과 관련된 규정 및 기업 정책을 준수하도록 하는 데 사용되는 전략과 기술을 포함합니다. |
| **데이터 관리자** | 데이터 관리자는 조직의 데이터 에셋을 관리, 감독 및 관리하는 담당자입니다. 또한 데이터 관리자는 데이터 거버넌스 정책을 안전하게 보호하고 정부 규정 및 조직 정책을 준수하도록 유지 관리할 수 있습니다. |
| **데이터 사용 레이블** | 데이터 사용 레이블은 사용자에게 개인 정보 관련 고려 사항 및 계약 조건을 반영하는 데이터를 분류하여 규정 및 기업 정책을 준수하도록 합니다. |
| **데이터 세트 레이블** | 데이터세트에 레이블을 추가할 수 있습니다. 데이터 세트 내의 모든 필드는 데이터 세트의 레이블을 상속합니다. |
| **필드 레이블** | 필드 레이블은 데이터 세트에서 상속되거나 필드에 직접 적용되는 데이터 거버넌스 레이블입니다.  필드에 적용된 데이터 거버넌스 레이블은 데이터 세트에 상속되지 않습니다. |
| **Geofence** | 지오펜스는 GPS 또는 RFID 기술로 정의된 가상 지리적 경계로, 모바일 장치가 특정 지역에 들어오거나 나갈 때 소프트웨어가 응답을 트리거할 수 있습니다. |
| **ID 레이블** | ID &quot;I&quot; 레이블은 특정 개인을 식별하거나 연락할 수 있는 데이터를 분류하는 데 사용됩니다. |
| **관심 기반 타깃팅** | 다음 3가지 조건이 충족되면 개인화라고도 하는 관심 기반 타깃팅이 발생합니다.온사이트에서 수집된 데이터는 사용자의 관심사에 대한 참조를 위해 사용되며 다른 사이트 또는 앱(오프라인) 등의 다른 컨텍스트에서 사용되며 이러한 참조를 기반으로 제공되는 컨텐트 또는 광고를 선택하는 데 사용됩니다. |
| **마케팅 활동** | 데이터 거버넌스 프레임워크의 컨텍스트에서 마케팅 작업은 데이터 사용 정책 위반을 확인해야 하는 [!DNL Experience Platform] 데이터 소비자가 취하는 작업입니다 |
| **정책** | 데이터 거버넌스 프레임워크에서 정책은 특정 데이터에 대해 허용되거나 허용되지 않는 마케팅 작업의 종류를 설명하는 규칙입니다. |
| **중요 레이블** | 민감한 &quot;S&quot; 레이블은 사용자와 조직에서 민감한 데이터를 분류하는 데 사용됩니다. |
